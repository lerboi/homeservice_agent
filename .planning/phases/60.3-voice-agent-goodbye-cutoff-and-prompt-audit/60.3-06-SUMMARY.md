## Decisions Made

- **EN body: verbatim preservation over D5/D6 compression.** The 60.3-PROMPT-AUDIT.md flagged Rule 3's "cancellation" rationale as VAD-redundant (the 60.2 Fix G `silence_duration_ms=1500` now handles cancellation-from-silence) and the 39-line section as token-economy-candidate. However, the post-60.2 EN body is the source of the Pitfall 6 invariant that ALL six Plan 6 EN-branch tests anchor on. Reworking it to trim D5/D6 would require re-grounding the invariants, risking regression. The audit explicitly noted this tradeoff as acceptable: "the 60.2 invariants are stronger than D6 token economy for this section" (Plan 06 action block, line 174). **D5/D6 deferred; if a future UAT justifies compression, do so as an isolated follow-up.**
- **Signature without default locale.** Plan 06 explicitly specified `(locale: str)` — no `= "en"` default. The plan acknowledged this requires touching the two 60.2 guard-rail tests in `tests/test_prompt.py`. Kept this choice: forces all callers to pass `locale` explicitly, eliminates silent-default-locale bugs if build_system_prompt's locale threading ever regresses.
- **USTED register for ES body.** Consistent with Plan 05 call_duration block and the existing info_gathering Spanish block. Same rationale as Plan 05: in-prompt register consistency so the model doesn't flip between formal/informal 2nd-person across sections.
- **Tool names NOT translated.** `check_availability`, `book_appointment`, `capture_lead`, `transfer_call` are code identifiers wired to the actual tool registry in `src/tools/`. A Spanish-translated name would be ungrounded (no tool would match). Test 7 asserts they appear literally in the ES body.
- **Updated Plan 05's forward-looking test.** Plan 05's `test_section_is_position_5_or_earlier_in_both_locales` included a comment: "tool_narration header is English-only in the current codebase (Plan 6 will add the es branch). For locale='es' the assembled prompt still contains the English tool_narration header — this is expected pre-Plan-6." Plan 06 fulfills that expectation. Updated the test to use the locale-appropriate header instead of the English header for both cases. This is a Plan-05-anticipated update, not a drift.
