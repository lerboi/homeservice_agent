<objective>
Run the first UAT call against the Plan-1-instrumented Railway deployment, capture the `[goodbye_race]` diagnostic evidence, and produce a short decision doc that picks the Stream A fix candidate — Branch P (prompt-hardening of `_build_call_duration_section`) OR Branch G (tool-level `end_call` pre-guard on audio playout).

**This plan has an unusual shape:** Task 1 is a manual-only blocking checkpoint that serializes on owner availability (real Twilio call). Task 2 is an autonomous writeup that converts the captured evidence into a 2–3-paragraph decision doc.

Purpose: CONTEXT D-A-05 mandates the fix be picked from evidence, not blind prompt hardening. Without this iteration step, Plan 3 could ship a Branch P fix where evidence actually supports Branch G, wasting an owner UAT cycle.

Output: `60.3-HUMAN-UAT.md` populated with a "Stream A — UAT #1" section; `60.3-STREAM-A-ANALYSIS.md` new file with the fix selection.
</objective>
