## Decisions Made

- **Invariant-lock plan classification.** Unlike Plans 05-09 (each added an `if locale == "es":` branch to an English-only section), Plan 10 operated on a section that already had inner locale parity — the goal was 'verify what exists + compress where flagged + extend where incomplete'. First plan in Stream B of this shape; Plan 11 (`_build_booking_section`, 85% English-only) will be a hybrid with similar mechanics for the partial-parity blocks.
- **D6 compression dropped the language-switch clause, nothing else.** The audit flagged a specific redundant clause (`"This applies in every language..."`) as covered by `_build_language_section`. Every other line in the section earns its place — over-compression would risk D1/D2 drift. Compressed the single flagged sentence, applied symmetrically to both locales.
- **Phone-readback added as Rule 2 (auto-add missing critical functionality).** The prompt had no mid-call guidance on alternate phone numbers — a real anti-hallucination gap. If a caller offers a different callback number, the model previously had no STATE+DIRECTIVE framing for how to handle it. Added to both locales in the same commit.
- **postal_label wired rather than removed.** The parameter was dead in the section body. Wire vs remove tradeoff: wiring preserves market parity (SG vs US), signature stability (`build_system_prompt` call site unchanged), and closes a latent D7 gap. Removing would have been a breaking change with no benefit.
- **ES register: USTED (consistent with pre-60.3 inner blocks).** info_gathering's inner name_use + service_address blocks already used USTED (`Haga una pregunta`, `Extraiga`, `Capture`). Plan 10 extends USTED to the new outer frame (`Debe tener`, `Clasifique la urgencia`, `Mida la gravedad`). The register-consistency here is _internally_ clean within this section. Broader ES register normalization across the whole prompt (USTED vs TÚ inconsistency across Plans 05/06/07 USTED vs Plans 08/09 TÚ) still pending for Plan 12 or a dedicated polish plan.
- **Phase 60 lockdown via lowercased substring tests, not exact-phrase matching.** `"one" + "at a time"` admits `"one piece at a time"`, `"one focused thing at a time"`, `"one question at a time"`. Allows prose flexibility while catching real drift where the single-question invariant is dropped entirely. Same pattern applied to ES (`"una"` + (`"a la vez"` OR `"por vez"`)).
- **30% length-drift tolerance is empirically generous.** Spanish prose for equivalent content is typically ~15-25% longer than English. Setting the tolerance to 30% avoids flagging healthy natural-prose expansion while catching any deliberate one-sided compression. Current delta 14% — well within budget.
- **Test row count: 9 tests → 10 collected** due to `pytest.mark.parametrize` split. Plan spec listed 9 behavioral requirements; the postal_label parametrization has 2 rows (SG + US), so pytest collects 10 test invocations. Letter-of-spec compliance via the 9 logical behaviors; pytest implementation detail is the 10-test count.
- **No downstream test updates required.** No existing test calls `_build_info_gathering_section` directly. The signature is unchanged (still `(t, postal_label, locale="en")`); only the body semantics changed.

## Deviations from Plan

### Auto-added content (Rule 2: missing critical functionality)

**1. [Rule 2 - Security/Correctness] Phone-readback block added to both locales**
- **Found during:** Task 1 RED (tests `test_en_phone_readback_present` / `test_es_phone_readback_present` forced the issue — section had no phone intake prose at all).
- **Issue:** The prompt previously had no mid-call guidance on alternate callback numbers. The caller's CallerID phone is captured at session start (`customer_context`), but if a caller says "actually, call me back on 555-1234 instead" mid-call, the model had no anti-hallucination frame for digit capture. Opens a risk where the model could fabricate or misremember digits.
- **Fix:** New `PHONE NUMBER` / `NÚMERO DE TELÉFONO` block in both locales: STATE (already captured from CallerID) + DIRECTIVE (alternate number → digit-by-digit readback, confirm before saving, never fabricate).
- **Files modified:** `src/prompt.py`.
- **Commit:** `b2b1027`.

**2. [Rule 2 - Correctness] postal_label wired into section body**
- **Found during:** Task 1 RED (test `test_postal_label_parametrized_both_locales` forced the issue — `postal_label` was in the signature but never used in the body).
- **Issue:** Dead parameter. The section ignored the SG vs US market distinction at the info-gathering layer; only `_build_booking_section` consumed it. Meant the model had no field-name guidance for the address intake phase in US markets vs SG markets.
- **Fix:** EN service_address block uses `f"{postal_label}"` in the fields enumeration. ES block surfaces the EN market label in a parenthetical (for model-side routing continuity) but uses 'código postal' idiomatically in caller-facing prose.
- **Files modified:** `src/prompt.py`.
- **Commit:** `b2b1027`.

Both additions are symmetric across locales — no one-sided parity regression. Both are flagged in the `feat(60.3-10)` commit body.
