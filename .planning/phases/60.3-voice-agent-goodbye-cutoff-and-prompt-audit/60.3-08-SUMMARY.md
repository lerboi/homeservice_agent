## Decisions Made

- **EN body: verbatim preservation over D2 negation-reframe.** The audit flagged `"Never reference, compare with, or fall back to the earlier incorrect version"` as a candidate for positive reframing (`"Only reference the corrected version going forward"`). This section is the anti-hallucination spine — R-B5 Assumption A1 says negations on realtime models can invert, but the current EN text has been in production through Phases 60.1/60.2/60.3 without reported correction-handling regressions. Reframing the load-bearing NEVER clauses without concrete UAT evidence of negation-inversion is a regression risk, not a safety win. Plan 08 explicitly treats this as a D7-primary + D4-preserve plan per the audit's Plan Assignment Summary row ("D2 reviewed — D7 primary"). Deferred.
- **Mixed TÚ/USTED register in ES body.** The plan's action block used TÚ forms throughout (repetiste, tu versión, pídele). Plans 05/06/07 adopted USTED for consistency with the pre-existing info_gathering ES block. Plan 08 kept the plan's TÚ verbatim because:
  - Tests assert specific phrases ("descarta completamente", "siempre correcta", etc.) that are register-neutral.
  - The register mismatch is contained inside this one section and doesn't cross-contaminate other sections.
  - Normalizing to USTED now would require re-running the plan's test authoring; better to batch all ES register normalizations in a single later pass (Plan 12 or a follow-up polish plan) so the register flip is auditable end-to-end.
  Documented as a Deviation below.
- **Spanish address example chosen for structural parallelism.** "Calle Principal 123" / "Avenida Roble 456" mirrors EN "123 Main Street" / "456 Oak Avenue": common street-name root + numeric suffix, swapped-out variant. Spanish conventionally places the number after the street name ("Calle Principal 123"), so the number-position differs from EN but the semantic pairing is identical. Plan spec required both literal strings; tests enforce both.
- **Gender-tolerant `always correct` assertion.** The plan's test spec allowed only masculine forms (`"siempre correcto"`, `"siempre es correcto"`). The ES body correctly uses feminine `"correcta"` because it modifies `"corrección"` (feminine noun). Widened the assertion to accept both genders before committing GREEN — this is a plan-spec correction, not a code change. Rule 1 auto-fix (plan assertion was grammatically wrong).
- **Signature without default locale.** `(locale: str)` matches Plans 06/07. `build_system_prompt` already threads `locale` as a positional through all builders; no silent-default risk.
- **No downstream test updates required.** No existing test calls `_build_corrections_section()` directly — the signature change is contained to `prompt.py`'s call site in `build_system_prompt`. Clean, minimal change.
