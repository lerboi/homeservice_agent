## Top 3 Findings Influencing Downstream Patches

1. **Plan 05 owns the UAT #2 decline-case gap (D1 anti-hallucination on `_build_call_duration_section`).** Current CRITICAL RULE enforces two-step farewell *mechanics* (speak → silence → separate-turn end_call) but does NOT mandate farewell *content*. On UAT #2, agent said only "I understand." and invoked end_call without any farewell phrase. Plan 05 must add a STATE+DIRECTIVE clause: "every end_call requires a farewell phrase. `'I understand.'`, `'Okay.'`, `'Got it.'` are acknowledgments, not farewells." Plus a decline-path example alongside the existing happy-path WRONG/RIGHT example, plus Spanish branch.

2. **Locale parity is the single largest cross-cutting issue.** D7 scored ✗ on 13 of 15 active sections. Spanish callers receive 85% English prose (only info_gathering has full Spanish branch; booking has partial — readback only). Plan 12 must batch 8 sections' locale branches; Plans 05-11 each add Spanish for their targeted section. The inline-branch strategy (vs JSON extraction) is locked.

3. **Two VAD-redundant directives identified (D5 ✗ on voice_behavior + tool_narration).** `voice_behavior`'s "After the caller answers, briefly acknowledge what you heard before moving forward" duplicates Fix G VAD `silence_duration_ms=1500` turn-boundary semantics. `tool_narration` Rule 3's cancellation-avoidance rationale is stale post-Fix-G (cancellations were the pre-60.2 problem the VAD now prevents). Plan 07 drops the VAD-redundant pacing; Plan 06 updates the rationale to tool-latency coverage (keeping the "aim for ~3 seconds" directive itself).

## Decisions Made

- **Locale-parity strategy:** inline `if locale == "es":` branches. Rationale: mirrors the existing `_build_info_gathering_section` gold-standard; single source of truth; no JSON refactor cost; `t()` stays reserved for small reused strings. Exception: small dynamic label dicts (e.g., day-short names) may live at module level if shared across sections.
- **Plan 05 scope expansion:** UAT #2 farewell-content gap is owned by `_build_call_duration_section` patch (not `_build_outcome_words_section` extension). Escalation path: if post-Plan-05 UAT shows the gap persists, add "farewell" to outcome_words reserved-word list as a Plan 09 extension.
- **Customer_account reorder flag:** at position 11 it's a CRITICAL RULE far from the top-6 attention zone. Flagged in Plan 12 batch as a 1-line reorder candidate (move to position 6 after tool_narration). Low-priority — the section is conditionally rendered and lookup-driven.
- **Negation audit:** only 2 of 7 NEVERs are safely reframe-able (corrections "never reference earlier version" → "only reference corrected version"; booking "never read out slot times" → already covered by positive clause). Anti-hallucination NEVERs stay negated.
