## Decisions Made

- **EN body: verbatim preservation over audit compression.** D1 is the HIGHEST-stakes dimension in the entire prompt. Every reserved-word clause + the 3pm failure-mode example is load-bearing. Paraphrasing any of it without concrete UAT evidence that the current text is failing is a regression risk that outweighs any D2 advisory-tone or D6 length-compression benefit. Same logic governed Plan 08's corrections decision.
- **D7 (locale parity) — primary objective.** The audit's "Dimension × Section" matrix marks outcome_words as the single most-important D7 gap remaining in the top-attention zone. Closing this gap means Spanish callers now receive the same anti-false-confirmation protocol English callers get — no more locale regression surface on the prompt's most-important invariant.
- **Tool names NOT translated in ES — codified as a test assertion.** `check_availability` and `book_appointment` are code identifiers wired to `src/tools/` registry. Translating them would break model→tool dispatch. Plan 06 established the convention in prose for tool_narration; this plan enforces it as a test assertion by including the tool names in the ES reserved-words test. If a future edit ever translates the names in the ES branch, the test fails loudly.
- **ES register: TÚ verbatim from plan action block.** Plans 05/06/07 used USTED. Plans 08/09 used TÚ verbatim from their plan action blocks. Flagged (second time now) for future batched ES register normalization in a Plan 12 or dedicated polish plan — register flip can then be audited end-to-end across all ES branches at once rather than piecemeal.
- **Spanish 3pm example format matches EN.** Test accepts '3pm' / '3 pm' / '15:00' per plan spec (Spanish-market variation). Body uses '3pm' to mirror EN exactly — the model must still dispatch check_availability with English-parseable times regardless of conversational language; keeping the example format identical preserves the pedagogical force. Test flexibility is for future rewording safety, not an invitation to diverge now.
- **'peor modo de falla' over 'peor escenario' for ES 'worst failure mode'.** More literal translation preserves the technical-register mapping to EN 'failure mode' (both strings anchor the reader to the concept of a specific engineering failure state). Test accepts both for future rewording flexibility.
- **No downstream test updates required.** No existing test calls `_build_outcome_words_section()` directly. The signature change is contained to `prompt.py`'s single call site in `build_system_prompt`. Clean, minimal change.
