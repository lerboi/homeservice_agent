## Decisions Made

- **Branch P executed; Branch G skipped** — per [60.3-STREAM-A-ANALYSIS.md](./60.3-STREAM-A-ANALYSIS.md) selection, applying Plan 2's ambiguity-resolution rule (lines 285-294): when UAT evidence matches both #5096 pipeline-race signature (`text_done=false, audio_done=true`) AND Branch P directional signal (`end_call_invoked_at` 11ms before `last_text_token_at`, `playback_finished_at` stale on earlier segment), prefer Branch P as the lower-risk fully-revertable prompt-only change.

New failure mode (missing farewell content in decline cases) is a prompt *content* issue, not a prompt *mechanics* issue, and is scope-appropriate for the Stream B prompt audit. - **The 11-12ms `end_call_invoked_at` → `last_text_token_at` delta is an instrumentation/timing artifact** — identical magnitude in UAT #1 and UAT #2 (both ~11-12ms) indicates tool invocation and final text token fire within the same turn; 11-12ms is within event-loop dispatch measurement noise for the same call phase. This is NOT a prompt-defiance signal; originally read as Branch P directional evidence in Plan 2 but UAT #2 reinterpretation shows it is a measurement-floor artifact.
