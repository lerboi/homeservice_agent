## Decisions Made

- **EN body: verbatim preservation over D5/D2 audit-flagged changes.** The 60.3-PROMPT-AUDIT.md flagged:
  - D5 ✗: "After the caller answers, briefly acknowledge what you heard before moving forward" overlaps with Fix G's `silence_duration_ms=1500` VAD turn-boundary semantics.
  - D2 ~: "natural back-and-forth matters more than efficiency" is advisory prose with paraphrase-to-caller risk.
  The acknowledgment semantics remain load-bearing for realtime coaching — Fix G tells the model *when* the caller finished; the prompt tells the model *what to do* on that boundary (acknowledge, then advance, not barge-in-interrupt). Removing the acknowledgment copy risks nudging Gemini toward different boundary behavior. Plan 07 explicitly treats this as a minimal-change plan per the audit's own Plan Assignment Summary row ("D2, D4, D5 reviewed — minimal en change"). Deferred: if a future UAT shows paraphrase-to-caller or acknowledgment-is-redundant behavior, revisit.
- **USTED register for ES body.** Consistent with Plan 05 call_duration, Plan 06 tool_narration, and the existing info_gathering Spanish block. Same rationale as prior plans: in-prompt register consistency so the model doesn't flip between formal/informal 2nd-person across sections.
- **Signature without default locale.** `(locale: str)` matches Plan 06's explicit-threading choice. `build_system_prompt` already threads `locale` as a positional; no silent-default risk.
- **No downstream test updates required.** Unlike Plan 06, which had to update two 60.2 guard-rail tests that called `_build_tool_narration_section()` directly, no existing test calls `_build_voice_behavior_section()` directly — the signature change is contained to prompt.py's call site. Clean, minimal change.
- **Preserved Spanish accent diacritics.** Prompt prose is presentation-layer (model pronunciation guidance); ASCII-normalization would lose value. This matches the pattern in Plan 05 ES (terminado, día, despedida) and Plan 06 ES (herramienta, gracias, conexión). It intentionally differs from `messages/es.json` which is ASCII-safe for logging use.
