### Strategic choice (per RESEARCH Open Question 3)

**Recommended: inline `if locale == "es":` branches** for all Stream B patches.

Rationale:
- Mirrors the existing `_build_info_gathering_section` pattern (gold-standard for locale branching in this codebase).
- Single source of truth — prompt structure and its Spanish counterpart live adjacent in code, easy to keep in sync on future edits.
- No JSON refactor cost — extracting all prose to messages JSON would require restructuring 15 section builders and a templating layer, out of scope per CONTEXT.
- Existing `t()` accessor remains the vehicle for small reused strings (greeting disclosure, language labels); it should NOT be expanded to carry multi-paragraph prose — that would entangle JSON and Python responsibilities.

**Exception:** for dynamic labels that appear in multiple sections (e.g., "emergency"/"emergencia" or day-short names "Lun"/"Mon"), consider a small module-level dict in `prompt.py` rather than inlining per-section. Planner judgment in Plan 12.
