## Spacing Scale

Declared values (multiples of 4 only). Inherits from Phase 02.1 contract.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, time slot tick marks |
| sm | 8px | Intra-element spacing (label + input, icon + text), day column gaps |
| md | 16px | Card internal padding, form field vertical gap, calendar cell padding |
| lg | 24px | Section separation within a page card, flyout internal padding |
| xl | 32px | Between major sections (e.g., hours editor and zone management) |
| 2xl | 48px | Page-level vertical padding (top of dashboard content area) |
| 3xl | 64px | Not used in dashboard surfaces — reserved for landing page |

Exceptions:
- Touch targets for time-slot toggle buttons: minimum 44px height (WCAG 2.5.5)
- Calendar appointment blocks: minimum 48px height at 30-minute granularity so appointment text is readable
- Zone postal code tag chips: 28px height (compact, not a primary action target)
- Travel buffer ghost blocks in calendar: 8px minimum height to remain visible without occupying a full slot

## Typography

Inherits from Phase 02.1 contract. Dashboard surfaces use the same scale — no new sizes introduced.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 16px | 400 (regular) | 1.5 | Appointment detail text, zone description, settings copy |
| Label / Caption | 14px | 400 (regular) | 1.5 | Form labels, table headers, time labels in calendar, badge text |
| Heading | 20px | 600 (semibold) | 1.2 | Section headings (e.g., "Working Hours", "Service Zones"), flyout titles |
| Page title | 20px | 600 (semibold) | 1.2 | Dashboard page h1 (matches existing services page pattern) |

Notes:
- Calendar day column headers: 14px / weight 600 — distinguishable from time labels
- Appointment block primary line (caller name): 14px / weight 600
- Appointment block secondary line (job type, address): 13px / weight 400 — one step below caption for dense blocks; acceptable exception to 4-size rule because this is pure data density context
- Time axis labels (8 AM, 9 AM…): 12px / weight 400 / color: `text-slate-400` — de-emphasized, read-only spatial orientation labels (exception to 4-size rule: non-interactive axis labels)
- Conflict alert headline: 14px / weight 600 / color: `text-amber-700`

Source: Established in 02.1-UI-SPEC.md. Dashboard pattern confirmed in `services/page.js` (uses `text-xl font-semibold text-slate-900` for page heading = 20px/600).

## Surface 1: Working Hours Editor

Replaces `WorkingHoursStub` in `src/app/dashboard/services/page.js` (lines 316–328). Renders below the services table after a `Separator`.

### Layout

```
[Separator — my-6]
<section aria-labelledby="working-hours-heading">
  <div class="flex items-center justify-between mb-4">
    <h2 id="working-hours-heading">Working Hours</h2>
    [Quick-set dropdown + Save button]
  </div>
  [7-row day grid]
  [Slot duration row]
</section>
```

The section lives inside the existing white `bg-white rounded-xl shadow-sm border border-slate-200` dashboard card — no additional card wrapper.

### Day Grid

7 rows (Mon through Sun). Each row: `grid grid-cols-[80px_44px_1fr_auto] gap-3 items-center py-2`.

Columns:
1. Day label: 14px / weight 400 / `text-slate-700` (e.g., "Monday")
2. Switch toggle (shadcn `Switch`): 44px touch target wrapper. On = blue (`bg-blue-600`), Off = slate
3. Time range: Two `Input` fields (`type="time"`) — open time + close time, separated by an em dash — conditionally visible only when day is toggled on. Lunch break toggle: a secondary `+ Add lunch break` text button that expands to two more time inputs (lunch start / lunch end). Inputs: `h-8 text-sm`
4. Apply button: ghost icon button (`Copy` Lucide icon, 16px, `text-slate-400 hover:text-slate-700`). Tooltip: "Apply to other days". On click, opens a `Popover` below the button with checkboxes for each day of the week (excluding the current day). Owner checks the days they want to copy this schedule to, then clicks "Apply". Popover closes and the selected day rows update with the copied schedule. Does not auto-save — owner must click Save Hours

When day is off: time inputs are hidden (not just disabled) — the row shows only the day label + toggle to keep the grid scannable.

### Quick-Set Patterns

Dropdown (`Select` component) in the section header, `w-48`:
- "Mon–Fri 8 AM–5 PM"
- "Mon–Fri 7 AM–6 PM"
- "Mon–Sat 8 AM–5 PM"
- "Mon–Sat 7 AM–6 PM"
- "Custom" (default — selected when any day deviates from a preset)

Selecting a preset immediately populates all relevant day rows. Does not auto-save — owner must click Save.

### Slot Duration

Below the day grid, `mt-4`:
```
Default appointment duration:
[Select: 30 min | 45 min | 60 min | 90 min | 2 hours]
```
Label: 14px / `text-slate-700`. Select: `w-36`.

Caption below: "Calls booked with no specific duration use this default." — 13px / `text-slate-400`.

### Save Interaction

"Save Hours" button: `variant="default"` (filled, `bg-primary`), size `sm`, positioned top-right of section header alongside the Quick-set dropdown. On click: optimistic UI — button shows `Loader2` spin icon + "Saving…" text for 600ms, then reverts to "Save Hours" on success. On error: Sonner toast "Working hours couldn't be saved. Check your connection and try again." No changes are discarded — the form remains editable.

### States

- Loading: Render 7 `Skeleton` rows (`h-10 w-full rounded-md`) while fetching existing hours
- Saved (no changes): "Save Hours" button is `disabled` with `opacity-50` — visually signals no pending changes
- Has unsaved changes: "Save Hours" button is active. A subtle `text-amber-600 text-sm` label "Unsaved changes" appears inline left of the Save Hours button
- Error fetching: Show `Alert` component (shadcn) with `variant="destructive"` — "Working hours couldn't be loaded. Refresh to try again."
- All days toggled off: Allowed (owner may be on vacation). Save proceeds normally. A subtle inline warning below the grid: "All days are set to closed. Your AI will not offer booking slots." — 13px / `text-amber-600`

### Responsive

- Mobile (<768px): Day label + toggle stack above the time inputs. `grid-cols-[1fr_44px]` with time row wrapping to next line below
- Desktop (≥768px): Single-row layout as described

## Surface 2: Calendar / Appointments View

New dashboard page: `src/app/dashboard/calendar/page.js`.

### Page Structure

```
DashboardLayout (inherits: max-w-4xl mx-auto px-4 py-8, bg-slate-50)
  breadcrumb: Dashboard › Calendar
  [View mode toggle: Week | Day] (top-right of page header)
  [bg-white rounded-xl shadow-sm border border-slate-200]
    [Calendar header row: prev/next nav + date label + "Today" button]
    [Calendar body: time axis + day columns]
  [Today's Agenda sidebar — desktop only, ≥1024px]
```

On desktop (≥1024px): Two-column layout. Calendar occupies left ~65% (`max-w-none`), Today's Agenda sidebar occupies right ~35%. Both inside a `flex gap-6` wrapper. The outer `max-w-4xl` constraint expands to `max-w-6xl` on this page only to accommodate the two-column layout.

On mobile/tablet (<1024px): Single column. Today's Agenda sidebar moves below the calendar as a list.

### Calendar Header Row

```
[ChevronLeft button] [date label] [ChevronRight button]    [Today button] [Week|Day toggle]
```

Date label: Week view → "March 17–23, 2026" / Day view → "Monday, March 20, 2026" — 16px / weight 600 / `text-slate-900`.

"Today" button: `variant="outline"` size `sm`. Navigates to current week/day. Disabled when already viewing today.

Week/Day toggle: Two-button toggle group. Active state: `bg-slate-900 text-white`. Inactive: `bg-white text-slate-600 border border-slate-200`. No shadcn ToggleGroup needed — implement as two controlled buttons.

### Calendar Body — Week View

7 day columns + left time axis column: `grid grid-cols-[48px_repeat(7,1fr)]`.

Time axis: 7 AM through 8 PM in 1-hour increments. Each row: 48px height = 1 hour. 30-minute rows: 24px height (minor gridline, dashed, `border-slate-100`). Hour rows: 48px height (solid gridline, `border-slate-200`).

Day column headers: `text-center py-2 border-b border-slate-200`. Format: "Mon\n17" — day abbrev on top (12px / weight 600 / `text-slate-600`), date number below (18px / weight 700 / `text-slate-900`). Today's column: date number uses `bg-blue-600 text-white rounded-full w-8 h-8 flex items-center justify-center`.

Appointment blocks: Absolutely positioned within their day column, top/height calculated from start time and duration.

```
[urgency-color bg border-l-2 border-urgency rounded-md px-2 py-1 overflow-hidden]
  [caller name — 14px/600]
  [job type — 13px/400 text-slate-600]
  [address — 13px/400 text-slate-500 truncate]
  [urgency badge — top-right corner, 11px]
```

Travel buffer blocks: Same grid position logic, `bg-slate-100 border border-dashed border-slate-300 rounded-md`, centered label "Travel buffer" in 11px / `text-slate-400`. Not clickable.

External Google Calendar blocks (timed events): `bg-purple-50 border border-dashed border-purple-300 rounded-md px-2 py-1`. Label: event title, 13px / `text-purple-600`. "Google Calendar" caption in 11px. Not clickable — display only.

External Google Calendar all-day events: Rendered as a horizontal strip at the top of the day column, above the first time row. `bg-purple-50 border border-dashed border-purple-300 rounded-md px-2 py-0.5 text-xs text-purple-600 truncate`. Multiple all-day events stack vertically. Maximum 2 visible rows — "+N more" text if overflow. All-day events do not block time slots — they are informational only.

Current time indicator: Horizontal red line (`border-red-500 border-t-2`) with a `w-2 h-2 bg-red-500 rounded-full` dot on the left axis. Rendered only on today's column. Updates every 60 seconds via `setInterval`.

### Calendar Body — Day View

Single day. Same time axis. One wide column for appointments. Appointment blocks show full detail (caller name, job type, full address, urgency badge, duration label). Travel buffers show zone labels if zones configured ("North → South buffer").

### Appointment Click — Detail Flyout

Clicking any appointment block opens a `Sheet` component (shadcn, side="right", width 400px on desktop / full-width on mobile).

Flyout contents:
```
[Sheet header]
  [urgency badge]  [caller name — 20px/600]
  [job type — 16px/400 text-slate-600]

[Sheet body — space-y-4]
  [Section: Appointment Details]
    date + time (formatted: "Monday March 20, 2026 · 10:00 AM – 11:00 AM")
    address (with MapPin icon, 14px)
    duration label ("60 minutes")
    booked via ("AI call" or "Manual")

  [Section: Caller]
    phone number (with Phone icon)
    call timestamp ("Called 2 hours ago")

  [Section: Links]
    [Button variant="outline" size="sm"] "Listen to Recording" (ExternalLink icon)
    [Button variant="outline" size="sm"] "View Transcript" (FileText icon)
    [Button variant="outline" size="sm"] "Open Lead" (if routine — links to lead record)

  [Section: Notes — if any AI-extracted notes]
    italic text, 14px, text-slate-600

  [Section: Conflict — only shown when this appointment has a Google Calendar conflict]
    [div class="rounded-md bg-amber-50 border border-amber-200 p-3"]
      [AlertTriangle icon 16px text-amber-600 inline]
      [span "Conflicts with:" — 14px/600/text-amber-800]
      [p external event title — 14px/400/text-amber-700]
      [p external event time — 13px/400/text-amber-600]
      [p "Your booking is held. Resolve this conflict in Google Calendar or dismiss." — 13px/400/text-amber-600]
      [Button variant="outline" size="sm"] "Dismiss Conflict"

[Sheet footer]
  [Button variant="outline" size="sm" text-red-600] "Cancel Appointment"
```

"Cancel Appointment" opens an `AlertDialog` (shadcn) with:
- Title: "Cancel this appointment?"
- Body: "This will remove the booking and free the slot. The caller will not be notified automatically."
- Confirm: "Yes, Cancel Appointment" (`variant="destructive"`)
- Cancel: "Keep Appointment"

### Today's Agenda Sidebar

Sticky sidebar on desktop. Heading: "Today's Agenda" — 14px / weight 600 / `text-slate-600` (uppercase, tracking-wide style — `text-xs font-semibold uppercase tracking-wider text-slate-500`).

Appointment list: Cards in chronological order. Each card:
```
[border-l-2 border-urgency-color bg-white rounded-lg px-3 py-2 shadow-sm]
  [time label — 12px/600 text-slate-500]  [urgency dot]
  [caller name — 14px/600 text-slate-900]
  [job type — 13px/400 text-slate-500]
```

Empty state (no appointments today): Centered icon (`CalendarOff`, 24px, `text-slate-300`) + "No appointments today" in 14px / `text-slate-400`.

### Conflict Alert Banner

When unresolved conflicts exist, render a dismissible alert at the top of the calendar page, above the calendar card:

```
[bg-amber-50 border border-amber-200 rounded-lg px-4 py-3 flex items-start gap-3]
  [AlertTriangle icon — text-amber-600]
  [div]
    [p "1 calendar conflict needs your review" — 14px/600/text-amber-800]
    [p "A Google Calendar event overlaps an existing booking. Your booking is held. Review to resolve." — 14px/400/text-amber-700]
  [Button variant="outline" size="sm" class="ml-auto shrink-0"] "Review Conflicts"
  [X dismiss button — text-amber-500]
```

Clicking "Review Conflicts" scrolls to the conflicted appointment block and opens its flyout, with an additional conflict section displayed.

### Empty State (no appointments this week)

Centered in the calendar body:
- Icon: `Calendar` Lucide, 40px, `text-slate-300`
- Heading: "No appointments this week" — 16px / weight 600 / `text-slate-500`
- Body: "Appointments booked by your AI will appear here." — 14px / `text-slate-400`

### Loading State

Skeleton appointment blocks: 3 random-width `Skeleton` components positioned across the week grid, heights between 48–96px. Day headers and time axis render immediately.

## Surface 3: Google Calendar OAuth Connection

Lives in a new settings section. Location: extend `src/app/dashboard/services/page.js` OR create `src/app/dashboard/settings/page.js` — the planner decides the page location. This spec defines the component regardless of page location.

### Component: `CalendarSyncCard`

Rendered inside the standard dashboard card (`bg-white rounded-xl shadow-sm border border-slate-200`). Occupies a distinct `<section>` with a `Separator` above it, same pattern as WorkingHoursStub.

```
<section aria-labelledby="calendar-sync-heading">
  <div class="flex items-center justify-between mb-4">
    <div>
      <h2 id="calendar-sync-heading">Google Calendar Sync</h2>
      <p class="text-sm text-slate-500">Sync appointments with your Google Calendar in real time.</p>
    </div>
    [connection status badge — right]
  </div>
  [connection panel — conditional]
</section>
```

### State: Disconnected

```
[div class="rounded-lg border border-dashed border-slate-300 bg-slate-50 p-6 text-center"]
  [CalendarDays icon — 32px — text-slate-400 — mb-3]
  [h3 "Connect Google Calendar" — 16px/600/text-slate-700]

```

Clicking "Connect Google Calendar" opens the Google OAuth consent flow in a popup window (600×700px). On successful OAuth callback, the popup closes and the component transitions to Connected state via optimistic update + API re-fetch.

### State: Connected

```
[div class="flex items-start justify-between gap-4"]
  [div class="flex items-center gap-3"]
    [CalendarDays icon — 24px — text-emerald-600]
    [div]
      [p "Connected" — 14px/600/text-emerald-700]

  [Button "Disconnect" variant="outline" size="sm" class="text-red-600 border-red-200 hover:bg-red-50"]

[div class="mt-3 flex items-center gap-2"]

  [sync status text — 14px/400]
```

Sync status dot states:
- Syncing: animated pulse, `bg-blue-500` — "Syncing now…"
- Up to date: solid `bg-emerald-500` — "Up to date"
- Error: solid `bg-red-500` — "Sync error — check your Google connection"
- Re-auth required (token expired): solid `bg-amber-500` — "Reconnect required" — with a `text-amber-700 underline` "Reconnect" link that re-triggers the OAuth popup

### Disconnect Interaction

"Disconnect" button opens `AlertDialog`:
- Title: "Disconnect Google Calendar?"
- Body: "Your existing appointments will remain. New bookings will no longer sync to Google Calendar."
- Confirm: "Disconnect" (`variant="destructive"`)
- Cancel: "Keep Connected"

On confirm: optimistic transition to Disconnected state + Sonner toast "Google Calendar disconnected."

### State: OAuth Error

If OAuth flow fails or is cancelled:
```
[Alert variant="destructive"]
  "Google Calendar connection failed. Try again or check your Google account permissions."
  [Button "Try Again" variant="outline" size="sm" — inline right]
```

## Surface 5: Booking Status Displays

These are status indicators embedded in existing and new dashboard views — not a standalone page.

### 5a: Booking Status in Call History

Call history rows (existing call list — Phase 2 surface) gain a booking status column/badge.

Badge variants (reuse `Badge` component, `variant="outline"`):

| Booking Status | Badge Class | Label |
|----------------|------------|-------|
| Confirmed | `border-emerald-200 bg-emerald-50 text-emerald-700` | "Booked" |
| Lead with slots | `border-blue-200 bg-blue-50 text-blue-700` | "Lead — slots suggested" |
| No booking | `border-slate-200 bg-slate-50 text-slate-500` | "No booking" |
| Booking failed | `border-red-200 bg-red-50 text-red-600` | "Booking failed" |

Badge placement: Right-most column of call history table row, before the action buttons column.

### 5b: Real-Time Slot Availability Indicator

Not a user-facing dashboard widget in Phase 3. Availability is served from the local DB to the AI — no live widget required. This indicator is internal-only.

Exception: In the Calendar view, the absence of appointment blocks in a time slot is itself the availability indicator. No additional availability widget is needed.

### 5c: Conflict Alert Banner

Defined in Surface 2 (Calendar View). Reused here as canonical reference:
- Component: `ConflictAlertBanner`
- Condition: Rendered when `conflicts.length > 0` in calendar page data
- Placement: Above calendar card, full width
- Colors: amber (warning) palette as specified in Surface 2

### 5d: Booking Confirmation Toast

When the system confirms a new booking (either via AI call or manual creation):

Sonner toast (success variant):
```
toast.success("Booking confirmed", {
  description: "Dave R. — Burst pipe — Monday 10 AM",
  duration: 5000,
  action: { label: "View", onClick: () => router.push('/dashboard/calendar') }
})
```

When a booking fails (slot taken by race condition):

Sonner toast (error variant):
```
toast.error("Slot no longer available", {
  description: "That time was just booked. The next available slot is [time].",
  duration: 6000,
})
```

### 5e: Google Calendar Sync Status (inline)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Working hours day toggle (off → on) | Time inputs animate in with `Framer Motion` layout animation, 150ms ease-out. Toggle transitions color over 200ms. |
| Working hours save | "Save Hours" button shows Loader2 spinner for 600ms, success triggers Sonner toast, error reverts and triggers error toast |
| Quick-set pattern select | All 7 day rows update simultaneously — no animation, instant |
| Calendar prev/next navigation | Calendar body fades out (opacity 0, 100ms) and fades in (opacity 1, 150ms) with new week data |
| Calendar appointment block click | Sheet slides in from right over 200ms. Background dims with 30% black overlay. |
| Appointment block hover | Block elevates with `shadow-md`, cursor `pointer`. Tooltip shows full address if truncated. |
| Cancel appointment confirm | Sheet closes, block disappears from calendar with fade-out (150ms), success toast. |
| Travel buffer blocks | Not interactive — `pointer-events-none` |
| External calendar blocks | Not interactive — `pointer-events-none` — display only |
| Conflict banner dismiss (X) | Banner slides up and fades out (Framer Motion), 200ms. Conflict is not resolved — only hidden for session. |
| Google OAuth popup | Opens via `window.open()` centered on screen. Parent page polls `localStorage` or listens for `postMessage` for OAuth result. |
| Zone name inline edit | Click to edit — input renders with cursor at end. Escape cancels. Enter or blur saves. 200ms border transition on focus. |
| Postal code tag add | Typing + Enter adds a chip with scale-in animation (Framer Motion, 120ms). Invalid format: chip shakes (translateX animation) and does not save. |
| Postal code tag remove (X) | Chip scales out and fades (120ms). |
| Zone delete | AlertDialog confirmation required. On confirm: card slides up and fades out (200ms). |
| Responsive calendar (mobile) | Day view only on mobile. Week view hidden on <768px. View toggle shows only "Day". |
| Today column highlight | Blue date number circle is always visible — no animation needed |
| Current time indicator | Renders on page load, updates position every 60 seconds |
| Flyout "Listen to Recording" | Opens recording URL in new tab |
| Flyout "Open Lead" | Phase 3: button hidden (lead detail page not yet built). Phase 4+: navigates to lead detail page. Conditionally rendered based on route existence. |
| Switch (working day toggle) | shadcn Switch default interaction — space key toggles, click toggles, focus ring visible |
