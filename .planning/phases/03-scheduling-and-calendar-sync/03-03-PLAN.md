Output: Google Calendar OAuth API routes, push notification webhook, sync engine, and calendar event CRUD.
</objective>

  <action>
    Install googleapis and google-auth-library:
    ```bash
    npm install googleapis google-auth-library
    ```

    Create `src/lib/scheduling/google-calendar.js`:

    1. `createOAuth2Client()` — Uses `google.auth.OAuth2` with env vars `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, callback URL `${process.env.NEXT_PUBLIC_APP_URL}/api/google-calendar/callback`

    2. `getAuthUrl(oauth2Client)` — Returns `oauth2Client.generateAuthUrl({ access_type: 'offline', prompt: 'consent', scope: ['https://www.googleapis.com/auth/calendar.events'] })`

    3. `createCalendarEvent({ credentials, appointment })`:
       - Creates OAuth2Client, sets credentials `{ access_token, refresh_token, expiry_date }`
       - Calls `calendar.events.insert({ calendarId: 'primary', requestBody: { summary: [urgency prefix] + jobType + callerName, location: serviceAddress, start: { dateTime, timeZone }, end: { dateTime, timeZone }, extendedProperties: { private: { platform_appointment_id, tenant_id } } } })`
       - Returns the created event ID

    4. `registerWatch(tenantId, credentials)`:
       - Creates OAuth2Client with credentials
       - Calls `calendar.events.watch({ calendarId: 'primary', requestBody: { id: crypto.randomUUID(), type: 'web_hook', address: NEXT_PUBLIC_APP_URL + '/api/webhooks/google-calendar', token: tenantId, params: { ttl: '604800' } } })`
       - Updates calendar_credentials with watch_channel_id, watch_resource_id, watch_expiration from response
       - Returns watch response data

    5. `syncCalendarEvents(tenantId)`:
       - Loads calendar_credentials for tenantId where provider='google'
       - Creates OAuth2Client with stored credentials
       - Calls `calendar.events.list({ calendarId, syncToken: stored_last_sync_token })` — if 410 response, discard syncToken, re-call with `timeMin: now` for full sync
       - For each event in response: upsert into calendar_events using `(tenant_id, provider, external_id)` unique key
       - For cancelled events (status='cancelled'): delete from calendar_events
       - Store new syncToken in calendar_credentials.last_sync_token
       - Update last_synced_at timestamp
       - Check for conflicts: any synced event that overlaps a confirmed appointment gets flagged (don't auto-cancel)

    6. `pushBookingToCalendar(tenantId, appointmentId)`:
       - Load appointment by ID
       - Load credentials for tenant
       - If no credentials, silently return (calendar not connected)
       - Call createCalendarEvent
       - Update appointment.google_event_id with returned event ID

    7. `revokeAndDisconnect(tenantId)`:
       - Load credentials
       - Create OAuth2Client, call `oauth2Client.revokeToken(refresh_token)`
       - If watch_channel_id exists, call `calendar.channels.stop({ requestBody: { id: watch_channel_id, resourceId: watch_resource_id } })`
       - Delete calendar_credentials row
       - Delete all calendar_events for this tenant

    Create `src/lib/webhooks/google-calendar-push.js`:
    - Export `handleGoogleCalendarPush(request)` that reads headers: X-Goog-Resource-State, X-Goog-Channel-Token (tenantId), X-Goog-Channel-ID
    - If state === 'sync': return immediately (handshake)
    - If state === 'exists': call syncCalendarEvents(tenantId) — this is called from within after() in the route handler
    - Return { ok: true }

    - Test: createOAuth2Client uses correct env vars
    - Test: getAuthUrl includes calendar.events scope and prompt=consent
    - Test: pushBookingToCalendar silently returns when no credentials exist

    Create `src/app/api/google-calendar/auth/route.js`:
    - GET handler — authenticated route (use createSupabaseServer, check auth)
    - Import createOAuth2Client, getAuthUrl from google-calendar.js
    - Generate auth URL, include `state` parameter with tenant_id for CSRF protection
    - Return `Response.json({ url: authUrl })`

    Create `src/app/api/google-calendar/callback/route.js`:
    - GET handler — receives `code` and `state` query params from Google OAuth redirect
    - Exchange code for tokens via `oauth2Client.getToken(code)`
    - Extract `access_token`, `refresh_token`, `expiry_date` from response
    - Fetch calendar metadata via `calendar.calendarList.get({ calendarId: 'primary' })` to get display name
    - Upsert into `calendar_credentials` table: tenant_id (from state), provider='google', access_token, refresh_token, expiry_date, calendar_id='primary', calendar_name
    - Call `registerWatch(tenantId, credentials)` to start push notifications
    - Perform initial full sync via `syncCalendarEvents(tenantId)`
    - Redirect to `/dashboard/services?calendar=connected` (or wherever settings page lives)
    - On error, redirect to `/dashboard/services?calendar=error`

    Create `src/app/api/webhooks/google-calendar/route.js`:
    - POST handler — no auth (Google sends unsigned notifications)
    - Read X-Goog-Resource-State, X-Goog-Channel-Token, X-Goog-Channel-ID headers
    - Validate channel exists in calendar_credentials (reject stale channels)
    - If state === 'sync': return Response.json({ ok: true })
    - If state === 'exists': use `after()` to run `syncCalendarEvents(tenantId)` asynchronously
    - Always return Response.json({ ok: true }) immediately (Google requires fast 200)

    Create `src/app/api/cron/renew-calendar-channels/route.js`:
    - POST handler — protected by `Authorization: Bearer CRON_SECRET` header check
    - Query calendar_credentials where watch_expiration < (now + 24 hours in ms)
    - For each expiring credential: call `registerWatch(tenantId, credentials)` to create new channel
    - Log results, return summary Response.json
    - This route is called by Supabase pg_cron (set up separately via Supabase dashboard)
