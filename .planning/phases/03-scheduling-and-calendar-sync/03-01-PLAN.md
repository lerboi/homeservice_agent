  <behavior>
    - calculateAvailableSlots({workingHours, slotDurationMins, existingBookings, externalBlocks, zones, zonePairBuffers, targetDate, tenantTimezone, maxSlots}) returns array of {start, end} slot objects
    - Returns empty array when targetDate is a day-off (working hours disabled)
    - Returns correct slots within working hours window (e.g., 8AM-5PM for a 60-min slot: 8AM, 9AM, ... 4PM)
    - Excludes slots that overlap with existingBookings
    - Excludes slots that overlap with externalBlocks (calendar mirror events)
    - Applies 30-minute cross-zone travel buffer after last booking's end_time before next available slot
    - Applies 0-minute buffer for same-zone consecutive bookings
    - Applies flat 30-minute buffer when no zones configured (zones array empty)
    - Respects lunch break gaps in working hours
    - Returns at most maxSlots results
    - atomicBookSlot calls supabase.rpc('book_appointment_atomic', params) and returns the result
    - atomicBookSlot throws on supabase error
  </behavior>

    Create `src/lib/scheduling/slot-calculator.js`:
    - Export `calculateAvailableSlots(config)` as a pure function (no DB calls)
    - Working hours config shape: `{ monday: { open: '08:00', close: '17:00', enabled: true, lunchStart: '12:00', lunchEnd: '13:00' }, ... }`

    - Return first `maxSlots` available slots sorted by start time
    - Each returned slot: `{ start: ISO string, end: ISO string }`
