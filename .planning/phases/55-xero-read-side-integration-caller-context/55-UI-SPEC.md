### New UI elements introduced in Phase 55

**Reconnect-needed banner** — inline amber-tinted banner rendered inside the Xero card when `error_state` is set. Pattern: `<Alert>` with `variant="default"` and amber border override, `AlertTriangle` icon, short message, Reconnect primary `Button` (size sm, full width or inline), Disconnect kept as `Button` variant="ghost" size="sm" underneath.

**`last_context_fetch_at` timestamp** — muted `text-xs text-muted-foreground` line below the connected status line. Rendered only when non-null. Format: relative time via `date-fns` `formatDistanceToNow` (same as CalendarSyncCard.js). Example: "Last synced 3 min ago". Omit entirely when null.

**`connect_xero` checklist item** — no new visual component. Appends to the existing setup checklist array rendered by `/api/setup-checklist`. Follows established `{id, label, completed, target_url}` shape.

## Interaction States: Xero Card

### State 1 — Disconnected (no change from P54)
- Status line: `"Connect Xero to share customer history with your AI receptionist during calls."` (muted text)
- Button: `"Connect Xero"` — accent background, white text, full width, size sm
- Loading variant: `Loader2` spinner + `"Connecting…"` label, button disabled

### State 2 — Connected, no error (extended in P55)
- Status line: invoicing-off: `"Connected. Sharing customer context with your AI receptionist."` in emerald-600/emerald-400
- Status line: invoicing-on: `"Connected. Sharing customer context and sending invoices."` in emerald-600/emerald-400
- Timestamp: `"Last synced {relative time}"` in `text-xs text-muted-foreground`, below status line, only when `last_context_fetch_at` is non-null
- Button: `"Disconnect"` — `variant="outline"` with `border-red-600 text-red-600 hover:bg-red-50 dark:hover:bg-red-950/40`, full width, size sm

### State 3 — Reconnect needed / token error (new in P55)
- Inline banner inside card, above button area: amber border, `AlertTriangle` icon (amber-600), message: `"Reconnect needed — Xero token expired. Your AI receptionist can't access Xero customer info until you reconnect."`
- Primary action: `Button` size sm, full width, accent background: `"Reconnect Xero"` — initiates same OAuth flow as Connect
- Secondary action: `Button` variant="ghost" size="sm", destructive text color, below primary: `"Disconnect"` — opens existing disconnect AlertDialog
- Loading: Reconnect button shows `Loader2` + `"Reconnecting…"` while OAuth redirect pending

### State 4 — Loading skeleton (no change from P54)
- `Skeleton` for icon area, provider name, status lines, and button
- `aria-busy="true"` on grid container

## Accessibility

- All async buttons enter `disabled` state during pending operations (Connect, Reconnect, Disconnect)
- `Loader2` spinner has `aria-hidden="true"` on icon; visible label `"Connecting…"` / `"Reconnecting…"` / `"Disconnecting…"` provides accessible name
- `AlertDialog` uses Radix Dialog semantics; focus trap and escape-to-close are handled by Radix
- Reconnect-needed `Alert` uses `role="alert"` (via shadcn Alert component default)
- Timestamp element has no interactive role; rendered as `<p>` or `<span>` inside card
- Grid container uses `aria-busy="true"` during skeleton state
- Icon-only brand logos have `aria-label` and `<title>` (already present in `XeroIcon` and `JobberIcon`)
