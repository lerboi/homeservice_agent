## Scope Declaration

Phase 57 is a **delta-spec** relative to Phase 56. The Phase 56 UI-SPEC (`56-UI-SPEC.md`) is the authoritative baseline for the integrations page and all design tokens. This document specifies only the net-new visual and interaction surfaces introduced in Phase 57:

1. **External-event overlay model** — muted/non-editable render treatment for ALL external calendar events (Google, Outlook, Jobber) with provider pill. Retrofit Google/Outlook; introduce Jobber. (CONTEXT D-07, D-08)
2. **"Not in Jobber yet" pill** — permanent inline badge on Voco-booked appointments when Jobber is connected but push is unavailable. (CONTEXT D-09)
3. **Copy-to-clipboard action** — one-click appointment detail copy + Jobber new-visit deep link, shown on per-appointment flyout/detail when Jobber connected. (CONTEXT D-10)
4. **Dismissible banner** — calendar page banner educating the manual-copy workflow. (CONTEXT D-12)
5. **Bookable-users picker** — step injected into OAuth connect flow + edit affordance on the Jobber integrations card. (CONTEXT D-02, D-03)
6. **Booking-complete email conditional block** — new Jobber-copy section in existing email template. (CONTEXT D-11)

All spacing, typography, color foundation, component inventory, and registry rules from Phase 56 carry forward unchanged and are restated here for completeness.

### New UI elements introduced in Phase 57

#### 1. Muted External Event Block (retrofit + Jobber)

Replaces the existing `ExternalEventBlock` component in `CalendarView.js`. The existing violet treatment (`bg-violet-50 border-violet-400`) is replaced with unified slate muted treatment for all external providers. A provider pill replaces the plain provider-name text.

**Block visual contract:**
- Background: `bg-slate-50 dark:bg-slate-800/40`
- Left border accent: `border-l-[3px] border-l-slate-300 dark:border-l-slate-600`
- Border: `border border-slate-200 dark:border-slate-700`
- Opacity: `opacity-75` (further reduces visual weight vs Voco-booked blocks at full opacity)
- z-index: `z-[5]` (unchanged — stays below Voco appointments at `z-[10]`)
- Hover: `hover:opacity-90 hover:shadow-sm transition-all` (subtle affordance — block IS clickable to open in source)
- Cursor: `cursor-pointer` (click-through to source-of-truth)

**Provider pill (inside block, bottom-left, only when height >= 36px):**
- Jobber: `bg-[#1B9F4F]/10 text-[#1B9F4F] dark:bg-[#1B9F4F]/20 dark:text-emerald-300`
- Google: `bg-violet-50 text-violet-600 dark:bg-violet-950/30 dark:text-violet-300`
- Outlook: `bg-blue-50 text-blue-600 dark:bg-blue-950/30 dark:text-blue-300`
- Shape: `inline-flex items-center rounded-full px-2 py-1 text-[10px] font-medium`
- Label text: `"From Jobber"` / `"From Google"` / `"From Outlook"`

**Click behaviour:** `onClick` opens the external visit/event URL in a new tab (`window.open(url, '_blank', 'noopener')`). For Jobber, use per-visit URL if available; fall back to `https://app.getjobber.com/calendar` (Jobber calendar day-view). Google/Outlook: existing `onExternalEventClick` handler (already opens in source).

**No flyout:** Clicking a muted external block does NOT open the Voco AppointmentFlyout. The click is purely a navigation to the source-of-truth.

#### 2. "Not in Jobber yet" Pill

Inline badge rendered on `AppointmentBlock` in `CalendarView.js` AND in the `AppointmentFlyout` header row. Appears only when `jobber_connected === true` (passed as prop from parent) AND the appointment has not been pushed to Jobber (Phase 999.3 push not yet available, so this condition is always true during Phase 57).

**On calendar block (compact — only when block height >= 44px):**
- Position: top-right corner of the appointment block, above the off-hours indicator
- Shape: `inline-flex items-center rounded-full px-1 py-1 text-[10px] font-medium`
- Colors: `bg-amber-50 text-amber-700 border border-amber-200 dark:bg-amber-950/40 dark:text-amber-300 dark:border-amber-700`
- Label: `"Not in Jobber"` (shortened for block — "yet" omitted for space)

**On AppointmentFlyout (full label):**
- Position: inline in the flyout header row, adjacent to urgency badge
- Label: `"Not in Jobber yet"`
- Same color tokens as above

#### 3. Copy-to-Clipboard Action (AppointmentFlyout)

Renders inside `AppointmentFlyout` only when `jobber_connected === true`. Positioned below the existing appointment details section, above the status/action buttons.

**Copy block layout:**
```
[ clipboard section card ]
  heading:    "Copy to Jobber"   (text-sm font-medium)
  subtext:    "Paste into a new Jobber visit"   (text-xs text-muted-foreground)
  [ Copy details  ] button — accent background, Copy icon, full-width or 50% + adjacent "Open Jobber" link
  [ Open in Jobber ↗ ] button — variant="outline", ExternalLink icon, opens Jobber new-visit URL
```

**Clipboard paste block format (exact string produced by the action):**
```
Client: {caller_name}
Phone: {from_number}
Address: {service_address}
Start: {start_time formatted as "Mon Apr 21, 2026 at 10:00 AM"}
Duration: {duration_minutes} min
Notes: {notes or "—"}
```

**Copy feedback:**
- On success: `toast.success("Copied to clipboard")` — sonner default duration
- On failure (clipboard API unavailable): `toast.error("Couldn't copy — try manually selecting the text")` — show fallback text-select area
- Button state during write: `Loader2` spinner + disabled, 500ms debounce to prevent double-tap

**"Open in Jobber" link:**
- URL: `https://app.getjobber.com/visits/new` (Jobber new-visit screen deep link)
- Opens in new tab, `noopener noreferrer`
- Renders as `Button variant="outline" size="sm"` with `ExternalLink` icon

#### 4. Dismissible Calendar Page Banner

Renders at the top of the `/dashboard/calendar` page content area, above the CalendarView. Only shown when `jobber_connected === true`. Dismissed state stored in `localStorage` key `voco_jobber_copy_banner_dismissed` (follows the pattern of `voco_calendar_show_completed`). On first-visit for new team members the banner is always shown.

**Layout:**
```
[ Info icon ]  Jobber push is coming soon — Voco bookings stay in Voco until then.
               Click any booking to copy it into Jobber.
                                                        [ × dismiss ]
```

**Visual contract:**
- Container: `flex items-start gap-3 rounded-lg border border-blue-200 bg-blue-50 p-4 dark:border-blue-800 dark:bg-blue-950/40`
- `Info` icon: `h-4 w-4 text-blue-500 dark:text-blue-400 shrink-0 mt-0.5`
- Body text: `text-sm text-blue-800 dark:text-blue-200`
- Dismiss button: `Button variant="ghost" size="icon" className="h-6 w-6 shrink-0 ml-auto text-blue-400 hover:text-blue-600 dark:hover:text-blue-200"` with `X` icon `h-3.5 w-3.5`
- Animation: `AnimatePresence` with `initial={{ opacity: 0, height: 0 }} animate={{ opacity: 1, height: 'auto' }} exit={{ opacity: 0, height: 0 }}` — 200ms ease-out (matches existing accordion tokens)

#### 5. Bookable-Users Picker

Rendered in two contexts:
- **Connect flow** (post-OAuth callback, before checklist completes) — full-page or sheet step
- **Settings panel** (Jobber card on `/dashboard/more/integrations`) — inline collapsible section

**Picker list item contract (each Jobber user):**
```
[ Checkbox ]  Avatar placeholder (initials, 28px circle, bg-muted)  Name  [ "Active" badge if ≥1 visit in 30d ]
```

- Checkbox: shadcn `Checkbox`, `size="sm"`
- Avatar: `w-7 h-7 rounded-full bg-muted flex items-center justify-center text-xs font-medium text-muted-foreground` (initials from first+last name)
- User name: `text-sm font-medium text-foreground`
- "Active" badge (recent-visits indicator): `inline-flex items-center rounded-full px-2 py-1 text-[10px] font-medium bg-emerald-50 text-emerald-700 dark:bg-emerald-950/40 dark:text-emerald-300` — label: `"Active"`
- Loading state (while fetching Jobber users): 3 `Skeleton` rows, each `h-9 rounded-md`
- Empty state (zero users returned from Jobber API): `"No team members found in Jobber."` in `text-sm text-muted-foreground`, with a `Loader2`-based "Retry" button

**Auto-skip behaviour (solo account):** If exactly one Jobber user is returned, skip the picker entirely and auto-select that user. No UI rendered for the picker step — proceed directly to completion.

**Zero-recent-visits default:** If no users have visits in the last 30 days, pre-select ALL users (D-03). Display a muted footnote: `"No recent visits found — all members selected. Deselect office or admin accounts."` in `text-xs text-muted-foreground`.

**Save button:** `"Save team members"` — accent background, full width on mobile, auto-width (min 160px) on desktop. During save: `Loader2` + `"Saving…"` + disabled.

**Re-sync footnote (settings panel, after initial connect):** `text-xs text-muted-foreground` below the save button: `"Changing this set triggers a re-sync of your Jobber schedule."` — plain text, no icon.

## Accessibility

- External event blocks retain `type="button"` and `onClick` — keyboard navigable, accessible name = `{event.title} — {providerLabel} event`
- `"Not in Jobber yet"` pill: purely informational `<span>` with `role="status"` (announces on mount for SR users)
- Copy button: `aria-label="Copy appointment details to clipboard"`, `aria-live="polite"` region for copy success/failure announcement
- "Open in Jobber" link: `aria-label="Open this appointment in Jobber (opens in new tab)"` — alerts SR users of new-tab behaviour
- Calendar banner: `role="alert"` is NOT used (informational, not urgent); use `role="status"` instead. Dismiss button: `aria-label="Dismiss Jobber notification banner"`
- Bookable-users checkboxes: each has an associated visible label; group wrapped in `<fieldset>` with `<legend>` matching picker heading
- All async buttons enter `disabled` + spinner during pending operations (copy, save team members)
