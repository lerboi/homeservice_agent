## Spacing Scale

Declared values (multiples of 4). Standard 8-point scale applies across all public pages.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pulse dot, eyebrow pill padding |
| sm | 8px | Compact element spacing, stacked form field gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section content padding, tier card gaps |
| xl | 32px | Layout column gaps, hero content gaps |
| 2xl | 48px | Major section breaks, vertical hero padding |
| 3xl | 64px | Page-level section vertical padding |

Exceptions:
- Touch targets (hamburger toggle, accordion trigger, CTA buttons): minimum 44px height — enforced via `min-h-[44px]` (source: ui-ux-pro-max skill, CRITICAL touch rule)
- Nav height: 64px fixed (carried from LandingNav — `h-16`) (source: CONTEXT.md existing code)
- Hero section top padding: 112px / 144px (pt-28 / pt-36) to clear fixed nav (source: HeroSection.jsx)
- Pricing tier card "Most Popular" ring: 2px accent-colored ring offset, not part of spacing scale

## Typography

Four sizes, two weights. Covers all roles across hero, body, labels, and display headlines on public pages.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Paragraph text, form helper text, footer links, FAQ body |
| Label | 13–14px | 500 (medium) | 1.4 | Nav links, badge text, eyebrow labels, table cell metadata |
| Heading | 20px | 600 (semibold) | 1.3 | Section headings (h2), tier names, card headings, about values |
| Display | 36–60px (fluid) | 600 (semibold) | 1.1 | Page hero h1 — Pricing page, About page, Contact page hero |

Notes:
- Display size is fluid: `text-[2.25rem] md:text-[3rem] lg:text-[3.75rem]` — consistent with established HeroSection pattern (source: HeroSection.jsx)
- Body line-height 1.5 is the global default via `@layer base` (source: globals.css)
- Heading line-height 1.3 applied on section h2 elements (`leading-[1.3]`)
- Two declared weights only: 400 (regular) and 600 (semibold). The 500 (medium) for labels is a minor usage reserved for small text legibility — treated as the label sub-role, not a third weight in the system.
- Letter-spacing tight (`tracking-tight`) on Display and Heading roles — carried from established landing page pattern

## Interaction States

### Pricing Page

**Billing Toggle**
- Default state: "Monthly" selected
- Active pill: Heritage Copper (#C2410C) background, white label text
- Inactive pill: transparent background, muted text (#475569)
- Annual badge: "Save 20%" in Success green (#166534), small pill shape, appears inline on toggle
- Transition: 150ms ease for pill background, prices update instantly (no animation on numbers)

**Tier Cards**
- Default: white card, neutral border (`border-border`)
- Growth (Most Popular): 2px Heritage Copper ring (`ring-2 ring-[#C2410C]`), slightly elevated shadow
- Hover (all cards): `translateY(-2px)` lift, 200ms ease, shadow increase
- CTA button hover: 400ms Heritage Copper glow (`shadow-[0_4px_16px_0_rgba(194,65,12,0.4)]`) — matches existing landing button pattern

**Comparison Table**
- Tier header row: `position: sticky; top: 64px` (clears fixed nav)
- Check cells: Lucide `<Check>` icon in Success green; dash (—) for excluded features in muted text
- Row hover: subtle `bg-[#F5F5F4]/60` highlight on light background

**FAQ Accordion**
- Collapsed: chevron pointing down, full border-bottom visible
- Expanded: chevron rotates 180deg (150ms), answer body fades in (200ms ease)
- Only one item open at a time (single open mode)

### Contact Form

**Field states**
- Default: standard shadcn input border (`--border`)
- Focus: `ring-2 ring-[#C2410C]/30` — Heritage Copper focus ring (matches `--ring` token set to same hue)
- Error: red border (`--destructive`), error message appears below field in 13px destructive color
- Honeypot field: `aria-hidden="true"`, `tabIndex={-1}`, `display: none` via CSS — not visible, not tabbable

**Submit states**
- Idle: Heritage Copper button, white text
- Loading: disabled state + Lucide `<Loader2>` spinning icon replacing label text
- Success: Sonner toast — "Message sent. We'll reply within 24 hours." (green variant)
- Error: Sonner toast — "Something went wrong. Please try again or email us directly." (destructive variant)

### Navigation Mobile Drawer

- Trigger: hamburger icon (Lucide `<Menu>`) at 44px touch target
- Open: drawer slides in from right, full viewport height, dark overlay (#0F172A/80) on content
- Close: X icon (Lucide `<X>`) at same position, or overlay tap
- Animation: `x: '100%'` → `x: 0` via Framer Motion, 250ms ease-out
- CTA button pinned to drawer bottom: same Heritage Copper style as nav CTA

## Page-Level Sections

### Pricing Page (`/pricing`)

1. **PricingHero** — Server Component. Dark (#0F172A) section. h1 + subheading + "No credit card required" micro-copy below toggle. Radial copper glow overlay (matches HeroSection pattern).
2. **BillingToggle** — Client island inside PricingHero or PricingTiers. Centered pill toggle.
3. **PricingTiers** — Client Component (owns toggle state). 4-column grid desktop, single column mobile. Growth card `order-first` on mobile.
4. **ComparisonTable** — Server Component. Sticky thead. Feature rows × tier columns. Horizontal scroll container on mobile.
5. **FAQSection** — Client Component. 4 accordion items. Radix AccordionRoot with `type="single"` and `collapsible`.
6. **PricingCTABanner** — Server Component. Dark section (matches FinalCTASection pattern). "Ready to stop losing jobs to voicemail?" + "Get Started" CTA.

### About Page (`/about`)

1. **AboutHero** — Server Component. Dark section. h1 + 1–2 sentence mission statement.
2. **MissionSection** — Server Component. Light section. Mission paragraph + 3 core values as card grid (icon + value title + 1-sentence description). Lucide icons.
3. **AboutCTABanner** — Server Component. Dark section. CTA to onboarding.

### Contact Page (`/contact`)

1. **ContactHero** — Server Component. Dark section. h1 + "We respond within 24 hours" SLA displayed prominently below heading.
2. **ContactFormSection** — Server Component wrapper. Light section. Renders ContactForm client island.
3. **ContactForm** — Client Component. Fields: Name (required), Email (required), Inquiry Type (dropdown, required), Message (textarea, required). Honeypot field hidden. Submit → `POST /api/contact`.

## Animation Contract

All animations must respect `prefers-reduced-motion` via the existing AnimatedSection `useReducedMotion` pattern (source: STATE.md Phase 02.1-01 decision).

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Page sections on scroll | AnimatedSection fade + slide up (direction="up") | 500ms | ease-out |
| Pricing tier cards | Stagger via AnimatedStagger (delay 0.1s between cards) | 400ms | ease-out |
| Mobile drawer | Framer Motion `x: '100%'` → `x: 0` | 250ms | ease-out |
| Drawer overlay | Opacity 0 → 0.8 | 200ms | ease |
| FAQ accordion item | Height expand + opacity | 200ms | ease |
| CTA button hover lift | `translateY(-2px)` | 200ms | ease |
| Pricing toggle pill | Background color transition | 150ms | ease |
| Heritage Copper glow on CTA hover | Box shadow | 400ms | ease |

Reduced motion: AnimatedSection `initial={false}` when `useReducedMotion()` returns true — no translate, no fade (matches Phase 02.1-01 decision from STATE.md).

## Accessibility Checklist

Minimum requirements per ui-ux-pro-max skill (CRITICAL tier):

- [ ] Color contrast: white text on #0F172A — passes 4.5:1 (21:1 actual). Heritage Copper #C2410C on white — verify 3:1 for large text CTAs (WCAG AA Large).
- [ ] All interactive elements (toggle, accordion, mobile drawer, form fields) keyboard-navigable.
- [ ] FAQ accordion: `aria-expanded` toggled on trigger, `aria-controls` pointing to answer region.
- [ ] Mobile nav: `aria-label="Navigation"` on nav, `aria-expanded` on hamburger button.
- [ ] Form labels: explicit `<label htmlFor>` on every form field. No placeholder-only labels.
- [ ] Contact form submit: `aria-busy="true"` during loading state.
- [ ] Comparison table: `<th scope="col">` for tier headers, `<th scope="row">` for feature names.
- [ ] Honeypot field: `aria-hidden="true"` and `tabIndex={-1}`.
