```
  --animate-accordion-up: accordion-up 200ms ease-out;
```

2. After the closing `}` of the `@theme inline` block (after line 48), add the keyframe definitions:

```css
@keyframes accordion-down {
  from {
    height: 0;
    opacity: 0;
  }
  to {
    height: var(--radix-accordion-content-height);
    opacity: 1;
  }
}

@keyframes accordion-up {
  from {
    height: var(--radix-accordion-content-height);
    opacity: 1;
  }
  to {
    height: 0;
    opacity: 0;
  }
}
```

This registers `animate-accordion-down` and `animate-accordion-up` as Tailwind v4 utilities (via the `--animate-*` convention in `@theme inline`) and defines the keyframes that Radix accordion content uses for smooth height transitions. The `var(--radix-accordion-content-height)` CSS variable is automatically set by Radix Accordion on the content element.

Do NOT modify FAQSection.jsx — it already has the correct class names (`data-[state=open]:animate-accordion-down` and `data-[state=closed]:animate-accordion-up`).
