## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Button icon margins, compact row gaps |
| md | 16px | Default element spacing, card padding horizontal |
| lg | 24px | Section padding (matches `p-6` = 24px on pages) |
| xl | 32px | Card-to-card gaps within the page |
| 2xl | 48px | Separator spacing (`my-6` = 24px used; escalation section spacing 48px) |
| 3xl | 64px | Not used within dashboard pages — reserved for page-level layouts |

Exceptions:
- Touch targets for contact row action buttons (remove, drag handle): minimum 44x44px per WCAG 2.1 / ui-ux-pro-max accessibility rule.
- Timeout input field: minimum 80px width to accommodate "60 sec" display without truncation.
- Escalation chain contact card drag handle: 44px tap area even if visual handle is 16px.

Source: Existing services/page.js and WorkingHoursEditor.js use `p-6` (24px) page padding, `space-y-3` rows, `min-h-14` (56px) row height — all preserved.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `oklch(1 0 0)` (var(--background)) | Page background, table rows |
| Secondary (30%) | `#F5F5F4` (stone-100 / var(--color-warm-surface)) | Table header row, card section backgrounds, drag hover bg |
| Accent (10%) | `#C2410C` (var(--color-brand-orange)) | Primary save/confirm CTA buttons, empty state CTA, new contact "Add Contact" button |
| Destructive | `oklch(0.577 0.245 27.325)` (var(--destructive) ≈ red-600) | Contact remove hover (text-red-600 bg-red-50), chain delete confirmation |

Accent reserved for:
- "Save" / "Save Chain" primary action button in escalation chain section
- Empty-state primary CTA button ("Add Your First Contact")
- "Add Service" primary CTA on empty state (already established — do not change)

Do NOT apply accent to: tag badges, timeout inputs, notification preference toggles, drag handles, or secondary/outline buttons.

Urgency badge colors (pre-existing, do not change):
- Emergency: `bg-red-100 text-red-700`
- High ticket: `bg-amber-100 text-amber-700`
- Routine: `bg-[#0F172A]/[0.06] text-[#0F172A]/70`

Per-urgency escalation behavior indicator colors (new — Claude's Discretion D-07):
- Emergency row: left border `border-l-4 border-red-400`
- High ticket row: left border `border-l-4 border-amber-400`
- Routine row: left border `border-l-4 border-stone-300`

Source: Established in `globals.css`, `services/page.js` URGENCY_BADGE_CLASSES, and STATE.md Phase 02.1 token decisions.

## Interaction Contracts

### Service table improvements (D-03)

| Interaction | Behavior |
|-------------|----------|
| Drag-to-reorder | Drag handle icon (GripVertical from lucide-react) on left of each row. Cursor: grab. Drag active: row shadow + 2px offset. Drop: optimistic reorder + PATCH request. |
| Bulk tag edit | Checkbox column appears when user hovers any row. Check multiple rows, then "Set tag for selected" dropdown appears in table header. Single confirm action applies to all. |
| Tag badge click | Opens the Select dropdown directly (current behavior preserved). |
| Remove service | X button on right, hover: text-red-600 bg-red-50. Click: immediate row removal + undo toast (4s). Actual DELETE fires after 4.1s (existing pattern). |

### Escalation chain (D-04, D-05, D-06, D-11)

| Interaction | Behavior |
|-------------|----------|
| Add contact | Click "Add Contact" at bottom of chain list. Inline form appears as new card in edit mode. Tab order: name → role → phone → email → timeout → channel → "Save contact" button. |
| Edit contact | Click anywhere on ContactCard row (not delete button) to expand into edit mode inline. |
| Remove contact | Trash icon button on ContactCard. Click opens AlertDialog: "Remove [Name] from chain?" with "Remove" (destructive) and "Cancel" buttons. |
| Reorder contacts | Drag handle (GripVertical) on left of each ContactCard. Drag to new position. Optimistic reorder, PATCH on drop. |
| Timeout selector | Select component with options: 15 sec, 30 sec, 45 sec, 60 sec. Default: 30 sec for new contacts. |
| Channel picker | RadioGroup: "SMS", "Email", "Both". Default: Both. Shown below phone/email fields. Phone required if SMS or Both selected. Email required if Email or Both selected. Validation fires on blur. |
| Save chain | Single "Save Chain" button at bottom of escalation section. Disabled until any change has been made. On click: full chain PATCH. Success: toast.success("Escalation chain saved"). Failure: toast.error with retry guidance. |
| Per-urgency mapping | Switch per urgency row. Emergency: default ON, cannot be disabled (owner cannot turn off emergency escalation). High ticket: default ON. Routine: default OFF. |

### Empty states

| State | Location | Visual |
|-------|----------|--------|
| No services | Services table | Centered column: illustration placeholder (inline SVG wrench icon 48px text-stone-300), h2 "No services yet", p body, "Add Your First Service" accent CTA button |
| No escalation contacts | EscalationChainSection | Centered column: Phone icon 48px text-stone-300, h2 "No escalation contacts", p body explaining what chain does, "Add Your First Contact" accent CTA button |

## Accessibility Contract

- All interactive elements have `aria-label` or visible label. Icon-only buttons (drag handle, remove, tag dropdown trigger) carry `aria-label`.
- Escalation chain contact list uses `role="list"` with each ContactCard as `role="listitem"`.
- Drag-and-drop reorder also supports keyboard: Up/Down arrow keys to move focused ContactCard. Announce position change via `aria-live="polite"` region.
- Per-urgency Switch components use `aria-describedby` pointing to the row description text.
- Color is never the sole conveyor of urgency — badge text label always accompanies color.
- Focus ring: Radix/shadcn default focus ring preserved (`outline-ring/50` from globals.css).
- Minimum touch target: 44x44px for drag handle, remove button, and add-contact button.
- prefers-reduced-motion: Drag animations and row reorder transitions use `@media (prefers-reduced-motion: no-preference)` guard.
