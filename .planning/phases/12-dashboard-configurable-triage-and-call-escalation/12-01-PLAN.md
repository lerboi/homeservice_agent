Create `supabase/migrations/006_escalation_contacts.sql` with:

1. **escalation_contacts table:**
```sql
CREATE TABLE escalation_contacts (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id           uuid NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  name                text NOT NULL,
  role                text,
  phone               text,
  email               text,
  notification_pref   text NOT NULL DEFAULT 'both'
    CHECK (notification_pref IN ('sms', 'email', 'both')),
  timeout_seconds     int NOT NULL DEFAULT 30
    CHECK (timeout_seconds IN (15, 30, 45, 60)),
  sort_order          int NOT NULL DEFAULT 0,
  is_active           boolean NOT NULL DEFAULT true,
  created_at          timestamptz NOT NULL DEFAULT now(),
  updated_at          timestamptz NOT NULL DEFAULT now()
);
```

2. **Index:**
```sql
CREATE INDEX idx_escalation_contacts_tenant ON escalation_contacts(tenant_id, sort_order);
```

3. **RLS policies** (same pattern as services table in 002):
```sql
ALTER TABLE escalation_contacts ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Tenants can manage own escalation contacts"
  ON escalation_contacts FOR ALL
  USING (tenant_id = auth.uid())
  WITH CHECK (tenant_id = auth.uid());

CREATE POLICY "Service role full access to escalation_contacts"
  ON escalation_contacts FOR ALL
  TO service_role
  USING (true)
  WITH CHECK (true);
```

Note: Check existing RLS patterns in 002 — if the tenant policy uses a lookup (e.g., `tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid())`), match that pattern exactly. Do NOT use `tenant_id = auth.uid()` if that's not how existing policies work.

4. **Add sort_order to services table:**
```sql
ALTER TABLE services ADD COLUMN IF NOT EXISTS sort_order int NOT NULL DEFAULT 0;
```

5. **Backfill sort_order on existing services** (per RESEARCH.md — prevents all rows being 0):
```sql
UPDATE services SET sort_order = sub.rn
FROM (
  SELECT id, row_number() OVER (PARTITION BY tenant_id ORDER BY created_at) AS rn
  FROM services
) sub
WHERE services.id = sub.id;
```

6. **Index for services sort_order:**
```sql
CREATE INDEX IF NOT EXISTS idx_services_sort_order ON services(tenant_id, sort_order);
```

Create `src/app/api/escalation-contacts/route.js` with 5 HTTP methods. Follow the exact pattern from `src/app/api/services/route.js` (import supabase, import getTenantId, 401 guard).

**GET:**
- Query: `supabase.from('escalation_contacts').select('id, name, role, phone, email, notification_pref, timeout_seconds, sort_order, is_active, created_at, updated_at').eq('tenant_id', tenantId).eq('is_active', true).order('sort_order', { ascending: true })`
- Response: `{ contacts: data }`

**POST (add contact):**
- Body: `{ name, role, phone, email, notification_pref, timeout_seconds }`
- Validation: `name` required (non-empty string). At least one of `phone` or `email` required. If `notification_pref` is 'sms' or 'both', `phone` required. If 'email' or 'both', `email` required.
- **Max 5 active contacts per tenant** (per RESEARCH.md): Before insert, count active contacts: `supabase.from('escalation_contacts').select('id', { count: 'exact', head: true }).eq('tenant_id', tenantId).eq('is_active', true)`. If count >= 5, return 400 `{ error: 'Maximum 5 escalation contacts allowed' }`.
- Compute `sort_order` for new contact: query max sort_order for tenant, add 1. Default to 0 if no contacts exist.
- Insert with all fields + tenant_id + computed sort_order.
- Return `{ contact: data }` with status 201.

**PUT (update contact):**
- Body: `{ id, name, role, phone, email, notification_pref, timeout_seconds }`
- Same validation as POST (name required, phone/email per notification_pref).
- Update with `.eq('id', id).eq('tenant_id', tenantId)`.
- Also set `updated_at: new Date().toISOString()`.
- Return `{ contact: data }`.

**DELETE (soft-delete contact):**
- Body: `{ id }`
- Set `is_active: false, updated_at: new Date().toISOString()` where `id=X AND tenant_id=tenantId`.
- Return `{ deleted: true }`.

**PATCH (reorder contacts):**
- Body: `{ order: [{ id, sort_order }, ...] }`
- Upsert array: `order.map(({ id, sort_order }) => ({ id, tenant_id: tenantId, sort_order }))` with `{ onConflict: 'id' }`.
- CRITICAL: Include `tenant_id` in every upsert row (per RESEARCH.md Pitfall 3 — RLS requires it in WITH CHECK).
- Return `{ ok: true }`.
