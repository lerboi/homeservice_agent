**DnD reorder handler** (same pattern as services page):
```javascript
function handleDragEnd(event) {
  const { active, over } = event;
  if (active.id !== over?.id) {
    setContacts(items => {
      const oldIndex = items.findIndex(i => i.id === active.id);
      const newIndex = items.findIndex(i => i.id === over.id);
      return arrayMove(items, oldIndex, newIndex);
    });
    setHasChanges(true);
  }
}
```

**Add contact handler:** When "Add Contact" clicked, if contacts.length >= 5, show toast.error("Maximum 5 escalation contacts allowed") and return. **Update contact handler:** PUT to `/api/escalation-contacts` with full contact data.

**For DnD on contact cards:** Use useSortable hook inside each ContactCard. The EscalationChainSection wraps contact list in `<DndContext><SortableContext items={contacts.map(c => c.id)} strategy={verticalListSortingStrategy}>`.

But ContactCard needs to use useSortable internally. Restructure: create a `SortableContactCard` wrapper inside EscalationChainSection that calls useSortable and passes drag props to ContactCard. OR make ContactCard itself call useSortable. The cleanest approach: ContactCard receives all drag props from parent. Create an internal `SortableContactWrapper` in EscalationChainSection:

```jsx
function SortableContactWrapper({ contact, ...rest }) {
  const { attributes, listeners, setNodeRef, transform, transition, isDragging } = useSortable({ id: contact.id });
  const style = { transform: CSS.Transform.toString(transform), transition, opacity: isDragging ? 0.5 : 1 };
  return (
    <ContactCard
      contact={contact}
      dragHandleProps={{ ...listeners, ...attributes }}
      dragRef={setNodeRef}
      dragStyle={style}
      isDragging={isDragging}
      {...rest}
    />
  );
}
```
