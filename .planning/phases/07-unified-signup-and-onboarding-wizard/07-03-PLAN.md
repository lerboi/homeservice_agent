**New test-call-status endpoint (src/app/api/onboarding/test-call-status/route.js):**

Create directory `src/app/api/onboarding/test-call-status/` and file `route.js`.

This is a lightweight GET endpoint that Step 5 polls every 4 seconds to check if the test call has completed.

```javascript
import { createSupabaseServer } from '@/lib/supabase-server';
import { supabase } from '@/lib/supabase';

export async function GET() {
  const serverSupabase = await createSupabaseServer();
  const { data: { user } } = await serverSupabase.auth.getUser();
  if (!user) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 });
  }

  const { data: tenant } = await supabase
    .from('tenants')
    .select('onboarding_complete, retell_phone_number')
    .eq('owner_id', user.id)
    .single();

  if (!tenant) {
    return Response.json({ error: 'Tenant not found' }, { status: 404 });
  }

  return Response.json({
    complete: tenant.onboarding_complete ?? false,
    retell_phone_number: tenant.retell_phone_number ?? null,
  });
}
```

Key: No caching headers — the response must be fresh on every poll.

**Fix test-call route (src/app/api/onboarding/test-call/route.js):**

Remove the line that sets `onboarding_complete: true` from the update call. Keep `test_call_completed: true` to track that a test call was triggered.

Change this line:
```javascript
await supabase
  .from('tenants')
  .update({ test_call_completed: true, onboarding_complete: true })
  .eq('id', tenantId);
```

To:
```javascript
await supabase
  .from('tenants')
  .update({ test_call_completed: true })
  .eq('id', tenantId);
```

This ensures `onboarding_complete` is only set by the Retell webhook handler when the call actually finishes — the user must hear their AI before the wizard marks completion. The Retell webhook handler (per STATE.md) already sets `onboarding_complete=true` on call completion — this is the correct trigger.

Also remove the `onboarding_complete: true` from the `retell_llm_dynamic_variables` in the `createPhoneCall` call — this variable was being set prematurely. Change:
```javascript
retell_llm_dynamic_variables: {
  business_name: tenant.business_name,
  onboarding_complete: true,
  tone_preset: tenant.tone_preset,
},
```
To:
```javascript
retell_llm_dynamic_variables: {
  business_name: tenant.business_name,
  tone_preset: tenant.tone_preset,
},
```

Wait — actually, `onboarding_complete` as a dynamic variable is needed by the agent prompt to inject the TRIAGE-AWARE BEHAVIOR section. Since we are triggering a test call, we want the AI to behave as if onboarding is complete during the test call. So KEEP `onboarding_complete: true` in `retell_llm_dynamic_variables` but REMOVE it from the `.update()` call. The dynamic variable controls the AI behavior during the call; the DB flag controls wizard completion state. These are separate concerns.
