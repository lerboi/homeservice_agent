## Spacing Scale

Declared values (must be multiples of 4). This is an extension of the existing wizard layout which uses `px-6 py-8 sm:px-8 sm:py-12`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, OTP digit gaps |
| sm | 8px | Compact element spacing, helper text gap |
| md | 16px | Default form field spacing, label-to-input gap |
| lg | 24px | Section spacing within wizard card, step heading margin-bottom |
| xl | 32px | Card internal padding (px-8) |
| 2xl | 48px | Vertical breathing room between major step sections |
| 3xl | 64px | Not used inside wizard card; layout-level only |

Exceptions:
- OTP input: 6 individual digit inputs, each 44×44px minimum touch target (WCAG 2.5.5), with 8px gaps between digits
- Progress bar height: 4px (existing — `h-1`)
- "Call My AI Now" button: minimum 52px height on mobile to ensure confidence-triggering tap target
- Radio selection cards (trade type, tone preset): minimum 72px height

Source: Existing layout.js uses `px-6 py-8 sm:py-12 max-w-lg` — preserved. Wizard card uses `px-6 py-8 sm:px-8`.

## Interaction States

### Step 1: Create Account (auth)

**Default state:**
- Google OAuth button (full-width, white background, Google icon, `#0F172A` text, `border-stone-200` border, `min-h-11`)
- "or" divider
- Email input + Password input stacked, `min-h-11` each
- "Create account" primary CTA button (Heritage Copper fill)
- "Already have an account? Sign in" text link below

**OTP inline state (transforms in-place after email signup):**
- Heading changes to "Check your email"
- Subtext: "Enter the 6-digit code we sent to {email}"
- 6 individual digit inputs in a row: `size-11` (44×44px), `text-center`, `text-xl font-semibold`, `border-stone-200`, focus ring `ring-[#C2410C]`, auto-advance cursor to next digit on single character entry, backspace steps backward
- "Resend code" text link: `text-sm text-[#475569]`, becomes disabled for 30 seconds after send
- Loader replaces submit button while verifying
- On success: AnimatedSection fade-out → navigate to `/onboarding/profile`

### Step 2: Business Profile

**Trade selection (new UI):**
- Heading: "What type of business do you run?"
- Grid of trade cards: 2 columns on mobile, 3 columns on desktop (`grid grid-cols-2 sm:grid-cols-3 gap-3`)
- Each trade card: icon (Lucide or inline SVG), trade name, `min-h-[72px]`, rounded-2xl border
- Selected state: `border-[#C2410C] bg-[#C2410C]/[0.04] shadow-[0_0_0_1px_rgba(194,65,12,0.15)]`
- Unselected: `border-stone-200 bg-[#F5F5F4] hover:-translate-y-0.5 transition-transform duration-150`

**Business name + tone preset** (existing pattern, adapted):
- Shows below trade selection after trade is selected
- Pattern from existing `onboarding/page.js` preserved exactly

### Step 3: Services

- Existing `onboarding/services/page.js` pattern preserved
- Pre-populated list based on selected trade template
- Editable inline — add / remove services

### Step 4: Contact Details

- Existing `onboarding/verify/page.js` pattern preserved (renamed "Contact Details")
- Phone and email fields with `min-h-11` inputs
- Phone is now REQUIRED (not optional) — needed for test call delivery

### Step 5: Test Call Finale

**Ready state:**
- Heading: "Meet your AI receptionist"
- Subtext: "Call the number below — your AI will answer."
- AI phone number displayed: `text-3xl font-semibold text-[#0F172A] tracking-wider` in a highlighted box (`bg-white rounded-2xl border border-stone-200 px-6 py-4 text-center`)
- "Call My AI Now" primary button (Heritage Copper, `min-h-[52px]`) — triggers Retell outbound call to owner's phone
- Small text below: "Prefer to call manually? Dial the number above."

**Calling state:**
- Button replaced by `Loader2` spinner with label "Connecting..."
- Subtext updates to: "Your phone should ring in a few seconds"

**In Progress state:**
- Pulsing green ring around a phone icon: `animate-pulse` on `ring-2 ring-[#166534]`
- Status label: "Call in progress..." with elapsed time counter

**Complete state (celebration):**
- Full AnimatedSection fade-in over the step content
- Radial pulse: 3 concentric rings expanding outward in `#C2410C` with decreasing opacity, `animation-duration: 800ms`, `animation-timing-function: ease-out`, `prefers-reduced-motion: reduce` collapses to static ring
- Animated checkmark SVG: circle draws in (stroke-dashoffset animation, 500ms), then checkmark tick draws in (300ms delay)
- Heading: "Your AI receptionist is live!" — `text-3xl font-semibold text-[#0F172A]`
- Subtext: "Your first call has been answered. The AI is ready to capture every lead."
- "Go to Dashboard" button: Heritage Copper fill, full-width, `min-h-11`

## Interaction Patterns

### Progress bar
- Located in `onboarding/layout.js`, reads pathname
- `h-1` bar, `bg-[#0F172A]/[0.08]` track, `bg-[#C2410C]` fill
- `transition-all duration-500 ease-out` on width change
- 5 steps: `/onboarding`=20%, `/onboarding/profile`=40%, `/onboarding/services`=60%, `/onboarding/contact`=80%, `/onboarding/test-call`=100%
- Step counter text: "Step 2 of 5" pattern, `text-sm text-[#475569]`

### Step navigation
- "Continue" / "Next" buttons: Heritage Copper, right-aligned (or full-width on mobile)
- "Back" links: `variant="ghost"`, `text-[#475569] hover:text-[#0F172A]`, left-aligned, `px-0`
- No "Skip" buttons anywhere in the wizard (mandatory flow per CONTEXT.md)

### Form validation
- Inline field-level errors: `text-sm text-red-600` below the field, `role="alert"`, `aria-describedby` on input
- Alert component for API-level errors: `<Alert variant="destructive">` above form
- Inputs use `aria-invalid={!!error}` when errored

### Loading states
- Spinner: `<Loader2 className="h-4 w-4 animate-spin" aria-hidden="true" />` replaces button label text
- Button disabled during async operations: `disabled={loading}`
- No skeleton loaders needed (content is user-entered, not fetched)

### sessionStorage persistence
- Key prefix: `gsd_onboarding_` (e.g., `gsd_onboarding_step`, `gsd_onboarding_trade`, `gsd_onboarding_business_name`)
- Written on every field blur / selection change
- Read on component mount to restore state
- Cleared on wizard completion (after "Go to Dashboard" click or redirect)

### Reduced motion
- All Framer Motion animations wrap with `useReducedMotion()` check (existing pattern from `AnimatedSection`)
- Celebration radial pulse: reduced-motion collapses to `opacity-0 → opacity-1` fade only (no scale/pulse)
- OTP auto-advance: no motion, just focus transfer (no animation needed)

## Step 5 Animation Specification

### Radial pulse (celebration)

```
3 concentric rings centered on checkmark icon
Ring 1: scale 1→2, opacity 0.4→0, duration 800ms, delay 0ms
Ring 2: scale 1→2, opacity 0.3→0, duration 800ms, delay 200ms
Ring 3: scale 1→2, opacity 0.2→0, duration 800ms, delay 400ms
Color: #C2410C
easing: ease-out
repeat: once (not looping)
prefers-reduced-motion: omit rings entirely, show static checkmark only
```

### Checkmark SVG draw-in

```
Circle stroke-dashoffset: full path length → 0, duration 500ms, ease-out
Check tick stroke-dashoffset: full path length → 0, duration 300ms, delay 400ms, ease-out
Color: #166534 (success green)
prefers-reduced-motion: skip animation, render static checkmark at full opacity
```

### AnimatedSection wrapping complete state

```
initial: { opacity: 0, y: 8 }
animate: { opacity: 1, y: 0 }
transition: { duration: 0.35, ease: "easeOut" }
prefers-reduced-motion: initial={false} (Framer Motion v12 pattern — existing codebase)
```

## Accessibility Contract

- All form inputs have associated `<Label htmlFor>` — no placeholder-only labeling
- OTP inputs: `aria-label="Digit {n} of 6"` on each input, group wrapped in `role="group" aria-label="Email verification code"`
- Trade selector grid: `role="radiogroup" aria-label="Select your trade"`, each card `role="radio" aria-checked={selected}`
- Tone preset radio cards: existing ARIA pattern from `onboarding/page.js` preserved
- Progress bar: `role="progressbar" aria-valuenow={step} aria-valuemin={1} aria-valuemax={5} aria-label="Onboarding progress"`
- Test call status panel: `aria-live="polite"` wrapper so state changes are announced to screen readers
- All buttons have accessible names (no icon-only buttons)
- Error messages: `role="alert"` + `aria-describedby` linking to field
- Minimum contrast: `#475569` on `#FFFFFF` = 4.63:1 (passes AA); `#0F172A` on `#FFFFFF` = 16.8:1 (passes AAA); `#C2410C` on `#FFFFFF` = 4.52:1 (passes AA for large text / 18px+)
- Focus rings: `focus:ring-2 focus:ring-[#C2410C] focus:ring-offset-1` on all interactive elements
- Keyboard: Tab order follows visual DOM order; OTP auto-advance does not trap focus
