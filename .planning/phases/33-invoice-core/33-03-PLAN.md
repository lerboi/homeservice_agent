**Create `src/app/api/invoices/route.js`:**

**GET handler** (list invoices):
1. Auth: `createSupabaseServer()`, `getTenantId()`, return 401 if null.
2. Read optional query params: `status` (filter by status), `search` (filter by customer_name or invoice_number).
3. First, run overdue bulk update per D-13: update all invoices where `tenant_id = tenantId AND status = 'sent' AND due_date < today` to `status = 'overdue'`. Use `new Date().toISOString().split('T')[0]` for today's date.
4. Query `invoices` where `tenant_id = tenantId`, ordered by `created_at DESC`.
5. If `status` param provided and is one of ('draft', 'sent', 'paid', 'overdue', 'void'), filter by it.
6. Also query aggregate data for summary cards per D-15:
   - Total Outstanding: `SUM(total) WHERE status IN ('sent', 'overdue')`
   - Overdue Amount: `SUM(total) WHERE status = 'overdue'`
   - Paid This Month: `SUM(total) WHERE status = 'paid' AND paid_at >= first day of current month`
   - Status counts: `COUNT(*) GROUP BY status` for tab badges

   Run these as separate queries (3 aggregate queries + 1 list query). For paid_this_month, use `new Date(new Date().getFullYear(), new Date().getMonth(), 1).toISOString()` for first day of month.
7. Return `{ invoices, summary: { total_outstanding, overdue_amount, paid_this_month }, status_counts }`.

**POST handler** (create invoice):
1. Auth check.
2. Parse body: `{ lead_id?, customer_name, customer_phone, customer_email, customer_address, job_type, issued_date, due_date, notes, payment_terms, line_items: [...] }`.
3. Fetch invoice settings for this tenant: query `invoice_settings` where `tenant_id = tenantId`. If none exists, use defaults (prefix: 'INV', tax_rate: 0, payment_terms: 'Net 30').
4. Get next invoice number: call the Postgres RPC function:
   ```javascript
   const currentYear = new Date().getFullYear();
   const { data: seqData } = await supabase.rpc('get_next_invoice_number', {
     p_tenant_id: tenantId,
     p_year: currentYear
   });
   ```
   Then format: `formatInvoiceNumber(settings.invoice_prefix || 'INV', currentYear, seqData)`. Import `formatInvoiceNumber` from `@/lib/invoice-number`.
5. Calculate totals from line_items using `calculateLineTotal` and `calculateInvoiceTotals` from `@/lib/invoice-calculations`. Import both functions.
6. Insert into `invoices` table with all fields + `invoice_number`, `subtotal`, `tax_amount`, `total`, `tenant_id`.
7. Insert all `line_items` into `invoice_line_items` table with `invoice_id`, `tenant_id`, and each item's fields. Calculate `line_total` for each using `calculateLineTotal`.
8. Return 201 with the created invoice + line items.

**Create `src/app/api/invoices/[id]/route.js`:**

**GET handler** (single invoice detail):
1. Auth check.
2. Fetch invoice where `id = params.id AND tenant_id = tenantId`.
3. If not found, return 404.
4. Fetch associated `invoice_line_items` where `invoice_id = params.id`, ordered by `sort_order ASC`.
5. Return `{ invoice, line_items }`.

**PATCH handler** (update invoice):
1. Auth check.
2. Fetch current invoice to verify ownership and get current status.
3. Parse body. Allowed fields depend on status:
   - Draft: all fields editable (customer info, dates, notes, terms, line_items, status)
   - Sent/Overdue: only `status` can change (to 'paid' or 'void')
   - Paid/Void: no edits allowed (return 400)
4. If `status` is changing to 'paid': set `paid_at = new Date().toISOString()`.
5. If `status` is changing to 'void': set `voided_at = new Date().toISOString()`.
6. If `line_items` provided (draft edit): delete existing line items for this invoice, insert new ones, recalculate totals.
7. Update the invoice row.
8. Return updated invoice + line items.

Note: Bidirectional sync (D-16) is handled in Plan 07 — this plan only handles the core CRUD.
