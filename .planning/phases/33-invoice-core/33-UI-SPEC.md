## Interaction Patterns

### Invoice List Page

1. **Summary cards row**: 3 cards at top, equal width on desktop (3-col grid), stacked on mobile. Each card: icon, label, monetary value (Display size), subtle trend indicator optional.
2. **Status filter tabs**: Horizontal tab row below summary cards. "All" selected by default. Counts shown per tab (e.g., "Draft (3)"). Active tab uses brandOrange underline, 2px.
3. **Invoice table**: Full-width shadcn Table. Columns: Invoice #, Customer, Job Type, Amount, Issued, Due, Status. Rows clickable (entire row is click target). Hover: stone-50 background. Mobile: card layout instead of table (invoice #, customer, amount, status badge visible; other fields hidden).
4. **Empty state**: Centered in table area. Invoice illustration icon (FileText from lucide), heading, body, CTA button.

### Invoice Editor

1. **Header section**: Invoice number (auto-generated, read-only), customer info (name, email, phone, address — pre-filled from lead if applicable), issue date (default today), due date (default today + payment terms).
2. **Line items section**: Each row is a LineItemRow. "Add Line Item" button at bottom of list. Each row has a trash icon (lucide Trash2) to remove. Drag handle optional (Claude's discretion). Type selector changes which fields are visible (e.g., Materials shows markup %, Labor shows hours).
3. **Totals section**: Right-aligned below line items. Subtotal, Tax line (only if any taxable items), Total Due. Display typography for Total Due.
4. **Notes/Terms section**: Textarea for notes (pre-filled from settings default), textarea for terms (pre-filled from settings default). Both editable per invoice.
5. **Action bar**: Sticky bottom bar on mobile, right-aligned button group on desktop. "Save as Draft" (outline), "Preview PDF" (outline), "Send Invoice" (primary/accent).

### Invoice Detail View

1. **Approach**: Full page at `/dashboard/invoices/[id]` (not a flyout — invoices have too much content for a sheet).
2. **Layout**: Left column (70%) shows the invoice rendered as it will appear on PDF. Right column (30%) shows actions and metadata (status badge, dates, delivery history, action buttons).
3. **Actions**: "Send Invoice" (if Draft), "Download PDF", "Mark as Paid" (if Sent/Overdue), "Void" (if not already Void/Paid), "Edit" (if Draft only).
4. **Mobile**: Single column, actions as sticky bottom bar, invoice preview scrollable above.

### Invoice Settings Page

1. **Location**: `src/app/dashboard/more/invoice-settings/page.js` (linked from More menu and from Invoices tab gear icon).
2. **Sections**: Business Identity (name, address, phone, email, logo upload, license #), Tax Configuration (single rate input with % suffix), Invoice Defaults (payment terms dropdown, notes textarea, terms textarea), Numbering (prefix input, next number display read-only).
3. **Logo upload**: Square preview area (96x96px) with upload overlay. Accept PNG/JPG, max 2MB. Show current logo or placeholder icon.
4. **Save**: Single "Save Settings" button at bottom. Persists all fields in one API call.

### LeadFlyout Integration

1. **"Create Invoice" button**: Appears in LeadFlyout action area when lead status is "completed" or "paid" and no linked invoice exists. Uses brandOrange outline variant. Icon: FileText (lucide).
2. **If invoice already exists**: Show "View Invoice" link instead, navigating to the existing invoice detail page.
3. **Pre-fill**: Navigates to `/dashboard/invoices/new?lead_id={id}`. Editor pre-fills customer name, phone, address, job type from lead data.

### Navigation Changes

1. **Sidebar**: Replace `{ href: '/dashboard/analytics', label: 'Analytics', icon: BarChart3 }` with `{ href: '/dashboard/invoices', label: 'Invoices', icon: FileText }` in NAV_ITEMS.
2. **More menu**: Add `{ href: '/dashboard/analytics', label: 'Analytics', icon: BarChart3 }` to MORE_ITEMS array in `src/app/dashboard/more/page.js`.
3. **More menu**: Add `{ href: '/dashboard/more/invoice-settings', label: 'Invoice Settings', icon: Settings }` to MORE_ITEMS.

## PDF Layout Contract

The PDF is the customer-facing artifact. It must be fully white-labeled (D-09).

### PDF Structure (top to bottom)

1. **Header**: Business logo (left, max 120px wide), business name + address + phone + email + license # (right-aligned).
2. **Invoice meta**: "INVOICE" title, invoice number, issue date, due date. Right-aligned block below header.
3. **Bill To**: Customer name, address, phone, email. Left-aligned block.
4. **Line items table**: Columns: Description, Qty, Rate, Amount. Materials rows show markup in description (e.g., "Copper pipe (15% markup)"). Discount rows show negative amount. Alternating row shading (light gray / white).
5. **Totals**: Right-aligned: Subtotal, Tax ({rate}%), **Total Due** (bold, larger).
6. **Footer**: Payment terms, notes, "Thank you for your business!" (or custom).
7. **No Voco branding anywhere** — no "Powered by", no URLs, no logos.
