1. Create `src/app/api/invoices/batch/route.js`:

POST handler accepting `{ lead_ids: string[] }`:
- Auth: `createSupabaseServer()` + `getTenantId()` — 401 if no tenantId
- Validate `lead_ids` is a non-empty array, max 50 items — 400 if invalid
- Fetch all leads: `supabase.from('leads').select('*').in('id', lead_ids).eq('tenant_id', tenantId)`
- Validate all leads belong to tenant (RLS handles this but check count matches)
- Filter to only leads with status 'completed' — return error info for non-completed leads
- Check none already have an invoice: `supabase.from('invoices').select('lead_id').in('lead_id', lead_ids).eq('tenant_id', tenantId)` — exclude leads that already have invoices
- Fetch invoice settings: `supabase.from('invoice_settings').select('*').eq('tenant_id', tenantId).maybeSingle()`
- For each valid lead, create a draft invoice:
  - Get next invoice number via `supabase.rpc('get_next_invoice_number', { p_tenant_id: tenantId, p_year: currentYear })`
  - Format with `formatInvoiceNumber(prefix, year, seq)`
  - Insert invoice row with: `tenant_id`, `lead_id: lead.id`, `invoice_number`, `status: 'draft'`, `customer_name: lead.caller_name`, `customer_phone: lead.caller_phone`, `customer_email: lead.caller_email`, `customer_address: lead.address`, `job_type: lead.job_type`, `issued_date: today`, `due_date: null` (owner sets during review), `payment_terms: settings.payment_terms || 'Net 30'`, `subtotal: 0`, `tax_amount: 0`, `total: 0`
  - No line items created (owner adds during review per D-11 — drafts are shells)
- Collect results: `{ invoices: [...created], errors: [{ lead_id, reason }] }`
- Return 201 with results

2. Create `src/app/api/invoices/batch-send/route.js`:

POST handler accepting `{ invoice_ids: string[] }`:
- Auth: `createSupabaseServer()` + `getTenantId()` — 401 if no tenantId
- Validate `invoice_ids` is non-empty array, max 50 — 400 if invalid
- Fetch all invoices: `supabase.from('invoices').select('*').in('id', invoice_ids).eq('tenant_id', tenantId)`
- For each invoice, call the existing send logic sequentially (import from a shared function or inline):
  - Validate invoice has customer_email
  - Fetch line items for this invoice
  - Fetch invoice settings (once, reuse for all)
  - Generate PDF via `renderToBuffer(<InvoicePDF ... />)`
  - Send email via `getResendClient().emails.send(...)`
  - Update status to 'sent'
  - Track result: `{ invoice_id, status: 'sent' }` on success, `{ invoice_id, status: 'failed', error: message }` on failure
- Do NOT stop on first failure — continue sending remaining invoices
- Return 200 with `{ results: [...], summary: { sent: N, failed: M } }`
- Import `renderToBuffer` from `@react-pdf/renderer`, `InvoicePDF` from `@/lib/invoice-pdf`, `InvoiceEmail` from `@/emails/InvoiceEmail`, `getResendClient` from `@/lib/notifications`
