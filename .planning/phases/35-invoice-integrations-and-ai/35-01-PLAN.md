```sql
CREATE TABLE accounting_credentials (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  provider        text NOT NULL
    CHECK (provider IN ('quickbooks', 'xero', 'freshbooks')),
  access_token    text NOT NULL,
  refresh_token   text NOT NULL,
  expiry_date     bigint,
  realm_id        text,
  xero_tenant_id  text,
  account_id      text,
  display_name    text,
  connected_at    timestamptz NOT NULL DEFAULT now(),
  last_synced_at  timestamptz,
  created_at      timestamptz NOT NULL DEFAULT now(),
  UNIQUE (tenant_id, provider)
);

ALTER TABLE accounting_credentials ENABLE ROW LEVEL SECURITY;

CREATE POLICY "accounting_credentials_tenant_own" ON accounting_credentials
  FOR ALL USING (tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid()))
  WITH CHECK (tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid()));

CREATE POLICY "service_role_all_accounting_credentials" ON accounting_credentials
  FOR ALL USING (auth.role() = 'service_role');

CREATE TABLE accounting_sync_log (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  invoice_id      uuid NOT NULL REFERENCES invoices(id) ON DELETE CASCADE,
  provider        text NOT NULL,
  external_id     text,
  status          text NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending', 'synced', 'failed')),
  error_message   text,
  attempted_at    timestamptz NOT NULL DEFAULT now(),
  synced_at       timestamptz,
  UNIQUE (invoice_id, provider)
);

ALTER TABLE accounting_sync_log ENABLE ROW LEVEL SECURITY;

CREATE POLICY "accounting_sync_log_tenant_own" ON accounting_sync_log
  FOR ALL USING (tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid()))
  WITH CHECK (tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid()));

CREATE POLICY "service_role_all_accounting_sync_log" ON accounting_sync_log
  FOR ALL USING (auth.role() = 'service_role');

CREATE INDEX idx_accounting_sync_log_invoice ON accounting_sync_log(invoice_id);
CREATE INDEX idx_accounting_sync_log_tenant ON accounting_sync_log(tenant_id, status);
```

   - `@typedef {Object} AccountingAdapter` with methods: `pushInvoice(invoice, lineItems, settings) => Promise<{externalId: string}>`, `updateInvoiceStatus(externalId, status) => Promise<void>`, `getAuthUrl(tenantId, redirectUri) => string`, `exchangeCode(code, redirectUri, extraParams) => Promise<TokenSet>`, `refreshToken(tokenSet) => Promise<TokenSet>`, `findOrCreateCustomer(customerName, customerEmail) => Promise<string>`
   - `@typedef {Object} TokenSet` with `access_token`, `refresh_token`, `expiry_date`, `realm_id?`, `xero_tenant_id?`, `account_id?`, `display_name?`
   - `@typedef {Object} ExternalInvoice` with `customerName`, `customerEmail`, `lineItems` array, `invoiceNumber`, `issuedDate`, `dueDate`, `taxRate`
   - Export `PROVIDERS = ['quickbooks', 'xero', 'freshbooks']`

2. Create `src/lib/accounting/adapter.js`:
   - `export async function getAccountingAdapter(provider)` — validates provider is in PROVIDERS, dynamically imports the correct adapter module (`./quickbooks.js`, `./xero.js`, `./freshbooks.js`), returns adapter instance
   - `export async function refreshTokenIfNeeded(supabase, credentials)` — checks `credentials.expiry_date` against `Date.now()` with 5-minute buffer. If expired or within 5 minutes, calls `adapter.refreshToken(credentials)`, upserts new tokens to `accounting_credentials` table, returns updated credentials. If not expired, returns original credentials unchanged.

3. Create `src/lib/accounting/quickbooks.js`:
   - Import `intuit-oauth` and `node-quickbooks`
   - `export class QuickBooksAdapter` implementing the AccountingAdapter interface:
     - Constructor takes env vars: `QUICKBOOKS_CLIENT_ID`, `QUICKBOOKS_CLIENT_SECRET`, `QUICKBOOKS_ENVIRONMENT` (sandbox/production)
     - `getAuthUrl(tenantId, redirectUri)`: use intuit-oauth to generate authorization URL with scope `com.intuit.quickbooks.accounting`
     - `exchangeCode(code, redirectUri, extraParams)`: exchange auth code for tokens via intuit-oauth, extract `realmId` from `extraParams.realmId`, return TokenSet
     - `refreshToken(tokenSet)`: use intuit-oauth `refreshUsingToken()`, return new TokenSet
     - `findOrCreateCustomer(customerName, customerEmail)`: Query QBO customers by email, if not found create one, return customer ID string
     - `pushInvoice(invoice, lineItems, settings)`: Map to QBO Invoice payload per research (CustomerRef, Line array with SalesItemLineDetail, DueDate, DocNumber), POST to QBO API, return `{externalId: response.Id}`
     - `updateInvoiceStatus(externalId, status)`: If status is 'void', void the QBO invoice. If 'paid', record payment. Uses node-quickbooks SDK.

4. Create `src/lib/accounting/xero.js`:
   - Import `xero-node` XeroClient
   - `export class XeroAdapter` implementing the AccountingAdapter interface:
     - Constructor configures XeroClient with `XERO_CLIENT_ID`, `XERO_CLIENT_SECRET`
     - `getAuthUrl(tenantId, redirectUri)`: build Xero consent URL with scopes `openid profile email accounting.transactions accounting.contacts`
     - `exchangeCode(code, redirectUri, extraParams)`: exchange code, call connections endpoint to get tenant list, if single tenant auto-select, if multiple return first (xero_tenant_id stored). Return TokenSet with `xero_tenant_id`
     - `refreshToken(tokenSet)`: use XeroClient token refresh, return new TokenSet
     - `findOrCreateCustomer(customerName, customerEmail)`: search Xero contacts by email, create if not found, return ContactID
     - `pushInvoice(invoice, lineItems, settings)`: Map to Xero Invoice payload (Type: ACCREC, Contact, LineItems with Description/Quantity/UnitAmount/AccountCode '200', Status: AUTHORISED), return `{externalId: InvoiceID}`
     - `updateInvoiceStatus(externalId, status)`: void = update Status to VOIDED, paid = record Xero payment

5. Create `src/lib/accounting/freshbooks.js`:
   - Import `@freshbooks/api` Client
   - `export class FreshBooksAdapter` implementing the AccountingAdapter interface:
     - Constructor configures with `FRESHBOOKS_CLIENT_ID`, `FRESHBOOKS_CLIENT_SECRET`
     - `getAuthUrl`, `exchangeCode`, `refreshToken`: standard OAuth using FreshBooks SDK
     - `exchangeCode` extracts `account_id` from the token response and stores it
     - `findOrCreateCustomer`: search FreshBooks clients by email, create if not found, return clientId
     - `pushInvoice`: Map to FreshBooks invoice payload (customerid, create_date, lines with type/name/description/qty/unit_cost), return `{externalId: invoiceId}`
     - `updateInvoiceStatus`: void = delete/mark void, paid = mark as paid
