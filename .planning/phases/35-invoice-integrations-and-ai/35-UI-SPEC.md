## Interaction Patterns

### Integrations Page

1. **Location**: New page at `/dashboard/more/integrations`. Added to MORE_ITEMS in `src/app/dashboard/more/page.js` as `{ href: '/dashboard/more/integrations', label: 'Integrations', icon: Plug }`.
2. **Layout**: "Accounting Software" section heading with description body text. Below: 3 integration cards in a responsive grid (3-col on desktop, stacked on mobile).
3. **Integration card anatomy**: Provider logo placeholder (32x32 icon area, use BookOpen for QBO, FileSpreadsheet for Xero, Receipt for FreshBooks from lucide), provider name (Label weight), connection status text (Body weight, stone-500 when disconnected, emerald-600 when connected), last synced timestamp (Body, stone-400). Connect button (primary accent) or Disconnect button (destructive outline) at card bottom.
4. **Connect flow**: Click "Connect {provider}" triggers GET to `/api/accounting/{provider}/auth`, which redirects to OAuth consent screen. After consent, callback route stores tokens and redirects back to `/dashboard/more/integrations` with `?connected={provider}` query param. Page shows success toast on mount if param present.
5. **Xero multi-org**: If Xero OAuth returns multiple organizations, show XeroTenantSelector dialog with radio list of org names. User selects one, clicks "Connect". Single org auto-selected (no dialog).
6. **Disconnect flow**: Click "Disconnect" opens AlertDialog confirmation. On confirm, POST to `/api/accounting/disconnect` with provider. Card reverts to "Not connected" state.

### AI Describe (Invoice Editor Extension)

1. **Button placement**: "AI Describe" button placed in the line items section header, right-aligned next to "Add Line Item". Uses Sparkles icon (lucide) + "AI Describe" text. Accent outline style (`border-[#C2410C] text-[#C2410C] hover:bg-[#C2410C]/5`).
2. **Disabled state**: Button disabled with tooltip when invoice has no linked lead_id or linked lead has no call transcripts. Gray outline, cursor-not-allowed.
3. **Generation flow**: Click triggers POST to `/api/invoices/{id}/ai-describe`. Button shows spinner + "Generating..." text. On success, each line item shows a preview row below it with the AI-generated description in a highlighted card (stone-50 background, left border 2px brandOrange). Each preview row has "Accept" (fills the line item description) and "Discard" (removes preview) buttons. "Accept All" / "Discard All" buttons appear above the line items section.
4. **After accept**: The AI preview row disappears and the line item description field is populated with the accepted text. Owner can further edit. Accepting does not auto-save; owner must save the invoice normally.
5. **Error handling**: If generation fails, toast error. Button returns to idle state.

### Batch Invoicing (Leads Page Extension)

1. **Multi-select activation**: Checkbox column added to the leads table/list (leftmost column). Checkboxes only appear on leads with status "completed" that do not already have a linked invoice. A "select all (on page)" checkbox in the header row.
2. **Batch select bar**: When 1+ leads are selected, a sticky bottom bar appears (stone-900 background, white text). Shows count text + "Create Invoices" button (white background, stone-900 text). Bar sits above any existing bottom nav on mobile.
3. **Batch creation**: Click "Create Invoices" opens confirmation dialog. On confirm, POST to `/api/invoices/batch` with lead_ids array. Response returns created draft invoices. User is redirected to batch review page.
4. **Batch review page**: At `/dashboard/invoices/batch-review` (or modal/sheet -- Claude's discretion). Shows a card list of created drafts. Each card: invoice number, customer name, amount (pre-filled as $0 since no line items added yet), status "Draft", "Edit" link (navigates to invoice editor), "Remove" button (deletes the draft). "Send All" button at top right. Back/Cancel returns to invoices list.
5. **Batch send**: Click "Send All" opens confirmation dialog. On confirm, sequential POST to `/api/invoices/{id}/send` for each. Progress bar shows current/total. Results displayed per-invoice: green check for success, red X with error message for failure. When complete, summary toast. "Done" button navigates to invoice list.

### Invoice List Extension — Sync Status

1. **Sync indicator**: On the invoice list table, invoices that have been pushed to accounting software show a small icon next to the invoice number. Synced: CheckCircle2 in emerald-500 (16px). Failed: AlertCircle in red-500 (16px). Pending: Clock in amber-500 (16px). Icon wrapped in Tooltip showing "{provider}: {status}" (e.g., "QuickBooks: Synced").
2. **No sync column**: The sync indicator is inline with the invoice number cell, not a separate column. Keeps table compact.
3. **No indicator if no accounting connected**: If tenant has no accounting_credentials, no sync indicators shown anywhere.

## Loading & Error States

| View | Loading State | Error State |
|------|--------------|-------------|
| Integrations page | 3 Skeleton cards (same height as integration cards) | Toast: "Couldn't load integrations. Try refreshing." |
| OAuth callback | Full-page centered spinner with "Connecting to {provider}..." | Redirect to integrations page with error toast |
| AI Describe | Button spinner + "Generating..." (line items remain visible) | Toast: "Couldn't generate descriptions. Check that a call transcript is linked and try again." |
| Batch create | Button spinner + disabled during POST | Toast: "Couldn't create batch invoices. Try again." |
| Batch send progress | Progress bar with count text | Per-invoice error in results list; summary toast |
| Sync push (background) | No loading indicator (fire-and-forget) | Sync status badge shows "Failed"; error in accounting_sync_log |
