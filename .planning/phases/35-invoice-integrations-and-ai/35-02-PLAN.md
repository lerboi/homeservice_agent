```javascript
import { GoogleGenAI } from '@google/genai';

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
```

Export `async function getTranscriptsForLead(supabase, leadId, tenantId)`:
- Query `lead_calls` table: `select('call_id').eq('lead_id', leadId)`
- If no lead_calls rows, return `null`
- Extract call_id array, query `calls` table: `select('transcript_text, created_at').in('id', callIds).eq('tenant_id', tenantId).order('created_at', { ascending: true })`
- Filter out calls with no transcript_text
- Join transcripts with `'\n\n---\n\n'` separator
- Return combined transcript string, or null if all empty

Export `async function generateLineItemDescriptions(transcript, existingLineItems)`:
- Build prompt (from RESEARCH.md verbatim):
```
You are helping a home service contractor write professional invoice line item descriptions.

Given the call transcript below, generate a clear, professional description for each line item on this invoice.

RULES:
- Describe ONLY work explicitly discussed in the transcript
- Use professional trade terminology (plumbing, HVAC, electrical)
- Do NOT suggest pricing, labor hours, or materials quantities
- Do NOT invent brand names, model numbers, or part numbers unless the caller stated them
- Keep each description to 1-2 sentences
- If the transcript doesn't mention relevant work for a line item, return the existing description unchanged

EXISTING LINE ITEMS:
${existingLineItems.map((item, i) => `${i + 1}. [${item.item_type}] ${item.description || '(empty)'}`).join('\n')}

CALL TRANSCRIPT:
${transcript}

Return a JSON array of strings, one description per line item, in the same order. Return ONLY the JSON array, no markdown formatting.
```
- Call `ai.models.generateContent({ model: 'gemini-2.0-flash', contents: prompt })`
- Parse `response.text` as JSON — strip any markdown code fences before parsing
- Return the string array

   - `const [aiDescriptions, setAiDescriptions] = useState(null)` — array of generated description strings, or null
   - `const [aiGenerating, setAiGenerating] = useState(false)` — loading state
   - `const [aiAccepted, setAiAccepted] = useState([])` — array of booleans tracking which descriptions were accepted

4. Add handler `async function handleAiDescribe()`:
   - Set `setAiGenerating(true)`
   - `fetch(`/api/invoices/${invoiceId}/ai-describe`, { method: 'POST' })`
   - On success: parse response JSON, `setAiDescriptions(data.descriptions)`, `setAiAccepted(new Array(data.descriptions.length).fill(false))`, show toast "Descriptions generated. Review and accept below." via sonner

   - Set `setAiGenerating(false)`

5. Add handler `function handleAcceptDescription(index)`:
   - Copy current lineItems, set `lineItems[index].description = aiDescriptions[index]`
   - Update lineItems state
   - Mark `aiAccepted[index] = true` in aiAccepted state
   - If all accepted, clear `setAiDescriptions(null)`

6. Add handler `function handleAcceptAll()`:
   - Copy all lineItems, for each index set description to aiDescriptions[index]
   - Update lineItems state
   - Clear `setAiDescriptions(null)`

7. Add handler `function handleDiscardAll()`:
   - `setAiDescriptions(null)`

9. When `aiDescriptions` is not null and not all accepted, show "Accept All" / "Discard All" buttons above the line items list:
