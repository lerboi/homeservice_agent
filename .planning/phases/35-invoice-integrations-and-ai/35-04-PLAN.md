- On any error: redirect to `/dashboard/more/integrations?error=connection_failed&provider=${provider}`

1. Create `src/lib/accounting/sync.js`:

Export `async function pushToAccounting(supabase, tenantId, invoice, lineItems, settings)`:
- Fetch accounting credentials: `supabase.from('accounting_credentials').select('*').eq('tenant_id', tenantId).maybeSingle()`
- If no credentials, return early (no accounting connected — this is normal)
- Refresh token if needed: `await refreshTokenIfNeeded(supabase, credentials)` — returns updated creds
- Get adapter: `await getAccountingAdapter(credentials.provider)`
- Initialize adapter with credentials (pass access_token, realm_id/xero_tenant_id/account_id as needed)
- Find or create customer: `await adapter.findOrCreateCustomer(invoice.customer_name, invoice.customer_email)`
- Push invoice: `const result = await adapter.pushInvoice(invoice, lineItems, settings)`
- Log success to `accounting_sync_log`:
  ```javascript
  await supabase.from('accounting_sync_log').upsert({
    tenant_id: tenantId,
    invoice_id: invoice.id,
    provider: credentials.provider,
    external_id: result.externalId,
    status: 'synced',
    synced_at: new Date().toISOString(),
    attempted_at: new Date().toISOString(),
    error_message: null,
  }, { onConflict: 'invoice_id,provider' })
  ```
- Update `accounting_credentials.last_synced_at`
- On any error, log failure to `accounting_sync_log` with status 'failed' and error_message, then re-throw

Export `async function pushStatusUpdate(supabase, tenantId, invoiceId, newStatus)`:
- Only process 'paid' or 'void' statuses — return early for anything else
- Fetch sync log: `supabase.from('accounting_sync_log').select('*').eq('invoice_id', invoiceId).eq('status', 'synced').maybeSingle()`
- If no synced record, return (invoice was never pushed)
- Fetch credentials: `supabase.from('accounting_credentials').select('*').eq('tenant_id', tenantId).eq('provider', syncLog.provider).maybeSingle()`
- If no credentials (disconnected), return
- Refresh token, get adapter, call `adapter.updateInvoiceStatus(syncLog.external_id, newStatus)`
- Update sync log status if needed

2. Modify `src/lib/invoice-send.js` (the shared send function from Plan 03):

IMPORTANT: Hook the accounting push INTO the `sendSingleInvoice` function, AFTER the status update to 'sent' succeeds but BEFORE the function returns. This ensures both single-send and batch-send inherit the accounting sync automatically.

Add at the end of `sendSingleInvoice`, after the status update succeeds and before the final return:

```javascript
// ── Push to accounting software (non-fatal) ──────────────────────────
try {
  const { pushToAccounting } = await import('@/lib/accounting/sync.js');
  await pushToAccounting(supabase, tenantId, updatedInvoice, lineItems || [], settings);
} catch (err) {
  console.warn('[accounting-sync] Push failed (non-fatal):', err?.message || err);
}
```

Use dynamic `import()` so the accounting module is lazy-loaded and does not create a hard dependency — if accounting is not connected, the function returns early anyway. The push is non-fatal — catch and log only. The send response is NOT blocked by sync failures.

Because `sendSingleInvoice` returns `{ invoice, lineItems, settings }`, the accounting push has access to all data it needs without re-fetching.
