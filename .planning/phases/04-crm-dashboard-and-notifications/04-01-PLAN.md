Create `supabase/migrations/004_leads_crm.sql` with the following exact SQL:

**leads table:**
```sql
CREATE TABLE leads (
  id               uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id        uuid NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  from_number      text NOT NULL,
  caller_name      text,
  job_type         text,
  service_address  text,
  urgency          text NOT NULL DEFAULT 'routine'
    CHECK (urgency IN ('emergency', 'routine', 'high_ticket')),
  status           text NOT NULL DEFAULT 'new'
    CHECK (status IN ('new', 'booked', 'completed', 'paid', 'lost')),
  revenue_amount   numeric(10,2),
  primary_call_id  uuid REFERENCES calls(id) ON DELETE SET NULL,
  appointment_id   uuid REFERENCES appointments(id) ON DELETE SET NULL,
  created_at       timestamptz NOT NULL DEFAULT now(),
  updated_at       timestamptz NOT NULL DEFAULT now()
);
```

**lead_calls junction table** (one lead, many calls for repeat callers):
```sql
CREATE TABLE lead_calls (
  lead_id  uuid NOT NULL REFERENCES leads(id) ON DELETE CASCADE,
  call_id  uuid NOT NULL REFERENCES calls(id) ON DELETE CASCADE,
  PRIMARY KEY (lead_id, call_id)
);
```

**Indexes:**
```sql
CREATE INDEX idx_leads_tenant_status ON leads(tenant_id, status);
CREATE INDEX idx_leads_tenant_phone ON leads(tenant_id, from_number);
CREATE INDEX idx_leads_tenant_created ON leads(tenant_id, created_at DESC);
```

**RLS policies** (follow exact pattern from 001/002/003):
```sql
ALTER TABLE leads ENABLE ROW LEVEL SECURITY;
CREATE POLICY "leads_tenant_own" ON leads
  FOR ALL
  USING (tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid()))
  WITH CHECK (tenant_id IN (SELECT id FROM tenants WHERE owner_id = auth.uid()));
CREATE POLICY "service_role_all_leads" ON leads
  FOR ALL USING (auth.role() = 'service_role');
```

Same RLS pattern for lead_calls (join through leads.tenant_id).

**Realtime publication** (CRITICAL per RESEARCH.md Pitfall 1):
```sql
ALTER PUBLICATION supabase_realtime ADD TABLE leads;
ALTER TABLE leads REPLICA IDENTITY FULL;
```

**activity_log table** for dashboard home feed:
```sql
CREATE TABLE activity_log (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id   uuid NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  event_type  text NOT NULL,
  lead_id     uuid REFERENCES leads(id) ON DELETE SET NULL,
  metadata    jsonb,
  created_at  timestamptz NOT NULL DEFAULT now()
);
CREATE INDEX idx_activity_log_tenant ON activity_log(tenant_id, created_at DESC);
```
Add RLS for activity_log (same tenant_own + service_role pattern).

Create `src/lib/leads.js` with two exported functions:

**createOrMergeLead({ tenantId, callId, fromNumber, callerName, jobType, serviceAddress, triageResult, appointmentId, callDuration }):**
1. Short call filter: if `callDuration < 15`, return null
2. Query existing open lead: `supabase.from('leads').select('id, status').eq('tenant_id', tenantId).eq('from_number', fromNumber).in('status', ['new', 'booked']).order('created_at', { ascending: false }).limit(1).maybeSingle()`
3. If existing lead found: insert into `lead_calls` junction table `{ lead_id: existingLead.id, call_id: callId }`, return existing lead
4. If no existing lead (or existing is completed/paid/lost): create new lead:
   - `status`: appointmentId ? 'booked' : 'new'
   - `urgency`: triageResult.urgency || 'routine'
   - Insert lead, then insert into `lead_calls` junction
   - Insert into `activity_log`: `{ tenant_id: tenantId, event_type: 'lead_created', lead_id: newLead.id, metadata: { caller_name: callerName, job_type: jobType, urgency: triageResult.urgency } }`
5. Return created/existing lead

**getLeads({ tenantId, status, urgency, dateFrom, dateTo, search, jobType }):**
1. Base query: `supabase.from('leads').select('id, tenant_id, from_number, caller_name, job_type, service_address, urgency, status, revenue_amount, primary_call_id, appointment_id, created_at, updated_at, calls!lead_calls(id, urgency_classification, urgency_confidence, recording_url, duration_seconds, from_number, start_timestamp)').eq('tenant_id', tenantId)`
2. Conditional filters: `.eq('status', status)` if status, `.eq('urgency', urgency)` if urgency, `.gte('created_at', dateFrom)` if dateFrom, `.lte('created_at', dateTo)` if dateTo, `.eq('job_type', jobType)` if jobType
3. Search: `.or(`caller_name.ilike.%${search}%,from_number.ilike.%${search}%`)` if search
4. Order: `.order('created_at', { ascending: false })`
5. Return data

Import supabase from `@/lib/supabase` (service role — used by webhook handler). Dashboard API routes will use their own server client.

Tests use jest.unstable_mockModule pattern established in the project (see tests/webhooks/ for examples). Mock supabase with chainable query builders.
