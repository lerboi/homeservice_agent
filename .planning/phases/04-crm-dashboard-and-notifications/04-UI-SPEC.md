## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding |
| sm | 8px | Intra-card spacing, compact list item gaps |
| md | 16px | Card padding, form field spacing, nav item padding |
| lg | 24px | Section padding, flyout panel sections |
| xl | 32px | Dashboard column gaps, page content top margin |
| 2xl | 48px | Major section breaks between dashboard regions |
| 3xl | 64px | Page-level top padding on full-bleed views |

Exceptions:
- Sidebar nav links (existing DashboardSidebar): do not change the existing `py-2.5 px-3` spacing on existing nav link elements. This is a legacy value in an already-shipped component — new components must NOT use 10px (non-multiple-of-4) spacing anywhere.
- Touch targets for status change buttons: minimum 44px height (accessibility requirement)
- Filter bar pill buttons: 36px height — compact but above minimum for desktop-primary tool
- Kanban column cards: 12px internal padding — denser than standard card for pipeline overview

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Lead list row detail text, flyout description, table cells |
| Label | 12px | 600 (semibold) | 1.4 | Section sub-headers (uppercase tracking-wider), status badges, filter tags |
| Heading | 20px | 600 (semibold) | 1.2 | Flyout SheetTitle, page section headings, widget titles |
| Display | 28px | 600 (semibold) | 1.15 | Dashboard home stat numbers (today's new leads count, revenue total) |

**Permitted weights: 2 — regular (400) and semibold (600) only.** The size difference between Display (28px) and Heading (20px) provides sufficient visual hierarchy without a third weight.

**Font family:** Inter via `var(--font-inter)` — already loaded in root layout.

**Typography constraints from ui-ux-pro-max skill:**
- No body text below 12px
- Base body at 14px is appropriate for data-dense dashboard (matches AppointmentFlyout pattern)
- Stat display numbers at 28px semibold create visual hierarchy anchors on the home page

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F5F5F4 (warm surface) | Page background, main content area, filter bar background |
| Secondary (30%) | #0F172A (navy) | Sidebar, nav active states, heading text color |
| Accent (10%) | #C2410C (brand orange) | See reserved-for list below |
| Destructive | #DC2626 (red-600) | Mark lead as Lost, delete actions only |
| Success semantic | #166534 (green-800) | Paid status badge, revenue confirmation states |
| Warning semantic | #B45309 (amber-700) | High-ticket urgency badge, conflict indicators |

**Source:** `src/lib/design-tokens.js` — brandOrange, navy, warmSurface values locked.

**Accent (#C2410C) reserved exclusively for:**
1. Primary action buttons (View Lead CTA, Save status change)
2. Active sidebar nav item left border
3. Focus rings on interactive elements
4. "New Lead" badge dot pulse animation
5. Pipeline status chip for "New" status
6. Links in flyout panel (recording link, transcript link)

**Accent is NOT used for:** passive UI chrome, card borders, table row backgrounds, empty state illustrations.

**Pipeline status color mapping (do not use accent for all statuses):**
- New: `bg-[#C2410C]/10 text-[#C2410C]` (orange — accent reserved use #5 above)
- Booked: `bg-blue-50 text-blue-700`
- Completed: `bg-stone-100 text-stone-600`
- Paid: `bg-green-50 text-[#166534]`
- Lost: `bg-red-50 text-red-700`

**Urgency badge color mapping (TRIAGE-06):**
- Emergency: `bg-red-100 text-red-700` — matches existing URGENCY_STYLES in AppointmentFlyout
- Routine: `bg-[#0F172A]/[0.06] text-[#0F172A]/70` — matches existing pattern
- High Ticket: `bg-amber-100 text-amber-700` — matches existing pattern

**Card surface:** `bg-white` with `border border-stone-200/60` and `shadow-[0_1px_3px_0_rgba(0,0,0,0.04)]` — matches `card.base` token in design-tokens.js. Lead row cards add `hover:shadow-[0_4px_12px_0_rgba(0,0,0,0.06)] hover:-translate-y-0.5` on hover.

## Interaction Contracts

### Lead List View (/dashboard/leads)

**Page structure:** Filter bar (full width, sticky) → Lead count + view toggle (list/kanban) → Lead card stack

**Lead card row layout:**
- Left: Urgency color indicator dot (4px wide left border, full card height) + caller name (14px semibold) + phone number (12px muted)
- Center: Job type tag + address (truncated to 1 line) + triage score
- Right: Pipeline status badge + relative timestamp + "View" button (ghost, icon + label)
- Card minimum height: 72px
- On hover: shadow elevation + -0.5px translate (matches design-tokens.js card.hover)

**Realtime updates:** New lead cards animate in from top with `opacity-0 → opacity-100 + translate-y-2 → translate-y-0` over 200ms. Existing rows do not re-animate.

**View toggle:** List/Kanban toggle uses icon buttons (LayoutList / Columns3 from lucide) in a button group. Active state: `bg-[#0F172A] text-white`. Inactive: `bg-white text-stone-500 border border-stone-200`.

**Filter bar:** Horizontal scroll on mobile. Each active filter shows as a removable pill with an X. "Clear all" link appears when any filter is active.

### Kanban View

**Column structure:** 5 columns (New, Booked, Completed, Paid, Lost) in horizontal scroll container on mobile, side-by-side on lg+.

**Column header:** Status label + lead count badge. No color fill on column header — clean white card stack on warm surface background.

**Card in kanban:** Compact — caller name, job type, urgency dot, relative time. Tap/click opens LeadFlyout (same as list view). No drag-and-drop (deferred per CONTEXT.md). Status change via flyout only.

**Column width:** 280px fixed. On screens below lg, horizontal scroll with snap-x scroll-smooth.

### Lead Flyout (LeadFlyout.jsx)

**Trigger:** Any "View" action on a lead card, in either list or kanban view.

**Structure (top to bottom in Sheet):**
1. SheetHeader: Urgency badge + Pipeline status select + Lead age (relative time)
2. Caller section: Name, phone number, call date
3. Job section: Job type, address, triage label + score
4. Audio player: Custom HTML5 player (play/pause, scrub bar, duration)
5. Transcript: Collapsible section, speaker-labeled turns, monospace-adjacent font
6. Pipeline actions: Status change select + RevenueInput (shown when changing to Completed/Paid)
7. SheetFooter: "Mark as Lost" ghost destructive button

**Status change flow:** Select dropdown for status. When selecting Completed or Paid, RevenueInput appears inline below the select. Amount optional for Completed, required for Paid (input shows red border + "Enter amount to save" if empty on submit).

**Audio player design:**
- Background: `bg-stone-50 rounded-xl border border-stone-200/60`
- Play/pause: 36px circular button, `bg-[#0F172A] text-white hover:bg-[#0F172A]/90`
- Scrub bar: Custom range input, track `bg-stone-200`, progress `bg-[#C2410C]`, thumb `bg-[#C2410C]`
- Duration display: 12px monospace-adjacent (`font-variant-numeric: tabular-nums`)

### Dashboard Home (/dashboard)

**Layout:** 2-column stats row (lg+) / single column (mobile) → Recent activity feed

**Stat widget design:** White card (`card.base` token), large number (28px semibold, `text-[#0F172A]`), label below (12px, `text-[#475569]`), delta indicator (small arrow + percentage, green or red semantic color). Minimum 4 stat widgets: New Leads Today, Upcoming Appointments, Calls Today, Conversion Rate.

**Recent activity feed:** Chronological list of events (new lead, status change, booking, notification sent). Each item: icon (16px lucide) + description + relative time. No pagination — shows last 20 events. Skeleton state during load.

## Animation and Interaction States

**Duration budget:** 150ms for micro-interactions (button press, badge hover), 200ms for layout transitions (flyout open, card reveal), 300ms for page-level entrances.

**Flyout open/close:** Sheet component uses Radix animation — do not override. Ensure `data-[state=open]:slide-in-from-right` and `data-[state=closed]:slide-out-to-right`.

**Realtime lead arrival:** New card slides in from top — `@keyframes slide-in-from-top` 200ms ease-out. Trigger only on Supabase Realtime INSERT event, not on initial page load.

**Stat counter animation:** On dashboard home load, stat numbers count up from 0 to value over 600ms using requestAnimationFrame. Skip if prefers-reduced-motion.

**Skeleton loading:** Use `Skeleton` component from shadcn for lead list (3 placeholder rows), stat widgets (4 blocks), and flyout content. Skeleton duration: shimmer 1.5s loop.

**Pipeline status select:** Highlight current status option with orange left border in dropdown (`border-l-2 border-[#C2410C]`).

**Kanban cards:** No drag animation (deferred). Cards are static with hover shadow only.

## Accessibility

- All icon-only buttons (audio play/pause, filter clear X, view toggle) must have `aria-label`.
- Audio player scrub bar must have `aria-valuemin`, `aria-valuemax`, `aria-valuenow`, `aria-label="Seek audio"`.
- Urgency badges must not rely on color alone — always include text label (Emergency, Routine, High Ticket).
- Pipeline status select must be keyboard operable (inherits from Radix Select).
- Flyout Sheet must trap focus when open (inherits from Radix Dialog).
- Transcript: Use `role="log"` on the scrollable container. Speaker labels use `aria-label="Caller:"` / `aria-label="AI:"`.
- Color contrast: `#C2410C` on white = 4.6:1 — passes WCAG AA for normal text at 14px+.
- Dashboard stat numbers (28px semibold) use semantic `<p>` not decorative heading — label below is the accessible name.
- Kanban columns: `role="region"` with `aria-label="[Status] leads"` for each column.
