## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, card content gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical rhythm between billing sections |
| 3xl | 64px | Upgrade page hero top padding |

Exceptions: Ring gauge component uses 120px diameter (not on the 8-point scale) for visual clarity on mobile. Touch targets on banner CTA buttons maintain 44px minimum height.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Weights used: 2 (400 regular, 600 semibold).

Notes:
- Body at 14px matches existing dashboard text-sm pattern (`text-sm text-[#475569]`).
- Label at 12px / 400 for meter captions, invoice table headers, and banner countdown text. Size alone (12px vs 14px) provides sufficient visual differentiation from Body without requiring a third weight.
- Heading at 20px for section headings within the billing page (matches `text-xl font-semibold` in More page).
- Display at 28px for the usage meter center number and upgrade page hero headline.

## UsageRingGauge Specification

Implementation: Pure SVG (no external library). Matches existing SVG animation patterns in globals.css.

### Dimensions
- Outer diameter: 120px
- Stroke width: 10px
- Viewbox: `0 0 120 120`
- Center point: `60, 60`
- Radius: 50px (60 - stroke/2)

### Visual States

**Normal usage (calls_used < calls_limit):**
- Background track: full circle in `#E7E5E4` (stone-200)
- Fill arc: partial circle in `#C2410C` (brand orange), clockwise from 12 o'clock
- Arc percentage: `(calls_used / calls_limit) * 100`
- Center text: `{calls_used}` in 28px semibold `#0F172A`
- Below center: `of {calls_limit}` in 12px regular `#475569`

**Overage usage (calls_used > calls_limit):**
- Background track: full circle in `#E7E5E4`
- Base arc: full circle (100%) in `#C2410C`
- Overage arc: additional segment in `#F59E0B` (amber-500), starting where 100% ends, extending proportionally
- Overage percentage calculation: `((calls_used - calls_limit) / calls_limit) * 100`, capped at 50% additional arc visually
- Center text: `{calls_used}` in 28px semibold `#0F172A`
- Below center: `of {calls_limit}` in 12px regular `#475569`

**Below ring text (all states):**
- Line 1: `{calls_used} of {calls_limit} included calls used` in 14px regular `#475569`
- Line 2 (overage only): `{overage_count} overage calls at ${overage_rate}/call` in 12px regular `#F59E0B`

**Zero usage:**
- Background track only (no fill arc)
- Center text: `0` in 28px semibold `#0F172A`

### Animation
- On mount: arc draws from 0 to target value over 600ms ease-out
- Uses `stroke-dasharray` / `stroke-dashoffset` technique (matches existing `animate-draw-circle` pattern in globals.css)
- Respects `prefers-reduced-motion`: skip animation, show final state immediately

### Accessibility
- `role="img"` on SVG element
- `aria-label="Usage: {calls_used} of {calls_limit} calls used this billing period"`
- Overage state adds: `", plus {overage_count} overage calls"`

## Billing Page Layout (/dashboard/more/billing)

Page structure follows existing More subpage pattern (card.base containers, stone-200/60 borders).

**Focal point:** The UsageRingGauge (120px animated SVG donut, centered within Section 2) is the primary visual anchor of the billing page. Its size, animation on mount, and brand-orange fill arc draw the eye first, establishing at a glance how much of the plan has been consumed.

### Section 1: Plan Card
- Full-width card using `card.base` token
- Left side: Plan name in 20px semibold `#0F172A` + monthly price in 14px `#475569`
- Right side: Badge showing subscription status (`active` = green-100/green-800, `trialing` = blue-100/blue-800, `past_due` = amber-100/amber-800)
- If `cancel_at_period_end` is true: amber inline alert below plan info: "Your subscription will end on {date}. You'll lose access to call answering."

### Section 2: Usage Meter
- Full-width card using `card.base` token
- Heading: "Call Usage" in 20px semibold
- Layout: Ring gauge centered, text below
- Subtext: "Resets on {current_period_end formatted as MMM D, YYYY}"

### Section 3: Billing Details
- Full-width card using `card.base` token
- Heading: "Billing Details" in 20px semibold
- Row 1: "Next renewal" | "{date}" (or "Trial ends" | "{date}" if trialing)
- Row 2: "Payment method" | "Visa ending 4242" (from Stripe Customer Portal — show "Manage" link if available)
- Bottom: "Manage Subscription" button using `btn.primary` token, full-width on mobile, auto-width on desktop. Links to `/api/billing/portal`.

### Section 4: Recent Invoices
- Full-width card using `card.base` token
- Heading: "Recent Invoices" in 20px semibold
- Table with 3 columns: Date | Amount | Status
- Show 5 most recent invoices (Claude's discretion: 5 is optimal for scan-ability without scroll)
- Status badges: `paid` = green variant, `open` = blue variant, `uncollectible` = red variant
- Footer link: "View all invoices" links to `/api/billing/portal`
- Empty state if no invoices: "No invoices yet. Your first invoice will appear after your trial ends."

### Loading State
- All four sections render as Skeleton components (shadcn) matching their final dimensions
- Plan card: 1 skeleton line (200px wide) + 1 badge skeleton
- Usage meter: circle skeleton (120px) + 2 text line skeletons
- Billing details: 3 row skeletons
- Invoice table: 3 row skeletons with 3 columns each

### Error State
- If subscription data fails to load: full-page centered error card with AlertCircle icon, "Unable to load billing information" heading, "Please refresh the page or contact support if the problem persists." body, and a "Refresh Page" button that calls `router.refresh()`.

## Trial Countdown Banner Specification

Follows BillingWarningBanner pattern: sticky, top-0, z-39, 44px height, full-width, not dismissible.

### Visual States

**Trial with >3 days remaining (info state):**
- Background: `bg-blue-50`
- Border: `border-b border-blue-200`
- Icon: `Clock` from lucide at 16px in `text-blue-800`
- Text: `text-sm text-blue-800`: "{X} days left in your free trial"
- CTA: "Upgrade now" as text link, `text-sm font-medium text-blue-800 underline hover:text-blue-900`, links to `/dashboard/more/billing`

**Trial with <=3 days remaining (urgent state):**
- Background: `bg-amber-50`
- Border: `border-b border-amber-300`
- Icon: `AlertCircle` from lucide at 16px in `text-amber-800`
- Text: `text-sm text-amber-800`: "Your trial ends in {X} days - upgrade to keep your AI receptionist"
- CTA: "Upgrade now" as bordered button (matches BillingWarningBanner style), `border border-amber-400 text-amber-800 px-3 py-1 rounded-md hover:bg-amber-100`

**Trial last day:**
- Same as urgent state but text: "Your trial ends today - upgrade to keep your AI receptionist"

### Layout
- Flexbox row: icon + text on left, CTA on right (identical to BillingWarningBanner)
- Responsive: on mobile (<640px), text truncates with ellipsis, CTA stays visible
- `px-4 lg:px-8` horizontal padding (matches BillingWarningBanner)

### Data Source
- Reads `trial_ends_at` from subscriptions table where `status = 'trialing'`
- Calculates days remaining: `Math.ceil((trial_ends_at - now) / 86400000)`
- Shows only when status is `trialing` and trial has not ended

### Stacking with BillingWarningBanner
- TrialCountdownBanner and BillingWarningBanner are mutually exclusive (trialing vs past_due are different statuses)
- Both use `z-39` (below ImpersonationBanner at z-40)
- No simultaneous display needed

## Upgrade Page (/billing/upgrade)

### Layout
- Full page (not within dashboard layout -- no sidebar, no bottom tabs)
- White background
- Centered content, max-width 1024px
- Top padding: 64px (3xl)

### Header Section
- Heading: "Pick a plan to reactivate your AI receptionist" in 28px semibold `#0F172A`, centered
- Subheading: "Your subscription has ended. Choose a plan below to resume 24/7 call answering for your business." in 14px regular `#475569`, centered, max-width 480px
- Bottom margin: 48px (2xl) before cards

### Plan Cards
- 3-column grid (md breakpoint), 1-column on mobile
- Reuse PRICING_TIERS data from `pricingData.js` (Starter, Growth, Scale)
- Visual style matches PlanSelectionCards exactly (same border, hover, badge treatment)
- CTA text changes from "Start Free Trial" to "Choose {Plan Name}" (e.g., "Choose Growth")
- Growth tier keeps `ring-2 ring-[#C2410C]/40` highlight and "Most Popular" badge
- Each CTA creates a new Stripe Checkout Session via POST to `/api/billing/checkout-session` (new route, similar to onboarding checkout but without onboarding context)
- Loading state: Loader2 spinner + "Redirecting..." (matches PlanSelectionCards)

### Previous Plan Badge (Claude's Discretion: implement)
- If the tenant's last `plan_id` matches a tier, show a subtle badge: "Your previous plan" in `text-xs text-[#475569] bg-stone-100 px-2 py-0.5 rounded-full` positioned below the tier name
- Implementation is simple (single string comparison), adds helpful context for returning users

### Error Handling
- Same inline error pattern as PlanSelectionCards: red text below cards with support link
- "Something went wrong. Please try again, or [contact support](/contact?type=support)."

### Footer
- Enterprise note: "Need more? [Contact us for Enterprise.](/contact?type=sales)" in 14px `#475569`, centered
- "Return to sign in" link below for users who want to sign in with a different account: `text-sm text-[#475569] underline`, links to `/auth/signin`

## Responsive Behavior

- **Desktop (>=1024px):** 4 sections stack vertically, max-width 768px centered within dashboard content area

### Trial Countdown Banner
- **Desktop:** Full text + CTA visible in single row
- **Mobile:** Text truncates with ellipsis if needed, CTA remains visible and tappable (min 44px touch target)

### Upgrade Page
- **Desktop (>=768px):** 3-column card grid
- **Mobile:** Single-column stack with 16px padding

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Ring gauge | `role="img"` with descriptive `aria-label` |
| Trial banner | `role="alert"` for screen reader announcement |
| Invoice status | Badge text is the accessible name (no icon-only status) |
| Portal link | Button with clear label "Manage Subscription" |
| Upgrade cards | `role="article"` with `aria-label` including plan name and price |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 minimum). Blue-800 on blue-50 = 7.1:1. Amber-800 on amber-50 = 6.2:1 |
| Reduced motion | Ring gauge animation skipped when `prefers-reduced-motion: reduce` |
| Keyboard | All interactive elements reachable via Tab. Plan card CTAs are `<button>` elements. Banner CTA is `<a>` or `<button>`. |
