3. **Wizard Steps** deep dive — For each step page:
   - Step 1 (page.js): signup/OTP flow, useState toggle for OTP phase (NOT router.push), shouldCreateUser:false on signInWithOtp
   - Step 2 (profile/page.js): trade selection via TradeSelector + business_name, dual sequential POST to /api/onboarding/start (first creates tenant, second saves trade+services)
   - Step 3 (services/page.js): service list from TRADE_TEMPLATES, editable
   - Step 4 (verify/page.js): phone number + OTP verify
   - Step 5 (contact/page.js): contact info collection
   - Step 6 (test-call/page.js): TestCallPanel + clearWizardSession on completion
   - Step 7 (complete/page.js): redirect to /dashboard

4. **Onboarding Components** deep dive — TestCallPanel (polling from both 'calling' and 'in_progress' states), CelebrationOverlay (prefers-reduced-motion skips radial pulse divs entirely), TradeSelector (trade picker UI), OtpInput (digit boxes, focus ring: focus:border-[#F97316] + focus:ring-[#F97316]/30 without ring-offset).

5. **Session Management** — useWizardSession hook: `gsd_onboarding_` prefix for sessionStorage key isolation, `clearWizardSession()` bulk-removes all gsd_onboarding_* keys.

6. **API Routes** deep dive — For each:
   - /api/onboarding/start: create/upsert tenant, tone_preset
   - /api/onboarding/provision-number: Retell phone number provisioning
   - /api/onboarding/sms-confirm: save owner_phone + owner_email in one round-trip
   - /api/onboarding/sms-verify: OTP verification
   - /api/onboarding/test-call: trigger Retell test call (sets retell_llm_dynamic_variables with onboarding_complete:true for AI behavior)
   - /api/onboarding/test-call-status: poll for provisioned phone number
   - /api/onboarding/complete: set onboarding_complete flag in DB

7. **Trade Templates** — trade-templates.js: TRADE_TEMPLATES map structure (trade -> services[] with urgency_tag), how it pre-populates service list.

8. **Middleware Auth Guards** — middleware.js: AUTH_REQUIRED_PATHS (includes wizard sub-paths, NOT /onboarding root), onboarding_complete redirect logic (only checked on /onboarding paths, NOT /dashboard), OAuth callback default to /onboarding/profile.

9. **Database Tables** — tenants (onboarding_complete, tone_preset, owner_phone, owner_email columns relevant to onboarding), services (populated from trade templates).

10. **Environment Variables** — RETELL_API_KEY (phone provisioning + test call), NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, TWILIO_ACCOUNT_SID/AUTH_TOKEN/FROM_NUMBER (SMS OTP).

11. **Key Design Decisions** — Mine from STATE.md Phase 07 decisions. Must include:
    - shouldCreateUser:false on signInWithOtp (prevents duplicate user)
    - OTP uses useState toggle (avoids layout re-mount and progress bar flicker)
    - Two sequential POSTs to /api/onboarding/start (tenant must exist before trade+services)
    - onboarding_complete set on webhook callback, NOT at test call trigger time
    - retell_llm_dynamic_variables keeps onboarding_complete:true during test call (separate concern from DB flag)
    - useWizardSession gsd_onboarding_ prefix for key isolation
    - OAuth callback default is /onboarding/profile (skip Step 1 for Google OAuth)
    - TestCallPanel polls from both states (catches fast-completing calls)
    - AUTH_REQUIRED_PATHS excludes /onboarding root
    - Middleware checks onboarding_complete on /onboarding paths only
    - CelebrationOverlay skips radial pulse divs entirely (not just animation class) for prefers-reduced-motion
