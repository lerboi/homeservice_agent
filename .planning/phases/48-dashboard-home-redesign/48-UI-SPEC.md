# Phase 48 — UI Design Contract

> Visual and interaction contract for the dashboard-home redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Light mode only. Dark mode token migration is owned by Phase 49 — do NOT introduce `dark:` variants, `useTheme()` wiring, or CSS-variable swaps in this phase. Use existing warm-neutral surface + copper accent tokens from `src/lib/design-tokens.js` as-is.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized — `components.json` present) |
| Preset | new-york style, `baseColor: neutral`, `cssVariables: true`, `rsc: true`, `tsx: false` (JS project) |
| Component library | Radix primitives via shadcn (Card, Button, Badge, Accordion, Progress, Sheet, Skeleton, Dialog, Alert, Tooltip, Popover, Textarea, Input) |
| Icon library | lucide-react ^0.577.0 |
| Font | Inter (`next/font/google`, loaded in `src/app/layout.js` as `--font-inter`) |
| Shared tokens | `src/lib/design-tokens.js` — `colors`, `card`, `glass`, `gridTexture`, `heading`, `body`, `focus`, `selected`, `btn` |

**Composition rule:** Every new card on the home page MUST compose from `card.base` + `card.hover` (from `design-tokens.js`). Do not declare raw `bg-white rounded-2xl` classes inline. Text colors use the exported `heading` / `body` tokens, not raw hex.

**Icon sizing convention:** `h-4 w-4` for inline/button icons, `h-5 w-5` for card header icons, `h-6 w-6` for hero-tile header icons. Stroke width 1.5 (lucide default) — do not override.

## Spacing Scale

Declared values (8-point scale, multiples of 4 only — matches Tailwind defaults):

| Token | Value | Tailwind | Usage in Phase 48 |
|-------|-------|----------|-------------------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps inside badges, inline chip padding |
| sm | 8px | `gap-2`, `p-2` | Compact inline spacing, badge padding, chip gaps |
| md | 16px | `gap-4`, `p-4` | Default card interior padding (mobile), item row gap in checklist |
| lg | 24px | `gap-6`, `p-6` | Card interior padding (desktop), bento tile gap, section padding |
| xl | 32px | `gap-8`, `p-8` | Hero tile interior on lg+, vertical space between bento hub and RecentActivityFeed |
| 2xl | 48px | `gap-12` | Vertical rhythm between major home sections (hub → help → activity) |
| 3xl | 64px | `py-16` | Not used on the home page (reserved for landing-class rhythm only) |

**Bento grid gaps:** `gap-4` (16px) on mobile, `gap-6` (24px) on `md+`.

**Checklist accordion:** `p-4` section padding on mobile, `p-6` on `md+`; `gap-3` (12px — see exception below) between item rows inside a theme group.

**Mobile stack (D-16):** Vertical gap between stacked cards is `gap-4` (16px) at 375px, scaling to `gap-6` (24px) at `md+`.

**Exceptions:**
- `gap-3` (12px) is allowed for dense list-row spacing inside the setup checklist accordion (matches current SetupChecklist row rhythm — preserving in-place refactor per D-01).
- Mobile touch targets: interactive elements (jump-to-page, dismiss, mark-done buttons) must be ≥ 44px tall (`min-h-[44px]`) even when visual padding would be smaller. Not a scale violation — accessibility floor.

## Typography

Four sizes, two weights. Inter only.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Label | 12px | `text-xs` | 500 (`font-medium`) | 1.4 (`leading-[1.4]`) | Required/Recommended badges, meta rows ("3 missed • not attempted"), cycle-days-left caption, quick-link tile eyebrow |
| Body | 14px | `text-sm` | 400 (`font-normal`) | 1.5 (`leading-normal`) | Card body copy, checklist item descriptions, chat message text, empty-state body |
| Heading | 16px | `text-base` | 600 (`font-semibold`) | 1.4 (`leading-[1.4]`) | Card titles ("Today's Appointments", "Calls", "Hot/New Leads", "Usage", "Help & Discoverability"), checklist theme-group labels |
| Display | 24px | `text-2xl` | 600 (`font-semibold`) | 1.2 (`leading-tight`) | Page greeting ("Good morning, {name}"), usage-meter fraction `{callsUsed} / {callsIncluded}`, hero-tile next-appointment time |

**Weights allowed: 400 (normal) and 600 (semibold) only.** No 500 (medium) except for `text-xs` labels/badges where medium is the Inter sweet spot for legibility at small sizes. No 700 (bold). No italics.

**Heading color:** `text-[#0F172A]` (via `heading` token). **Body color:** `text-[#475569]` (via `body` token). **Muted/meta color:** `text-stone-500`.

**Tabular numbers:** Usage fraction, appointment times, and call counts must use `tabular-nums` (`font-variant-numeric: tabular-nums`) so column alignment holds.

## Color

Follows the existing Voco dashboard palette. No new brand colors introduced in this phase.

| Role | Value | Tailwind reference | Usage |
|------|-------|--------------------|-------|
| Dominant (60%) | `#F5F5F4` (warm neutral / stone-100) | `bg-[#F5F5F4]` (existing dashboard page background) | Home page background, checklist accordion idle rows |
| Secondary (30%) | `#FFFFFF` (white) | `bg-white` via `card.base` | All card surfaces — bento tiles, checklist container, chat panel, help tiles, activity feed |
| Accent (10%) | `#C2410C` (copper / orange-700) | via `btn.primary`, `focus.ring`, `selected.card` | See reserved-for list below |
| Destructive | `#B91C1C` (red-700) | `text-red-700` / `bg-red-50` for soft alerts | Usage meter fill when > 100% of cycle cap; missed-calls row indicator dot only when attention is required |

**Tertiary surface tones in use (not new colors):**
- Navy `#0F172A` — sidebar background only, already applied by `DashboardSidebar`. Phase 48 MUST NOT recolor the sidebar.
- Amber `#D97706` (amber-600) — usage bar fill at 75–100% threshold (D-13).
- Stone-200/60 borders — card borders via `card.base`.

**Accent (copper `#C2410C`) reserved for:**
1. Primary CTA buttons on each card ("View all appointments", "Jump to page" in checklist when it is the primary row action, "Ask Voco" chat send button). Use `btn.primary` token.
2. Focus rings on all interactive elements (`focus.ring` token).
3. Selected/active states on checklist theme-group accordions when expanded (`selected.card` token).
4. Progress ring fill in the setup-checklist header (existing conic-gradient — preserved per D-01).
5. Usage meter bar fill when below 75% threshold (neutral copper — signals "healthy usage, on brand").
6. Chat-panel send-button icon and the sender-bubble border accent.

**Accent NOT allowed on:**
- Card borders (use stone-200/60).
- Card titles or body text.
- Icons inside cards (icons inherit `text-stone-600` unless they are badge-status icons).
- Quick-link tile backgrounds — tiles are white with stone-200 hover border lift; only the tile's arrow icon on hover may tint to copper.
- More than ONE button per card (10% budget).

**Semantic colors for usage-meter thresholds (D-13):**

| Threshold | Bar fill | Caption tone | Rationale |
|-----------|----------|--------------|-----------|
| 0–74% | `bg-[#C2410C]` (copper/accent) | `text-stone-600` | Healthy; brand accent doubles as "on track" |
| 75–99% | `bg-amber-600` | `text-amber-700` | Approaching cap; warm attention without alarm |
| ≥100% | `bg-red-700` | `text-red-700` | Over cap; shows overage dollars emphatically |

**Required/Recommended badges in setup checklist (D-02):**
- `Required` → `bg-[#C2410C]/10 text-[#C2410C] border border-[#C2410C]/20` (accent soft)
- `Recommended` → `bg-stone-100 text-stone-600 border border-stone-200` (neutral)

Do NOT use red/green semaphore for Required/Recommended — those are reserved for destructive/success states only.

## Copywriting Contract

All copy is in American English, sentence case, no emoji. Owner-language framing ("where do I…", "you") over feature-language ("manage settings").

### Primary CTAs (per card)

| Surface | CTA label | Rationale |
|---------|-----------|-----------|
| Today's Appointments hero tile | `View full schedule` | Verb + noun; routes to `/dashboard/appointments` |
| Calls card | `View all calls` | Routes to `/dashboard/calls` |
| Hot/New Leads card | `Open Jobs` | Matches Phase 52 rename from "Leads" to "Jobs"; if Phase 52 ships after 48, fallback label is `View all leads` — planner resolves at implementation based on Phase 52 status |
| Usage card | `Manage plan` | Routes to `/dashboard/billing` |
| Help & Discoverability | (no CTA — tiles ARE the CTAs) | Each tile label IS the action: `Add a service`, `Change AI voice`, `Invite teammate`, etc. |
| Setup Checklist row (primary action) | `Finish setup` → when row not started; `Continue` → when row partially done; `Open settings` → for recommended-only rows | Context-sensitive verb |
| Chat panel | `Ask Voco` (send button) + placeholder `Ask anything about your dashboard…` | Matches existing ChatbotSheet tone |

### Empty States

| Card | Heading | Body |
|------|---------|------|
| Today's Appointments (no appts today) | `Nothing booked today.` | `When Voco books an appointment, it will show up here. You'll also get a notification.` |
| Calls — last 24h (no calls) | `No calls in the last 24 hours.` | `Voco is listening. The moment someone calls, you'll see it here.` |
| Hot/New Leads (no hot/new) | `No new leads right now.` | `New inquiries from calls and forms land here first. Check back after your next call.` |
| Usage (fresh cycle, 0 calls) | `0 of {callsIncluded} calls used` (use the meter itself as the content) | `Your cycle started {date}. {daysLeft} days left.` |
| Setup Checklist (all complete) | `You're all set.` | `Every setup item is done. Come back here if you change plans or reconnect a calendar.` — show `SetupCompleteBar` celebration modal once, then collapsed summary |
| Help & Discoverability | (never empty — tiles are static) | — |
| RecentActivityFeed (no activity) | `No recent activity.` | `Calls, bookings, and lead updates will appear here as they happen.` |
| Chat panel (fresh, no messages) | `Ask Voco anything.` | `Try: "How do I change my AI voice?" or "Where do I find invoices?"` — show as two clickable starter prompts below the heading |

### Error States

| Surface | Copy | Recovery path |
|---------|------|---------------|
| Any card data fetch fails | `Couldn't load {card name}.` / body: `Check your connection and try again.` + `Retry` button | Button refires the fetch; on second failure, append `If this keeps happening, reach out from the Help menu.` |
| Setup Checklist API 500 | `Checklist is temporarily unavailable.` / body: `Your setup progress is safe — this is just a display hiccup. Refresh the page to try again.` | Soft inline alert at top of checklist; does NOT hide existing state |
| Usage API 500 | `Usage data is temporarily unavailable.` / body: `Your plan is still active. Refresh to retry.` | Inline alert inside the Usage card; keep card mounted, do not collapse |
| Chat send fails | `Message didn't send.` / body: `Check your connection and try again.` + `Retry` on the failed bubble | Preserve the user's typed input; failed bubble is re-sendable in place |
| Checklist mark-done save fails | `Couldn't save that change.` / body: `Try again, or the checklist will refresh next time you open the dashboard.` | Revert the optimistic UI change; toast the error |

### Destructive Actions

Phase 48 introduces ONE class of destructive-adjacent action and ZERO fully destructive actions.

| Action | Destructive? | Confirmation |
|--------|--------------|--------------|
| Dismiss a setup checklist item (D-03) | **Soft-destructive** (reversible via "Show dismissed" toggle in checklist footer) | **No modal.** Inline confirmation toast: `Dismissed. Undo` with an `Undo` link that triggers a reverse `PATCH`. Matches current dismissal UX, preserved by D-01. |
| Mark done (manual override) | Not destructive | Optimistic UI update; silent success |
| Jump to page | Not destructive | Plain navigation |
| Reset dashboard tour | Not surfaced in Phase 48 | — |

No `<AlertDialog>` confirmations required for Phase 48. Existing `SetupCompleteBar` celebration modal is preserved verbatim per D-01 (not a destructive dialog).

### Microcopy Notes

- Time formatting: `2:30 PM` (12-hour, space before meridiem, capital letters). Relative for <24h (`23 min ago`, `2 hr ago`), absolute date for older (`Apr 12`).
- Numbers: comma-separated thousands (`1,247`). Currency: `$0.15` / `$12.40` (two decimals when dollars, no decimals under $1 where appropriate is still two — be consistent: always two decimals for money).
- Usage overage line appears only when `overageDollars > 0`. Format: `${overageDollars} over` (e.g. `$4.20 over`).
- Calls flagged row prefix: `Missed` badge + `{count} • not attempted` meta. No "urgent" or "critical" language — flat factual tone.
- Help tile labels: always `verb + noun` in sentence case (`Add a service`, `Invite teammate`, `Connect calendar`, `View invoices`, `Set escalation contacts`, `Change AI voice`). Planner picks 3–4 per D-15.

## Interaction States

Every interactive surface declares all five states:

| State | Contract |
|-------|----------|
| Idle | Default token values (`card.base`) |
| Hover (`md+` only, pointer devices) | `card.hover` (shadow lift + `-translate-y-0.5`); transitions 200ms ease |
| Focus | `focus.ring` token (copper 2px ring, 1px offset) on every interactive element |
| Active/Pressed | `active:scale-95` on buttons (matches `btn.primary`) |
| Disabled | `opacity-60 cursor-not-allowed pointer-events-none` |
| Loading | shadcn `Skeleton` for card data; spinner (lucide `Loader2` with `animate-spin`) for in-flight button actions |

**Motion budget:**
- Card entrance: `fade-in + translate-y-2 → 0` over 200ms, staggered 50ms between bento tiles. Use existing utility pattern — no new animation libraries.
- Progress ring updates (setup checklist): animate percentage transition over 400ms ease-out.
- Usage bar fill: animate width change over 400ms ease-out when threshold crosses.
- Reduced-motion: honor `prefers-reduced-motion: reduce` — disable all non-essential animations; instant state changes only.

**Checklist accordion behavior:**
- One theme group expanded at a time by default (collapsible-accordion pattern), OR all groups collapsible independently — planner's call, but must match shadcn `Accordion` defaults for consistency.
- Completed theme groups show a check glyph (`lucide CheckCircle2`, `text-stone-500`, not accent) next to the theme label.
- Dismissed items are hidden by default; a footer link `Show N dismissed` toggles visibility.

## Accessibility Contract

- All interactive elements reachable via keyboard in logical DOM order.
- Focus rings visible on all focusable elements (use `focus.ring` token).
- Setup checklist accordion uses shadcn `Accordion` primitives (Radix) — ARIA handled.
- Chat panel: message log is `role="log" aria-live="polite"`; input has visible label or `aria-label`.
- Usage meter: bar is `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label="Calls used this cycle"`.
- Color contrast: all text meets WCAG AA (4.5:1 for body, 3:1 for large text). Amber-600 on white clears 4.5:1; copper `#C2410C` on white clears 4.5:1. Red-700 on white clears 4.5:1. Stone-500 on white clears 4.5:1 for 14px+.
- Touch targets ≥ 44×44px on mobile (already declared in Spacing exceptions).
- Icons inside buttons must pair with visible text OR carry `aria-label`. Icon-only buttons (e.g., dismiss `×`) get `aria-label="Dismiss {item name}"`.
