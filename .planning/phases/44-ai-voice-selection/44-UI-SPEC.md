| Token | Value | Usage |

## Component Inventory

### VoicePickerSection (new)

A settings subsection rendered inside `SettingsAISection.jsx`, above the phone number display block.

**Structure:**
```
VoicePickerSection
  ├── Gender group label: "Female"
  ├── 2-column grid row (Aoede, Erinome, Sulafat)
  ├── Spacer (24px)
  ├── Gender group label: "Male"
  └── 2-column grid row (Zephyr, Achird, Charon)
```

**Grid spec:**
- Desktop: `grid grid-cols-2 gap-4` (2 columns × 3 rows = 6 cards)
- Mobile (< 640px): `grid grid-cols-1 gap-3` (single column)
- Source: CONTEXT.md D-01, D-05

### VoiceCard (new)

Renders a single selectable voice option.

**States:**

| State | Visual |
|-------|--------|
| Idle (unselected) | `card.base` + `selected.cardIdle` tokens — white card, stone border |
| Selected | `card.base` + `selected.card` tokens — copper border + #C2410C/4% tint |
| Hover (unselected) | `card.hover` token — subtle shadow lift, -translate-y-0.5 |
| Hover (selected) | No additional lift — border already signals selection |

**Card contents (top to bottom, left to right):**
- Voice name: 14px semibold, `#0F172A`
- Character description: 12px regular, `#475569`, 1-2 lines max
- Play/pause icon button: Lucide `Play` / `Pause` icon, 16px, positioned at top-right corner of card
  - Button container: 44×44px touch target, `rounded-full`, `hover:bg-stone-100`
  - Playing state: Lucide `Pause` icon, accent color `#C2410C`
  - Idle state: Lucide `Play` icon, stone-500
  - One card playing at a time — tapping play on a second card stops the first

**Click target:** Entire card surface selects the voice (not just the text area). Play button click is isolated (stopPropagation).

Source: CONTEXT.md D-04, D-07, D-08

### SaveButton (existing pattern — `btn.primary`)

- Label: "Save Voice" (see Copywriting Contract)
- Placement: Below the voice grid, right-aligned
- Width: auto (not full-width)
- States:
  - Default: `btn.primary` token styles
  - Loading/saving: disabled + spinner icon (Lucide `Loader2`, `animate-spin`, 16px) replaces label — button remains same width
  - Disabled (no change from loaded state): same as default — Save is always clickable once page is loaded, even if selection matches current saved value (graceful no-op on same-value save)

Source: CONTEXT.md D-11, Claude's Discretion (save placement, loading state)

### Toast (sonner — existing)

- Success: "Voice updated" — duration 3000ms
- No error-specific toast copy beyond the generic error state (see Copywriting Contract)

Source: CONTEXT.md D-12, Claude's Discretion (toast duration)

## Interaction Map

| Trigger | Response |
|---------|----------|
| Click voice card | Card border switches to copper (`selected.card`); previously selected card reverts to idle |
| Click play button on idle card | Audio begins; Lucide icon switches to Pause; play button accent-colored |
| Click play button on playing card | Audio pauses; icon reverts to Play |
| Click play on a different card while another plays | First card pauses; second card begins playing |
| Click Save Voice | Button enters loading state (spinner); PATCH /api/ai-voice-settings; on success: sonner toast "Voice updated"; on error: toast with error copy |
| Page load | Fetch current `ai_voice` from tenant; pre-select matching card (or no card if NULL) |

## Accessibility Contract

- All voice cards: `role="radio"`, `aria-checked={isSelected}`, keyboard-navigable with arrow keys within `role="radiogroup"`
- Play buttons: `aria-label="Play {voice name} preview"` / `aria-label="Pause {voice name} preview"`
- Gender group labels: `aria-label` on the grid container (`aria-label="Female voices"` / `aria-label="Male voices"`)
- Save button: `aria-busy={isSaving}` when loading, `aria-label="Save voice selection"`
- Touch targets: 44×44px minimum on play buttons
