## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, border offsets |
| sm | 8px | Compact element spacing, icon-to-label gap |
| md | 16px | Default element spacing, form field gap |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout gaps, section vertical padding |
| 2xl | 48px | Major section breaks, hero padding |
| 3xl | 64px | Page-level spacing, full-bleed section padding |

Exceptions:
- Nav height: 64px (4rem, existing — matches `h-16` in LandingNav.jsx)
- Mobile touch targets: 44px minimum (existing pattern — `min-h-[44px] min-w-[44px]` for all interactive controls)
- Auth form card internal padding: lg (24px) at sm breakpoint, xl (32px) at md, 2xl (48px) at lg — progressive padding

**Source:** Existing component patterns in LandingNav.jsx, auth/signin/page.js.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 (regular) | 1.5 |
| Label / Nav / Caption | 14px (text-sm) | 500 (medium) | 1.4 |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 |
| Display | 36–48px (text-4xl–text-5xl, responsive) | 600 (semibold) | 1.15 |

Notes:
- Tracking: `tracking-tight` on all headings and display text — established in existing components.
- Subheadings/eyebrows: 11–12px, weight 600, `uppercase tracking-[0.15em]` — existing pattern (e.g., "AI-powered answering" in auth right panel).
- Link text in nav and footer: 14px, weight 400/500 at rest.
- All font sizes use existing Inter variable — no new font loading.

**Source:** Observed from LandingNav.jsx, auth/signin/page.js, LandingFooter.jsx existing patterns.

## Component-Level Design Contracts

### Navigation (LandingNav.jsx)

- **At page top:** `bg-transparent` — content bleeds through.
- **On scroll (>20px):** `bg-[#0F172A]/95 backdrop-blur-[12px]` + `border-b border-white/[0.06]`. Transition: 500ms ease.
- **Active page link:** Copper underline — `border-b-2 border-[#C2410C]` relative to nav link.
- **Mobile drawer:** `bg-[#1E293B]` (charcoal), `border-l border-white/[0.06]`. CTA button: full-width copper, 44px min-height.
- **Note:** backdrop-blur is permitted only on nav (small surface). All other surfaces: no backdrop-blur. Source: D-08, D-23.

### Footer (LandingFooter.jsx)

- **Top border replacement:** Remove `border-t border-white/[0.06]`. Replace with copper gradient line: `background: linear-gradient(90deg, transparent 0%, #C2410C 50%, transparent 100%)`, height 1px.
- **Link hover:** `text-white/50` at rest → `text-[#C2410C]` on hover, `transition-colors 200ms`.
- **Logo size:** Increase from `size-7` to `size-8` (32px). Tagline remains.
- **Vertical spacing:** Increase `space-y-3` list spacing to `space-y-4`. Source: D-10, D-11, D-12.

### Cards (all pages)

- **Rest state:** `bg-[#1E293B]`, `border border-white/[0.06]`, no shadow, no transform.
- **Hover state:** `border-[#C2410C]/40`, `shadow-[0_0_20px_rgba(194,65,12,0.15)]`, `transform: translateY(-2px)`. Transition: border 300ms ease, shadow 300ms ease, transform 200ms ease.
- **Stone section cards:** `bg-white`, `border border-stone-200/60` — standard light card when inside stone break section.
- Source: D-05, D-06.

## Animation Contract

All animations must use transform/opacity only — no layout-triggering properties.

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Scroll reveal (AnimatedSection) | 200ms | ease-out | IntersectionObserver enter |
| AnimatedStagger child delay | 50ms stagger offset | ease-out | On parent enter |
| Card hover lift | 200ms | ease | Mouse enter |
| Card hover copper glow | 300ms | ease | Mouse enter |
| Nav background transition | 500ms | ease | scrollY > 20px |
| Link underline scaleX | 200ms | ease | Mouse enter |
| Footer link color | 200ms | ease | Mouse enter |
| Accordion expand/collapse | 200ms | ease-out | Click (existing keyframes) |
| Auth mode toggle | No animation — instant layout swap | — | Click |

- AnimatedSection direction assignment (Claude's discretion): Use `direction="up"` for all stacked content sections. Use `direction="left"` / `direction="right"` for side-by-side feature bento cards only.
- prefers-reduced-motion: AnimatedSection skips animation with `initial={false}` (Framer Motion v12 pattern, established Phase 02.1).
- No page transitions (Next.js app router default). Source: D-22.
- No backdrop-blur on large surfaces. Source: D-23.

**Source:** D-19, D-20, D-21, D-22, D-23.

## Mobile Fallback Contract

All swaps at `md` breakpoint (768px). Source: D-29.

| Desktop Component | Mobile Replacement |
|-------------------|--------------------|
| Spline 3D (SplineScene) | Gradient + Lucide icon (already implemented Phase 11 D-07) |
| Bento grid 6-col (FeaturesGrid) | Single-column stacked full-width cards (`grid-cols-1`) |
| HowItWorksTabs | Vertical accordion or simple stacked numbered steps (no JS tabs) |
| AnimatedSection | Fade-only (`direction` prop → remove directional offset) or skip entirely |
| Comparison table (ComparisonTable.jsx) | Card-per-tier (already implemented Phase 06) |
| Auth split layout | Form panel only, full-width single column |

Implementation pattern: `hidden md:block` / `block md:hidden` for component-level swaps. No JS-based media queries.
