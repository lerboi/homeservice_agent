<objective>
Stream B defense-in-depth. Pin Gemini Live's input language AND tighten the prompt-layer language directive — two orthogonal surfaces per D-B-03 RE-OPENED (2026-04-23).

Rationale (per RESEARCH §"Stream B Native-Audio Blocker"): `gemini-3.1-flash-live-preview` is a native-audio model; Google's official docs state `language_code` is explicitly NOT supported and models auto-detect the input language. A LiveKit community thread (410) reports identical symptoms (non-English tokens during silence) on the same native-audio model class and classifies it as model-side, unsolvable at framework layer. The only documented community workaround is a system-prompt directive. Ship BOTH surfaces:

1. **SDK layer (D-B-01):** pass `language="en-US"` / `"es-US"` on `RealtimeModel(...)` — defense-in-depth; the kwarg is syntactically accepted, cast to `LanguageCode` enum, and forwarded to `SpeechConfig(language_code=...)`. May be ignored by native-audio model but costs nothing.
2. **Prompt layer (D-B-03 re-opened):** add an anti-hallucination directive to `_build_language_section` — "Treat any unintelligible audio as English unless the caller explicitly speaks another supported language" — strengthening the existing default-to-locale instruction. Must maintain locale parity (60.3 Plan 12 closure).

Output: shipped code on livekit-agent/main; unit tests verifying locale → BCP-47 map + prompt directive presence; zero existing-test regressions.
