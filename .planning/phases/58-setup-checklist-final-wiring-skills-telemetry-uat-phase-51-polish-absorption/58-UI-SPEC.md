### 4.1 `<EmptyState />` — POLISH-01 generic primitive

**File:** `src/components/ui/empty-state.jsx` (NEW)

**Prop API (locked):**

```jsx
<EmptyState
  icon={Users}                  // required — lucide-react component reference
  headline="No jobs yet"        // required — string, sentence case
  description="..."             // optional — string, max ~160 chars
  ctaLabel="Make a test call"   // optional — if present, renders CTA
  ctaHref="/dashboard/..."      // optional — mutually exclusive with ctaOnClick
  ctaOnClick={() => void}       // optional — mutually exclusive with ctaHref

```

**Rules:**
- `ctaLabel` requires exactly one of `ctaHref` | `ctaOnClick`. Planner enforces via runtime warning in dev.
- If no `ctaLabel`, no CTA rendered.
- Icon is **always** rendered (no icon-less variant).

**Layout & visual contract:**

| Element | Class tokens | Notes |
|---------|--------------|-------|
| Outer container | `flex flex-col items-center justify-center py-16 text-center` | 64px vertical padding — matches `EmptyStateLeads` exactly |

| Headline | `text-base font-semibold text-foreground mb-2` | 16px / 600, `--foreground` (works both modes) |

| CTA | shadcn `<Button>` default variant (copper bg via `btn.primary`) `asChild` when `ctaHref` | 44px min-height for touch target |

**Dark-mode parity:** All colors flow through `--foreground` / `--muted-foreground` / `--brand-accent` CSS variables — already declared for both modes in `globals.css`. No mode-specific classes.


**Rules:**
- `onRetry` is optional; the primitive works as read-only error display without it.
- `message` is optional; when absent, use the Copy Contract default (see §10).

**Layout & visual contract:**

| Element | Class tokens | Notes |
|---------|--------------|-------|
| Outer container | `flex flex-col items-center justify-center py-16 text-center` `role="alert"` | `alert` role announces to SR immediately |
| Icon | `AlertTriangle` from lucide-react at `h-8 w-8 text-destructive/70 mb-3` | 32px, 70% destructive (softer than solid) |
| Headline | `text-sm font-medium text-foreground mb-1` | "Something went wrong" (locked copy) |
| Message | `text-xs text-muted-foreground max-w-sm mb-4` | 12px, 384px max-width |
| Retry button | shadcn `<Button variant="outline" size="sm">` | Not a primary action — outline keeps copper reserve list intact |

**Dark-mode parity:** `--destructive` is declared for both modes (light `#EF4444`-family, dark muted variant). `text-destructive/70` works via CSS `color-mix` — **verify contrast ratio 4.5:1 against `--background` in both modes** (executor confirms during implementation — if dark-mode contrast fails, raise to `text-destructive/90`).

**WCAG 2.1 AA contrast floor:**

- Headline text against `--background`: ≥4.5:1 (normal text). `text-foreground` already passes.
- Message text (12px, still normal): ≥4.5:1. `text-muted-foreground` is pinned at this floor in Phase 49 tokens; executor verifies no regression.

### 4.3 `<AsyncButton />` — POLISH-05 pending-state wrapper

**File:** `src/components/ui/async-button.jsx` (NEW)

**Prop API (locked):**

```jsx
<AsyncButton
  pending={isSaving}            // required — boolean
  pendingLabel="Saving…"        // optional — falls back to children when absent
  {...buttonProps}              // spread to underlying Button (variant, size, onClick, etc.)
>
  Save changes                  // children = default label
</AsyncButton>
```

**Rules:**
- When `pending={true}`: button is `disabled`, shows `Loader2` spinner, swaps label to `pendingLabel` (or keeps children if absent).
- When `pending={false}`: behaves as a standard `<Button>`.
- All shadcn `<Button>` props pass through (`variant`, `size`, `className`, `onClick`, `type`, `asChild`).
- Do NOT swallow `disabled` from parent — OR with `pending` (parent may disable for a different reason).

**Layout & visual contract:**

| State | Content |
|-------|---------|
| Idle | `{children}` |
| Pending | `<Loader2 className="h-4 w-4 mr-2 animate-spin" aria-hidden="true" />` + `{pendingLabel ?? children}` |

**Implementation reference** — matches the existing ad-hoc pattern at `BusinessIntegrationsClient.jsx:271-328` verbatim:

```jsx
export function AsyncButton({ pending, pendingLabel, disabled, children, ...rest }) {
  return (
    <Button disabled={pending || disabled} {...rest}>
      {pending && <Loader2 className="h-4 w-4 mr-2 animate-spin" aria-hidden="true" />}
      {pending ? (pendingLabel ?? children) : children}
    </Button>
  );
}
```

**Dark-mode parity:** Inherits from underlying `<Button>`. No new concerns.

**Accessibility:** `aria-busy="true"` is NOT added on the button itself (screen readers already announce disabled state; double-announcement is noise). If the page-level section has an async fetch, `aria-busy` goes on the **section container**, not the button.

## 5. Focus Ring Token — POLISH-03 Global Migration

**Current state** (`src/lib/design-tokens.js:34-36`):

```js
export const focus = {
  ring: 'focus:outline-none focus:ring-2 focus:ring-[var(--brand-accent)] focus:ring-offset-1',
};
```

**Locked new state:**

```js
export const focus = {
  ring: 'focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-[var(--brand-accent)] focus-visible:ring-offset-1 focus-visible:ring-offset-[var(--background)]',
};
```

**Exact ring contract:**
- **Trigger:** `focus-visible` only (keyboard + programmatic focus; NOT mouse-click).
- **Color:** `var(--brand-accent)` (copper `#C2410C` light / same hex dark — accent is pinned across modes per Phase 49 tokens).
- **Width:** `ring-2` → 2px.
- **Offset:** `ring-offset-1` → 1px gap between element edge and ring.
- **Offset color:** `var(--background)` — critical for dark mode so the offset blends into the page background instead of showing a light halo.
- **Outline reset:** `focus-visible:outline-none` (prevents double-ring).

**Migration scope (executor sweep):**
1. Update `src/lib/design-tokens.js` — one-line change.
2. Grep-sweep `src/` for `focus:ring`, `focus:outline-none`, `focus:ring-2`, `focus:ring-offset-*`. For each hit, rewrite `focus:` → `focus-visible:`. Expected hit count: 20-40 sites (ChecklistItem has 3; BusinessIntegrationsClient has a few; onboarding wizard has more).
3. Grep-sweep for `focus:ring-[` (custom color) — same migration.
4. Do NOT touch `:focus-within` selectors (different semantic).
5. Do NOT touch `focus:` on native form elements that still need mouse-focus rings (e.g., `<input>` borders) — confirm case-by-case; default is migrate unless the component owner justifies keeping.

**Regression guard:** Phase 58 UAT must include a Playwright (or manual) keyboard walk of each of the 7 pages confirming visible ring on Tab.

**Contrast check:** Copper `#C2410C` on typical page backgrounds (`--background` light `#fff`/warm and dark `oklch(0.145 0 0)`) yields ≥3:1 against background (non-text graphical requirement). **Executor verifies** — if dark-mode ring fails, switch to `var(--ring)` (shadcn's theme-aware token) as fallback.

## 11. Dark Mode Parity Matrix

All new primitives and UI additions must work in both modes. Pinned tokens:

| Element | Light | Dark |
|---------|-------|------|
| EmptyState icon | `text-muted-foreground/30` | same |
| EmptyState headline | `text-foreground` (oklch 0.145) | `text-foreground` (oklch 0.985) |
| EmptyState description | `text-muted-foreground` | same |
| EmptyState CTA | `bg-[var(--brand-accent)] text-[var(--brand-accent-fg)]` | same (copper pinned) |
| ErrorState icon | `text-destructive/70` | `text-destructive/70` (executor validates contrast; fallback `/90`) |
| ErrorState headline | `text-foreground` | same |
| ErrorState message | `text-muted-foreground` | same |
| ErrorState retry button | shadcn outline — border inherits `--border` | same |
| AsyncButton (default variant) | copper primary | copper primary (unchanged) |
| AsyncButton spinner | `animate-spin` on `Loader2` — inherits text color from Button variant | same |
| Skeleton | `bg-accent` (existing token, dark-aware) | same |
| Focus ring | `ring-[var(--brand-accent)]` + `ring-offset-[var(--background)]` | same (offset compensates) |
| Red dot (checklist error) | `bg-red-600` | `bg-red-500` (**only** mode-specific class in new primitives) |
| "Reconnect needed" subtitle | `text-red-600` | `text-red-400` (**only** mode-specific class in new primitives) |
| Last-synced line | `text-muted-foreground` | same |

**Rationale for red explicit mode classes:** CSS variable `--destructive` is theme-muted for buttons and doesn't read as "attention required" in status copy. Fixed Tailwind `red-600/500/400` reads reliably. This is the **only** exception to the "semantic tokens only" rule in new primitives — documented here so the checker doesn't flag it.
