3. **Create `src/app/api/admin/inventory/route.js`** — per D-05, D-07, D-09:

All handlers call `verifyAdmin()` first — return 403 if null. Use `supabase` (service-role) for all phone_inventory operations.

**GET** handler:
- Query params: `page` (default 0), `search` (optional text filter on phone_number)
- `PAGE_SIZE = 25` per UI-SPEC
- Query: `supabase.from('phone_inventory').select('*, assigned_tenant:tenants(business_name, owner_name)', { count: 'exact' }).order('created_at', { ascending: false }).range(page * PAGE_SIZE, (page + 1) * PAGE_SIZE - 1)`
- If `search` param provided: add `.ilike('phone_number', '%' + search + '%')` before `.order()`
- Return: `{ data, total: count, page, pageSize: PAGE_SIZE }`

**POST** handler (add single number) — per D-05:
- Body: `{ phone_number }` (local digits, e.g. "81234567")
- Normalize: prepend `+65` if not already present
- Validate: after normalization, must match `/^\+65\d{8}$/`
- If invalid format: return `{ error: 'Invalid SG number format. Expected 8 digits.' }` with status 400
- Insert: `supabase.from('phone_inventory').insert({ phone_number: normalized, country: 'SG', status: 'available' }).select().single()`
- If error code `23505` (unique violation): return `{ error: 'duplicate' }` with status 409
- On success: return `{ data }` with status 201

**PATCH** handler (retire/reactivate) — per D-07:
- Body: `{ id, action }` where action is `'retire'` or `'reactivate'`
- For `retire`: update status to `'retired'`, set `assigned_tenant_id` to null
- For `reactivate`: update status to `'available'`, set `assigned_tenant_id` to null
- Validate action is one of the two allowed values
- Return updated row
