## Design Theme: Admin vs Tenant

The admin interface uses a **slate-blue palette** intentionally distinct from the tenant Heritage Copper palette (`#0F172A` sidebar + `#C2410C` accent). This makes it visually impossible to confuse admin and tenant contexts.

| Context | Sidebar | Surface | Accent |
|---------|---------|---------|--------|
| Tenant dashboard | `#0F172A` (near-black) | `#F5F5F4` (warm stone) | `#C2410C` (copper) |
| Admin dashboard | `#0F1F3D` (navy) | `#F1F5F9` (cool slate) | `#1D4ED8` (blue) |

The admin layout also renders a persistent "ADMIN" badge in the header to make context unmistakable.

## Layout Structure

Admin uses a **top-tab navigation** (not a sidebar) for the primary admin sections. This differentiates layout from the tenant sidebar pattern and keeps the admin surface feeling like a dedicated operator tool.

### Admin Layout Anatomy

```
┌────────────────────────────────────────────────────────┐
│  [Voco Logo]  ADMIN  │  Phone Inventory  │  Tenants    │  [Signed in as: admin@...] [Sign Out]
├────────────────────────────────────────────────────────┤
│  Page content area (max-w-7xl, px-6)                   │
│                                                         │
│  [Section heading]                  [Primary CTA btn]  │
│  ┌──────────────────────────────────────────────────┐  │
│  │  Table / card content                            │  │
│  └──────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────┘
```

Header: `bg-[#0F1F3D]`, height 56px, sticky top-0 z-30
Header text: white
"ADMIN" badge: 10px / 600 (semibold) / uppercase / `#1D4ED8` bg / white text / rounded-sm, positioned inline after logo

### Impersonation Banner

When admin enters impersonation mode (View as tenant), a **44px sticky banner replaces the admin header** at the top of the page:

```
┌────────────────────────────────────────────────────────┐
│  [Eye icon]  Viewing as: {Tenant Business Name}  (read-only)         [Exit Impersonation]
└────────────────────────────────────────────────────────┘
```

Banner: `bg-amber-50`, `border-b border-amber-300`, text `#92400E` (amber-800)
Exit button: outline variant, `border-amber-400 text-amber-800 hover:bg-amber-100`
All tenant dashboard action buttons render with `opacity-50 pointer-events-none` during impersonation.

## Interaction Contracts

### Phone Inventory Table

- Sorted by: `created_at DESC` default
- Columns: Phone Number | Country | Status | Assigned Tenant | Created Date | Actions
- Actions column per row:
  - `available`: single "Retire" button (outline destructive)
  - `assigned`: single "Retire" button (outline destructive, dimmer) — shows warning confirmation about active tenant
  - `retired`: single "Re-activate" button (outline blue)
- Row hover: `bg-slate-50` (light highlight, no elevation change)
- Status badge: pill shape, 12px text, rounded-full
- Pagination: 25 rows per page, prev/next buttons, "Showing {start}–{end} of {total}"

### Add Number Form

- Layout: Card with "Add Singapore Number" heading
- Input row: non-editable "+65" prefix span (matching Phase 27 pattern) + text input for local digits
- Validation: client-side on blur — 8 digits for SG numbers
- Button: "Add Number" disabled until 8 digits entered
- On success: input clears, table refreshes, success toast "Number added to inventory"
- On duplicate error: inline error text below input (not toast)

### Bulk CSV Import

- Layout: Card with "Bulk Import" heading, collapsed by default behind "Import CSV" link/button
- File input accepts `.csv` only
- CSV format hint: "One phone number per row, local digits only (e.g. 81234567)"
- On upload: parse client-side, show preview table of valid/invalid rows before submitting
- Invalid rows highlighted in red with per-row error message
- Submit button "Import {N} Numbers" disabled if any invalid rows exist
- On success: success toast "Imported {N} numbers successfully", table refreshes

### Tenant Table

- Sorted by: `created_at DESC` default
- Columns: Business Name | Owner | Country | Phone Number | Plan | Sub Status | Onboarding | Prov. Failed | Actions
- Actions column per row: "View as" button (outline) + provisioning_failed toggle (Switch)
- "View as" triggers impersonation — opens `/dashboard` with impersonation context active
- `provisioning_failed` Switch: confirmation dialog before toggling in either direction
- Re-provisioning button appears inline for rows where `provisioning_failed = true`
- Search: text input filters by business name or owner name (client-side for small datasets, or server-side with debounce for large)
- Pagination: 25 rows per page

### Loading States

- Tables: skeleton rows (3 pulsing rows matching column structure) while data loads
- Action buttons show spinner icon replacing label text during async operations (Retire, Re-activate, Import, Trigger Re-Prov)
- No full-page spinners — only inline loading indicators

### Toast Notifications

- Success: green check, 3 second auto-dismiss
- Error: red X, 5 second auto-dismiss, manual dismiss also available
- Position: bottom-right
