### VIP Badge Color (new for Phase 46)

| Element | Background + Text | Border | Icon |
|---|---|---|---|
| VIP badge on LeadCard | `bg-violet-100 text-violet-700` | none (pill badge, no border) | `Star` (lucide, h-3 w-3, filled via `fill-current`) |
| VIP toggle active state in LeadFlyout | `bg-violet-100 text-violet-700` | `border-violet-200` | `Star` (lucide, h-3.5 w-3.5, filled) |

Rationale: Violet is unused by all existing badge palettes (red=emergency/lost, amber=urgent/fallback, stone=routine/AI, orange=new, blue=booked/owner-pickup, emerald=completed, green=paid, violet-50=partially-paid invoice). Using `violet-100/violet-700` creates clear visual distinction for VIP status without conflicting with any existing semantic color.

Source: Claude's discretion per CONTEXT.md -- "How the VIP badge renders on LeadCard (color, position, icon) is at Claude's discretion."

## Page Structure

### Call Routing Page (`/dashboard/more/call-routing`) -- Additive Change

The VIP Callers section is added BELOW the existing "Your Phone Numbers" section and ABOVE the "Ring Duration" section. This placement groups all phone-related configuration together (pickup numbers, then VIP numbers) before the timeout control.

```
[Existing: Page heading "Answer Your Own Calls"]
[Existing: Hero toggle card]

  [AnimatePresence: visible when schedule.enabled]
    [Existing: Your Schedule card]
    [Existing: Your Phone Numbers card]

    [NEW: VIP Callers card]  <-- always visible, NOT gated by schedule.enabled
      [Header row: Star icon (violet) + "VIP Callers" heading]
      [Description: "These callers always ring your phone -- day or night, regardless of schedule."]
      [List of existing VIP number entries -- each: Star icon + phone formatted + label + edit icon + delete icon]
      [Empty state when no VIP numbers]
      [Add VIP number inline form at bottom: phone input + optional label input + "Add VIP number" button]

    [Existing: Ring Duration card]

[Existing: Sticky save bar]
```

Important: The VIP Callers card is rendered OUTSIDE the `AnimatePresence` block that gates on `schedule.enabled`. VIP routing works regardless of whether schedule-based routing is enabled (per D-03: "VIP calls ring the owner 24/7"). The VIP section is always visible on the page.

Source: CONTEXT.md D-06, D-07. VIP section placement: Claude's discretion -- placed after pickup numbers for logical grouping.

### Leads Page (`/dashboard/leads`) -- Additive Change

**LeadCard:** Add VIP badge inline with existing badges. Position: left side of the badges row, before the urgency badge. The VIP badge renders as a small pill with filled Star icon + "VIP" text.

```
[Existing lead card layout]
  [Left side: name, phone, job type badge, urgency badge]
                                           ^-- VIP badge inserted here, before urgency
  [Right side: status badge, invoice badge, timestamp]
```

VIP badge only renders when `lead.is_vip === true`. When absent (false/null/undefined), no badge shown.

**LeadFlyout:** Add "VIP Caller" toggle row between the contact details section and the Pipeline Status section.

```
[Existing: SheetHeader with name, phone, email]
[Existing: Contact details section (address, job type, etc.)]

[NEW: VIP Caller toggle row]
  [Separator]
  [Row: Star icon (violet) + "VIP Caller" label + Switch toggle]
  [Description below: "Always ring your phone when this caller dials in."]
  [Separator]

[Existing: Pipeline Status section]
[Existing: SheetFooter with Mark as Lost]
```

Source: CONTEXT.md D-06 -- "a Mark as VIP / Remove VIP action in the lead flyout on the leads page."

## States and Interactions

### VIP Callers Section (Call Routing Page)

| State | Behavior |
|---|---|
| Page loading | Full-page Skeleton (existing pattern) covers all sections including VIP |
| Empty -- no VIP numbers | Muted text: "No VIP callers yet. Add a phone number below to give someone priority access." |
| VIP number row display | Row: Star icon (violet-500, h-4 w-4) + phone number (text-sm font-medium) + label (text-xs text-stone-500) + edit button + delete button |
| Add VIP number | Inline form at bottom: dashed border container with phone input + optional label input + "Add VIP number" button (variant="outline") |
| Editing VIP number | Row expands inline with pre-filled phone + label inputs, Save + Cancel buttons. Same pattern as existing pickup number editing. |
| Delete VIP number | Immediate removal from list, no confirmation dialog (low-risk, easily re-added). Sonner toast: "VIP number removed" |
| Invalid phone | Inline field error below input: "Enter a valid phone number including country code (e.g. +1 555 000 0000)." |
| Duplicate VIP phone | Inline field error: "This number is already in your VIP list." |
| Duplicate with pickup number | No cross-check needed -- a number can be both a pickup number AND a VIP number (different purposes) |
| Phone normalization | Accept formats: 555-123-4567, (555) 123-4567, +1 555 123 4567. Auto-prepend country code from tenant's country if omitted. Display cleaned E.164. Reuse `cleanPhone()` + `validatePhone()` from existing call-routing page. |
| Save (all settings) | VIP numbers included in the existing PUT /api/call-routing payload alongside schedule, pickup numbers, and dial timeout. Single "Save changes" CTA in sticky bar. |
| No cap on entries | No counter shown (unlike "5 of 5" for pickup numbers). No max-reached state. Per D-09: unlimited. |

### VIP Toggle (LeadFlyout)

| State | Behavior |
|---|---|
| Lead is not VIP | Switch is OFF, label reads "VIP Caller", description reads "Always ring your phone when this caller dials in." |
| Toggle ON | Immediate PATCH to `/api/leads/[id]` with `{ is_vip: true }`. Sonner toast: "Caller marked as VIP". Star icon fills with violet. No page reload needed -- Realtime subscription auto-updates LeadCard. |
| Toggle OFF | Immediate PATCH to `/api/leads/[id]` with `{ is_vip: false }`. Sonner toast: "VIP status removed". No confirmation dialog (low-risk, easily re-toggled). |
| Toggle error | Sonner toast: "Could not update VIP status -- try again". Switch reverts to previous position (optimistic revert). |
| Lead without from_number | VIP toggle row hidden -- VIP routing requires a phone number to match against at webhook time |

### VIP Badge (LeadCard)

| State | Behavior |
|---|---|
| `is_vip === true` | Violet badge with filled Star icon + "VIP" text rendered in badges row, before urgency badge |
| `is_vip === false` or absent | No badge rendered -- no visual change to existing card layout |
| Realtime update | When `is_vip` changes via Realtime subscription, LeadCard re-renders with/without badge immediately |
