<task type="auto">
  <name>Task 1: Create Checkout Session API endpoint</name>
  <files>src/app/api/onboarding/checkout-session/route.js</files>
  <action>
Create `src/app/api/onboarding/checkout-session/route.js` with a POST handler. Per D-01/D-02/D-03, this creates a Stripe Checkout Session for the selected plan with a 14-day trial and CC required.

**Request:** `{ plan: 'starter' | 'growth' | 'scale' }`
**Response:** `{ url: string }` — the Stripe Checkout URL

**Implementation:**
1. Authenticate user via `createSupabaseServer()` + `getUser()`. Return 401 if not authenticated.
2. Parse request body: `const { plan } = await request.json()`
3. Look up price ID from env var map:
   ```javascript
   const PRICE_MAP = {
     starter: process.env.STRIPE_PRICE_STARTER,
     growth:  process.env.STRIPE_PRICE_GROWTH,
     scale:   process.env.STRIPE_PRICE_SCALE,
   };
   const priceId = PRICE_MAP[plan];
   ```
   Return 400 if plan is invalid or priceId is missing.

4. Look up tenant using service role client:
   ```javascript
   const { data: tenant } = await adminSupabase
     .from('tenants')
     .select('id, owner_email, business_name')
     .eq('owner_id', user.id)
     .single();
   ```
   Return 404 if no tenant found.

5. Create Checkout Session:
   ```javascript
   const session = await stripe.checkout.sessions.create({
     mode: 'subscription',
     payment_method_collection: 'always',  // D-03: CC required
     line_items: [{ price: priceId, quantity: 1 }],
     subscription_data: {
       trial_period_days: 14,
       metadata: { tenant_id: tenant.id },  // Critical: webhook uses this to find tenant
     },
     customer_email: tenant.owner_email,
     metadata: { tenant_id: tenant.id },     // On session too for checkout.session.completed
     success_url: `${process.env.NEXT_PUBLIC_APP_URL}/onboarding/checkout-success?session_id={CHECKOUT_SESSION_ID}`,
     cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/onboarding/plan`,
   });
   ```

6. Return `{ url: session.url }`

**Error handling:** Wrap stripe.checkout.sessions.create in try/catch. On Stripe error, log it and return 500 with `{ error: 'Failed to create checkout session' }`.

**Critical:** The `metadata.tenant_id` on BOTH `subscription_data` and the session itself is essential. The webhook handler (Plan 02) reads `subscription.metadata.tenant_id` to find which tenant the subscription belongs to. Without it, webhook processing silently fails.
