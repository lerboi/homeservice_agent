  <action>
Create `src/app/api/stripe/webhook/route.js` with a single POST export. Follow the Retell webhook pattern but adapted for Stripe per D-06.

**Structure:**

1. **Raw body + signature verification (D-06):**
   - Use `await request.text()` (NOT request.json() — this breaks Stripe signature verification)
   - Get signature from `request.headers.get('stripe-signature')`
   - Call `stripe.webhooks.constructEvent(rawBody, sig, process.env.STRIPE_WEBHOOK_SECRET)`
   - On verification failure: log error, return 401

2. **Idempotency check (D-09):**
   - Insert into `stripe_webhook_events` with `{ event_id: event.id, event_type: event.type }`
   - If insert fails with error code `23505` (unique violation): return `{ received: true }` immediately (already processed)

3. **Event routing (D-08 — synchronous, no after()):**
   Route via if/else chain (not switch — matches Retell pattern):
   - `checkout.session.completed` -> handleCheckoutCompleted
   - `customer.subscription.created` -> handleSubscriptionEvent
   - `customer.subscription.updated` -> handleSubscriptionEvent
   - `customer.subscription.deleted` -> handleSubscriptionEvent
   - `customer.subscription.paused` -> handleSubscriptionEvent
   - `customer.subscription.resumed` -> handleSubscriptionEvent
   - `customer.subscription.trial_will_end` -> handleTrialWillEnd
   - `invoice.paid` -> handleInvoicePaid
   - `invoice.payment_failed` -> handleInvoicePaymentFailed (log only in Phase 22 — notification in Phase 24)
   - All others: log event.type, return 200 (D-07)

4. **Return** `Response.json({ received: true })` after processing.

**Helper: handleCheckoutCompleted(session):**
   - Extract `tenant_id` from `session.metadata.tenant_id`
   - If no tenant_id, log warning and return
   - Set `onboarding_complete = true` on the tenant row using service role client:
     `supabase.from('tenants').update({ onboarding_complete: true }).eq('id', tenantId)`
   - Retrieve the subscription from Stripe: `stripe.subscriptions.retrieve(session.subscription)`
   - Call handleSubscriptionEvent with the retrieved subscription to create the initial subscriptions row

**Helper: handleSubscriptionEvent(subscription):**
   This is the core sync function used for created/updated/deleted/paused/resumed events.
   - Extract `tenantId` from `subscription.metadata?.tenant_id`
   - If no tenantId, log warning and return

   **Out-of-order protection (D-10):**
   - Compute `stripeUpdatedAt = new Date(subscription.created * 1000).toISOString()` for created events, or `new Date(subscription.updated * 1000).toISOString()` for updates — use whichever timestamp the subscription object provides (prefer `.updated` if present, fallback to `.created`)
   - Query current row: `supabase.from('subscriptions').select('stripe_updated_at').eq('stripe_subscription_id', subscription.id).eq('is_current', true).maybeSingle()`
   - If currentRow exists AND `currentRow.stripe_updated_at >= stripeUpdatedAt`, skip (stale event)

   **Price-to-plan mapping (D-12):**
   ```javascript
   const PLAN_MAP = {
     [process.env.STRIPE_PRICE_STARTER]: { plan_id: 'starter', calls_limit: 40 },
     [process.env.STRIPE_PRICE_GROWTH]:  { plan_id: 'growth',  calls_limit: 120 },
     [process.env.STRIPE_PRICE_SCALE]:   { plan_id: 'scale',   calls_limit: 400 },
   };
   ```
   Extract priceId from `subscription.items.data[0]?.price?.id`

   **Status mapping (D-14):**
   Map subscription.status -> local status. Map `incomplete_expired` -> `canceled`.

   **History table insert (D-13):**
   1. Set `is_current = false` on all existing rows for this stripe_subscription_id
   2. Insert new row with `is_current = true` containing all fields:
      tenant_id, stripe_customer_id (subscription.customer), stripe_subscription_id,
      stripe_price_id, plan_id, status, calls_limit, calls_used (0 for new, carry forward from prior row for updates),
      trial_ends_at, current_period_start, current_period_end, cancel_at_period_end,
      stripe_updated_at, is_current: true

   **calls_used carry-forward:** When updating (prior row exists), read `calls_used` from the prior current row before marking it inactive, and carry it forward to the new row. Only reset to 0 on invoice.paid (handled separately). For new subscriptions (no prior row), default to 0.

**Helper: handleInvoicePaid(invoice):**
   - Only process if `invoice.billing_reason === 'subscription_cycle'` (cycle renewal, not first invoice)
   - Extract subscription_id from `invoice.subscription`
   - Reset calls_used to 0 on the current subscription row:
     `supabase.from('subscriptions').update({ calls_used: 0 }).eq('stripe_subscription_id', subscription_id).eq('is_current', true)`

**Helper: handleTrialWillEnd(subscription):**
   - Log for now: `console.log('[stripe/webhook] Trial ending soon for tenant:', subscription.metadata?.tenant_id)`
   - Phase 24 will add notification logic here

**Helper: handleInvoicePaymentFailed(invoice):**
   - Log for now: `console.log('[stripe/webhook] Payment failed for subscription:', invoice.subscription)`
   - Phase 24 will add notification logic here

**Critical pitfalls to avoid:**
- Do NOT use `request.json()` — raw text is required for signature verification
- Do NOT use `after()` — per D-08, processing is synchronous
- Do NOT upsert — per D-13, always INSERT new rows (mark prior inactive first)
- Do NOT forget to carry forward calls_used on subscription updates
