### New UI elements introduced in Phase 56

**"Preferred" badge** — inline badge rendered on the Jobber card header row only when BOTH Xero and Jobber are connected simultaneously. Communicates that Jobber is the preferred source for home-service context without cluttering the disconnected or single-provider-connected states.

Exact markup position: appended to the header row (`flex items-center gap-3 mb-4`) after the provider name `<span>`, right-aligned via `ml-auto`.

```
<span className="ml-auto inline-flex items-center rounded-full px-2 py-0.5 text-[11px] font-medium bg-emerald-50 text-emerald-700 dark:bg-emerald-950/40 dark:text-emerald-300">
  Preferred
</span>
```

Render condition: `connected === true && status.xero !== null` (both providers have a non-null row).

**connect_jobber checklist item** — no new visual component. Appended to the existing setup checklist response array. Same `{id, label, description, completed, href}` shape as `connect_xero`.

## Interaction States: Jobber Card

The Jobber card is rendered via the same `PROVIDER_ORDER.map` loop as the Xero card. States are structurally identical; only copy and render-conditions differ. All four states must be implemented:

### State 1 — Disconnected (no change from P54 scaffold)

- Status line: `"Connect Jobber to share customer and job history with your AI receptionist during calls."` — muted text (`text-muted-foreground`)
- Button: `"Connect Jobber"` — accent background, white text, full width, size sm
- Loading variant: `Loader2` spinner + `"Connecting…"` label, button disabled
- Source: `PROVIDER_META.jobber.disconnectedStatus` already in live code (do not change)

### State 2 — Connected, no error

- Status line (invoicing off): `"Connected. Sharing customer and job history with your AI receptionist."` — emerald text
- Status line (invoicing on): `"Connected. Sharing customer history and sending invoices."` — emerald text
- Timestamp: `"Last synced {relative time}"` in `text-xs text-muted-foreground`, rendered below status line when `last_context_fetch_at` is non-null, omitted when null
- Button: `"Disconnect"` — `variant="outline"` with `border-red-600 text-red-600 hover:bg-red-50 dark:hover:bg-red-950/40`, full width, size sm
- "Preferred" badge: rendered in the header row (right-aligned via `ml-auto`) when Xero is also connected (`status.xero !== null`)
- Source: `PROVIDER_META.jobber.connectedStatusInvoicingOff` and `connectedStatusInvoicingOn` already in live code

### State 3 — Reconnect needed / token error

- Inline banner inside card, above button area: amber border (`border-amber-300`), amber background (`bg-amber-50`), dark-mode variants (`dark:border-amber-700 dark:bg-amber-950/40 dark:text-amber-200`)
- `AlertTriangle` icon: `h-4 w-4 text-amber-600 aria-hidden="true"`
- Banner message: `"Reconnect needed — Jobber token expired. Your AI receptionist can't access Jobber customer info until you reconnect."`
- Primary action: `Button` size sm, full width, accent background: `"Reconnect Jobber"` — initiates same `/api/integrations/jobber/auth` OAuth redirect
- Loading variant for primary: `Loader2` + `"Reconnecting…"`, button disabled
- Secondary action: `Button` variant="ghost" size="sm" className="w-full text-red-600 hover:text-red-700 hover:bg-red-50 dark:hover:bg-red-950/40": `"Disconnect"` — opens disconnect AlertDialog
- "Preferred" badge: NOT rendered in error state (token is expired — Jobber is not actively preferred until reconnected)

### State 4 — Loading skeleton (no change from P54)

- `Skeleton` for icon area, provider name, status lines, and button
- `aria-busy="true"` on grid container
- Source: existing skeleton render path (shared across both cards)

## Integrations Page: Data Passed to Client Component

`/dashboard/more/integrations/page.js` (server component) must pass `last_context_fetch_at` and `error_state` for the Jobber row as part of the `initialStatus` prop. The Xero pattern from P55 is the exact template:

```js
// Server component query — add Jobber row alongside Xero row
const { data: jobberCreds } = await supabase
  .from('accounting_credentials')
  .select('id, last_context_fetch_at, error_state')
  .eq('tenant_id', tenantId)
  .eq('provider', 'jobber')
  .maybeSingle();

// Pass into BusinessIntegrationsClient
initialStatus={{ xero: xeroCreds || null, jobber: jobberCreds || null }}
```

No new page structure — only extending the existing data fetch.

## Accessibility

All rules from Phase 55 carry forward without change:

- All async buttons enter `disabled` state during pending operations (Connect, Reconnect, Disconnect)
- `Loader2` spinner has `aria-hidden="true"` on icon; visible label (`"Connecting…"` / `"Reconnecting…"` / `"Disconnecting…"`) provides the accessible name
- `AlertDialog` uses Radix Dialog semantics; focus trap and escape-to-close handled by Radix
- Reconnect-needed `Alert` uses `role="alert"` (via shadcn Alert component default)
- Timestamp element rendered as `<span>` inside card — no interactive role
- Grid container uses `aria-busy="true"` during skeleton state
- `JobberIcon` SVG already has `aria-label="Jobber"` and `<title>Jobber</title>` (confirmed in live code line 38-39)
- "Preferred" badge: purely informational — rendered as `<span>` with no interactive role, no focus target
