## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge dot, stagger delay increments |
| sm | 8px | Icon label gap in icon nav, inner badge padding |
| md | 16px | Card content padding, element spacing within step |
| lg | 24px | Card internal gap, gap between section heading and content |
| xl | 32px | Section inner padding (`p-8`), heading margin-bottom |
| 2xl | 48px | Section vertical padding (mid-density sections) |
| 3xl | 64px | Full viewport section min-height buffer, section heading margin-bottom |

Exceptions:
- How It Works full-viewport steps: each step uses `min-h-screen` (100vh) as its height unit — not a fixed pixel value
- Viewport-relative vertical padding per step: `py-24 md:py-32` (96px / 128px) — outside the 4-grid token scale but consistent with the viewport-proportion intent of D-01
- Features carousel card min-width (desktop peek cards): `min-w-[280px]` — practical carousel layout constraint, not a spacing token
- Active carousel card scale-up: `scale(1.04)` — unitless transform, not a spacing value
- 44px minimum touch target on arrow buttons and icon nav items (WCAG 2.5.5)

### How It Works step number treatment (D-09 stagger)

The step number is the FIRST element rendered per step. Its visual treatment is a large centered numeral:
- Font size: `text-[6rem] md:text-[8rem]` (96px / 128px)
- Weight: 600 (semibold)
- Color: Faint step-accent color at 15% opacity (same `numberColor` tokens from Phase 32 per-card color table)
- Position: Above the illustration icon, centered

Stagger order within each step: (1) step number → (2) illustration icon → (3) title → (4) description. Stagger delay: 200ms between each element.

Weights used: 400 (regular) + 600 (semibold). No other weights.

## Visual Focal Point

**How It Works section:** The illustration-style icons are the focal point of each viewport step. They appear at center stage after the step number fades in, before the title and description. The large faint step-number in the background creates depth without competing. Soft background geometric shapes (4-6% opacity circles) provide texture. The alternating white/off-white backgrounds give each step a distinct panel feel as the user scrolls.

**Features carousel:** The centered active card (slightly larger than peek cards) is the visual focal point. The icon nav grid below provides secondary orientation — the active dot directly links user attention between the icon strip and the card above.

## How It Works Section Layout Contract

### Full-Viewport Step Layout (D-01, D-02, D-05)

Each of the 4 steps is a full-viewport section. Layout per step:

```
┌─────────────────────────────────────────────────────────────┐
│  (background: white or off-white, alternating)              │
│                                                             │
│  [soft background shape — faint circle, 400px, centered]   │
│                                                             │
│              [step number — large, faint]                   │
│              [illustration icon — 80px / 96px]              │
│              [title — h2/h3 heading size]                   │
│              [gradient accent line — 80px, copper]          │
│              [description — one sentence, body]             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

Alignment: All content centered horizontally (D-05).
Max-width constraint on text content: `max-w-xl` (672px) — prevents overly wide lines on large viewports.
Vertical centering: `flex flex-col items-center justify-center min-h-screen`

### Tailwind structure per step

```
<section className="relative flex flex-col items-center justify-center min-h-screen
                    py-24 md:py-32 px-6 overflow-hidden
                    {alternating bg: bg-white / bg-[#FAFAF9]}">

  <div className="absolute inset-0 flex items-center justify-center pointer-events-none"
       aria-hidden="true">
    <div className="w-[400px] h-[400px] rounded-full {step.shapeFill}" />
  </div>

  {/* Centered content stack */}
  <div className="relative z-10 flex flex-col items-center text-center max-w-xl gap-6">
    {/* 1. Step number */}
    <span className="text-[6rem] md:text-[8rem] font-semibold leading-none
                     select-none pointer-events-none {step.numberColor}">
      {step.number}
    </span>

    {/* 2. Illustration icon */}
    <div className="w-20 h-20 md:w-24 md:h-24 rounded-3xl {step.iconBg}
                    flex items-center justify-center">
      <step.Icon className="w-10 h-10 md:w-12 md:h-12 {step.iconColor}" />
    </div>

    {/* 3. Title */}
    <h3 className="text-3xl md:text-4xl lg:text-5xl font-semibold tracking-tight
                   leading-[1.2] text-[#0F172A]">
      {step.title}
    </h3>

    {/* 4. Gradient accent line */}

      {step.description}
    </p>
  </div>

```

### Stagger animation per step (D-09)

Using Framer Motion `useInView` + `variants` with stagger:

| Element | Initial State | Animated State | Delay |
|---------|--------------|----------------|-------|
| Step number | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 0ms |
| Illustration icon | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 200ms |
| Title | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 400ms |
| Gradient accent line | `opacity: 0, scaleX: 0` | `opacity: 1, scaleX: 1` | 600ms |
| Description | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 800ms |

Duration per element: 500ms. Easing: `[0.22, 1, 0.36, 1]` (decelerate out).

`once: true` — triggers once per viewport entry, does not re-trigger on scroll back. This is the correct behavior for linear scroll-through content.

### Parallax on illustration icons (D-10)

Using Framer Motion `useScroll` + `useTransform` on each step's scroll progress:
- Icon vertical offset range: `-12px` to `+12px` as step scrolls through viewport
- Text block has zero parallax (only icon moves)
- Offset declared as Claude's discretion: `useTransform(stepScrollYProgress, [0, 1], [-12, 12])`

### Soft background shapes (D-11)

One centered circle per step:
- Size: `w-[400px] h-[400px]` (desktop) / `w-[280px] h-[280px]` (mobile)
- Shape: `rounded-full`
- Fill: Per-step accent color at 4-5% opacity (see per-step color table above)
- Position: Absolute, centered in the viewport step
- Not animated — static texture only

A secondary smaller rectangle (optional, Claude's discretion):
- Size: `w-[180px] h-[80px] rounded-3xl`
- Fill: Same accent at 2-3% opacity
- Position: Offset from center (e.g., top-right corner of step content area)

## Features Carousel Section Layout Contract

### Carousel structure (D-13, D-14, D-15, D-16, D-18)

```
┌────────────────────────────────────────────────────────────────────────┐
│  Section heading block (eyebrow + h2)                                  │
│                                                                        │
│  [← arrow]  [peek card]  [ACTIVE CARD — larger]  [peek card]  [→ arrow] │
│                                                                        │
│  [icon grid: 7 icons in a row, active underline dot]                   │
└────────────────────────────────────────────────────────────────────────┘
```

### Carousel viewport and card sizing

| Context | Visible Cards | Active Card Width | Peek Card Width |
|---------|--------------|-------------------|-----------------|
| Desktop (lg+) | 3 (1 active + 2 peek) | `max-w-[520px]` | `min-w-[280px]` |
| Tablet (md) | 2.5 (1 active + 1 full + 0.5 peek) | `max-w-[440px]` | `min-w-[240px]` |
| Mobile (< md) | 1.2 (1 active + 0.2 peek) | `w-full` | 16px peek on right side only |

Active card scale: `scale(1.04)` — subtle size lift, not dramatic zoom.
Active card top border: `border-t-2 border-t-[#F97316]` — accent indicator.
Peek cards: `opacity-70` + no scale transform.
Transition: `transition-all duration-400 ease-[0.22,1,0.36,1]`

### Carousel scroll behavior (D-18)

Implementation via CSS scroll-snap or a lightweight JS carousel (no external carousel library):
- Container: `overflow-x: auto`, `scroll-snap-type: x mandatory`, `scroll-behavior: smooth`
- Each card: `scroll-snap-align: center`
- Momentum scrolling: `-webkit-overflow-scrolling: touch` on mobile
- Pointer events: drag to scroll on desktop (mouse drag detection via `onPointerDown` / `onPointerMove`)

### Auto-advance (D-15)

- Interval: 5000ms (5 seconds)
- Pauses on: any user interaction (arrow click, icon nav click, swipe/drag)
- Resumes: 8000ms after last interaction
- State: `useRef` for intervalRef, `clearInterval` on interaction

### Arrow buttons (D-14)

Position: Vertically centered on the carousel track, outside the card area on desktop, overlaid on cards on mobile.
- Size: `w-11 h-11` (44px minimum touch target)
- Resting: `bg-white border border-stone-200 text-[#0F172A] rounded-full shadow-sm`
- Hover: `bg-[#F97316] text-white border-[#F97316] shadow-md transition-colors duration-200`
- Icons: `ChevronLeft` / `ChevronRight` from Lucide, 20px
- `aria-label="Previous feature"` / `aria-label="Next feature"`
- Hidden on mobile (`hidden md:flex`) — swipe is the primary mobile interaction

### Micro visual animation trigger strategy (D-16)

Micro visuals that have CSS `animation` properties (e.g. `langFloat`, `clockSpin`) should only run when their card is the active card. Implementation:
- Pass `isActive` boolean prop to each micro visual
- When `isActive === false`: apply `animation-play-state: paused` via inline style
- When `isActive === true`: apply `animation-play-state: running`
- This prevents all 7 animations from running simultaneously, reducing GPU load

## Animation Contract

### How It Works animations

| Element | Animation | Duration | Easing | Trigger |

### Breakpoints

| Breakpoint | Width | How It Works | Features Carousel |

| `lg` | 1024px | Full layout, icon `w-24 h-24`, step number `text-[8rem]` | 3 cards visible (1 active + 2 peek) |

### How It Works mobile

- Each step still uses `min-h-screen` — no change to viewport intent
- Reduced padding: `py-16 px-4` (64px vertical, 16px horizontal)
- Step number: `text-[5rem]` (80px) on mobile, `text-[6rem]` (96px) on md+, `text-[8rem]` (128px) on lg+
- Icon container: `w-16 h-16` mobile / `w-20 h-20` md / `w-24 h-24` lg
- Title: `text-2xl md:text-3xl lg:text-5xl` — scales down to 24px on mobile
- Background shape: `w-[280px] h-[280px]` mobile / `w-[400px] h-[400px]` desktop
- Parallax disabled on mobile (`useMediaQuery` check for `(max-width: 767px)` → set transform to static 0)

### Features mobile

- Carousel: `overflow-x: auto`, `scroll-snap-type: x mandatory`, each card `scroll-snap-align: center`
- One full card visible + 16px right-side peek to signal scrollability
- Arrow buttons: `hidden md:flex` — hidden on mobile
- Icon nav: horizontally scrollable row, `overflow-x: auto`, `scroll-snap-type: x mandatory`, each icon `scroll-snap-align: center`
- Icon nav scroll is independent of carousel scroll (two separate scroll containers)

## Accessibility Contract

- Step titles are `<h3>` elements under section `<h2>` (correct heading hierarchy)
- Step illustration icons: `aria-hidden="true"` (decorative — step title and description provide context)
- Background shapes: `aria-hidden="true"` + `pointer-events-none`
- Step number decorative numerals: `aria-hidden="true"` + `select-none pointer-events-none`
- Gradient accent lines: `aria-hidden="true"`
- Carousel: `role="region"` + `aria-label="Features"` on the carousel container
- Arrow buttons: `aria-label="Previous feature"` / `aria-label="Next feature"`, minimum `44×44px`
- Icon nav buttons: each has `aria-label="{feature title}"` + `aria-pressed={isActive}` or `aria-current="true"` on the active item
- Micro visuals: `aria-hidden="true"` (decorative — feature titles and descriptions are the accessible content)
- Color contrast: `#0F172A` on `#FFFFFF` = 21:1 (AAA). `#475569` on `#FAFAF9` = 5.8:1 (AA). `#94A3B8` on `#FAFAF9` = 3.2:1 (AA for large text only — icon nav labels at 11px are semibold, borderline; upgrade to `#64748B` if failing audit)
- Reduced motion: Framer Motion animations wrap in `useReducedMotion()` — return static positions when true. CSS keyframe micro visuals: wrap in `@media (prefers-reduced-motion: no-preference)`. Auto-advance carousel: when `prefers-reduced-motion: reduce`, disable auto-advance entirely.
- Keyboard: Arrow buttons reachable via Tab. Icon nav buttons reachable via Tab. Left/right keyboard arrows navigate carousel when focus is inside the carousel region.
- Touch: Swipe gesture on carousel does not interfere with vertical page scroll. `touch-action: pan-y` on carousel container (allows vertical scroll through, horizontal swipe captured only for horizontal motion).

Source: Phase 32 UI-SPEC accessibility section; WCAG 2.1 AA.
