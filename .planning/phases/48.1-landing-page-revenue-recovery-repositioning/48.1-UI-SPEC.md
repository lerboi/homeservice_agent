## Spacing Scale

Declared values (multiples of 4 only). The landing page uses px class values directly — all values must map to this grid:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding |
| sm | 8px | Chip padding (px-2), tight inline gaps |
| md | 16px | Default element spacing, section internal padding-x on mobile |
| lg | 24px | Card padding (p-6), section padding-x on desktop |
| xl | 32px | Card grid gaps, carousel track padding |
| 2xl | 48px | Section vertical padding (py-20 = 80px rounds to nearest; use py-12 = 48px for tighter sections) |
| 3xl | 64px | Full section vertical padding (py-16 = 64px) |
| section-y-default | 80px | `py-20` — standard section vertical padding matching existing landing sections |
| section-y-wide | 112px | `py-28` — wide-breathing sections like PracticalObjectionsGrid |

Exceptions:
- Touch targets on play/pause buttons: minimum 44px (WCAG 2.1 SC 2.5.5)
- Transcript highlight active line: 2px left border accent, no additional spacing exception
- Integration logo container: ~200px height on desktop as specified in D-30; wraps to 2-row grid (3+3) at 375px

## Typography

Source: `PracticalObjectionsGrid.jsx`, `HeroSection.jsx`, `FeaturesCarousel.jsx`, `globals.css`. All values match the existing "Premium Dark SaaS" design contract from Phase 13 and Phase 47.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px (`text-[15px]`) | 400 (regular) | 1.6 (relaxed) | Card body copy, section paragraphs, FAQ answers |
| Label / Chip | 14px (`text-[14px]`) | 600 (semibold) | 1.4 | Eyebrow pills, stat chips, micro-proof reassurance lines, tab labels, badge text |
| Heading | 30px / 36px (`text-3xl md:text-[2.25rem]`) | 600 (semibold) | 1.2 (tight) | Section H2s |
| Display | 48px–60px (`text-[2.25rem] md:text-[2.5rem] lg:text-[3rem]`) | 600 (semibold) | 1.2 (tight) | Hero H1 |

Heading exceptions:
- `$260,400/year` cost-of-silence stat: `text-5xl md:text-6xl` at weight 700 (bold) — single numeric display treatment to maximize shock value, matching existing OBJ-03 card pattern
- Stay-in-Control section pull quote: `text-[24px] md:text-[30px]` at weight 600 — matches existing `OwnerControlPullQuote.jsx` exactly
- Hero sub line: `text-base sm:text-lg md:text-xl` at weight 400 — matches existing hero sub pattern in `HeroSection.jsx`
- Transcript panel active line: 15px, weight 600, color `#F97316` — distinguishes from inactive lines at weight 400, color `#475569`

No new font families. No new font weights beyond 400 and 600 (700 for the single dollar-figure display only).

## Section-by-Section Visual Contract

- Sub: `text-base sm:text-lg md:text-xl text-white/60` regular. Exact copy: `Voco AI answers, triages, and books every call — in under 1 ring.`
- Micro-proof line (below CTAs): `text-xs sm:text-sm text-white/35`. Exact copy: `Answers in English, Spanish, Mandarin, Malay, and 70+ more. 5-minute setup. No installs.`
- Primary CTA button: `bg-[#F97316]` orange filled, white text, `text-[15px]` semibold. Label: `Hear Voco in action`. Anchor: `#audio-demo`. Touch target minimum 44px height.

### Section 2: Audio Demo

- `id="audio-demo"` attribute on section element (anchor target for primary CTA)
- Background: `bg-white` or `bg-[#FAFAF9]` — use white to contrast with hero dark
- Section eyebrow: `text-[14px] font-semibold text-[#F97316] tracking-wide uppercase` pattern
- Section H2: `text-3xl md:text-[2.25rem] font-semibold text-[#0F172A]` — suggested: `Hear Voco handle a real call`

- Player: extends `AudioPlayerCard.jsx` pattern. Waveform visualizer (24 bars), play/pause button, MM:SS timestamp. Single persistent component; tab switch via state only.
- Transcript panel: rendered beside player on desktop (2-column layout), below player on mobile. Inactive lines: `text-[15px] text-[#475569]`. ## Interaction Contracts

### Audio Player Interaction

- Tab switch: instant (no transition delay). Audio resets to 0:00 on tab switch regardless of playback state. Previous audio pauses automatically via existing `window.__vocoPlayingAudio` coordination pattern in `AudioPlayerCard.jsx`.
- Transcript sync: `timeupdate` event drives active-line highlight. Line boundary data is a static array of `{ startSec, endSec, text }` objects colocated with the audio source definition.
- `prefers-reduced-motion`: waveform bars remain static (no pulsing animation). Play/pause and transcript sync continue to function normally.
- Keyboard: play/pause button is a `<button>` — native keyboard and screen reader accessible. Transcript panel is `aria-live="polite"` so active-line changes are announced.

### Integration Strip

- No interaction. Static display only.
- "Coming soon" logos are NOT links or buttons. No tooltips needed.

### FeaturesCarousel

- Existing carousel navigation pattern preserved (chevron buttons + pill indicators).
- `useReducedMotion` check already present in `FeaturesCarousel.jsx` — preserve.

### Stay-in-Control Cards

- Hover state: `hover:-translate-y-1` and orange shadow (existing `CARD_CLS`). No click action — purely informational.
- `prefers-reduced-motion`: translate hover suppressed (`motion-safe:hover:-translate-y-1`).

### AnimatedSection / AnimatedStagger / AnimatedItem

All new sections MUST wrap content in `<AnimatedSection>` (and `<AnimatedStagger>` / `<AnimatedItem>` for card grids). No exceptions. `useReducedMotion` is handled internally in `AnimatedSection.jsx`.

### Dynamic Import Pattern

All sections below the fold MUST follow the existing dynamic import pattern from `page.js`:
- `dynamic(() => import(...))` with explicit loading skeleton
- Loading skeleton must reserve explicit pixel height matching the rendered section's approximate height (prevents CLS)
- `ssr: false` for components using audio API or browser-only APIs
