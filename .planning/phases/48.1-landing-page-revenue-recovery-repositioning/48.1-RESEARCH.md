## Common Pitfalls

### Pitfall 1: ScrollLinePath CSS Stacking Broken by New Sections
**What goes wrong:** New section inside `<ScrollLinePath>` has `relative` on its outer `<section>` element, causing the section background to paint over the copper SVG line.
**Why it happens:** Developers instinctively add `relative` to a section that has `overflow-hidden` or absolute-positioned decorative elements.
**How to avoid:** Only the inner content `<div>` (the `max-w-5xl mx-auto` container) should have `className="relative z-[1]"`. The outer `<section>` must have NO `relative` class.
**Warning signs:** The copper line disappears at the section boundary.

### Pitfall 2: Audio LCP Regression from Eager Loading
**What goes wrong:** Adding `preload="auto"` or forgetting `ssr: false` on the AudioDemoSection dynamic import causes audio metadata to block LCP.
**Why it happens:** Audio elements with `preload="auto"` trigger network requests on load.
**How to avoid:** Keep `preload="metadata"` (only load duration). Use `dynamic(..., { ssr: false })` for AudioDemoSection since it uses browser-only `window.__vocoPlayingAudio` and `HTMLAudioElement`.
**Warning signs:** Lighthouse LCP score increases.

### Pitfall 3: Tab Switch Doesn't Reset Transcript Highlight
**What goes wrong:** User plays Emergency tab, switches to Routine, but transcript still highlights the Emergency line position because `activeLineIndex` state is not reset.
**Why it happens:** Tab switch updates `src` but doesn't reset player state.
**How to avoid:** On tab switch: set `isPlaying = false`, reset `currentTime = 0`, reset `activeLineIndex = null`, call `audio.pause()`, set new `src`, call `audio.load()`.

### Pitfall 4: FeaturesCarousel Test Assertions Break `npm test`
**What goes wrong:** The existing `landing-sections.test.js` asserts 9 features, RotatingText in hero, BeyondReceptionistSection in page.js — all of which change in this phase. Running `npm test` after Wave 1/2 will produce failures until Wave 0 test updates land.
**Why it happens:** Tests written in Phase 47 assert the Phase 47 structure.
**How to avoid:** Update `landing-sections.test.js` in Wave 0 before any component changes ship.

### Pitfall 5: Missing #calculator Anchor Breaks "Calculate yours →" Link
**What goes wrong:** The CostOfSilenceBlock renders "Calculate yours →" pointing to `/pricing#calculator` but the anchor doesn't exist on the pricing page, so the link navigates to `/pricing` without scrolling to the calculator.
**Why it happens:** ROICalculator section currently has no `id` attribute.
**How to avoid:** Add `id="calculator"` to the outer section in `src/app/(public)/pricing/page.js` as a Wave 0 task.

### Pitfall 6: Audio Files Not Present in /public/audio/
**What goes wrong:** AudioDemoSection imports `demo-emergency.mp3` and `demo-routine.mp3` which don't exist yet, causing audio elements to show a broken state.
**Why it happens:** Recordings are produced outside the codebase (staging Gemini agent).
**How to avoid:** Add a placeholder stub (copy `demo-intro.mp3` to `demo-emergency.mp3` + `demo-routine.mp3`) for development. Real files replace stubs before phase verification. Document this as a human dependency in the plan.

### Pitfall 7: Landing-sections.test.js Asserts Exact page.js Whitespace
**What goes wrong:** Test at line 79 matches `/<HowItWorksSection[^>]*\/>\s*<BeyondReceptionistSection[^>]*\/>\s*<FeaturesCarousel[^>]*\/>/m` — this is a very specific whitespace regex that will fail when page.js changes.
**Why it happens:** Phase 47 test was regex-matching exact component order in page.js source.
**How to avoid:** Replace this test with the new component order assertion in Wave 0 test updates.
