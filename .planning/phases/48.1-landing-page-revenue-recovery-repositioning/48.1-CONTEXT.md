# Phase 48.1: Landing Page Revenue-Recovery Repositioning - Context

**Gathered:** 2026-04-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Restructure the public landing page at `src/app/(public)/page.js` so a first-time visitor grasps the "revenue being lost to missed calls" value proposition within 8 seconds, hears real call samples within one scroll, and trusts Voco integrates with their existing CRM. Section count drops from 10+ to 7 content sections (plus FinalCTA ribbon). Mental model shifts from "adopting a SaaS platform" to "hiring an AI dispatcher."

Scope is strictly limited to public landing surface. Zero dashboard changes. Zero backend changes. Zero new API routes. The only out-of-scope-but-related touch is an anchor link from the landing into the existing `ROICalculator` on `/pricing`.

Target audience: mixed US + Singapore home service business owners (plumbers, HVAC, electricians, handymen). Primary narrative arc is US-weighted; Singapore-specific language story survives as supporting detail, not a featured pillar.

</domain>

<decisions>
## Implementation Decisions

### Section lineup and order
- **D-01:** Target: exactly 7 content sections plus `FinalCTASection` as closing ribbon.
- **D-02:** Cut entirely from page (import and render removed): `SocialProofSection`, `HowItWorksSection`, `HeroDemoBlock` (the input-driven hero demo), `BeyondReceptionistSection`.
- **D-03:** Vertical order: Hero → Audio demo → Integrations strip → Cost-of-Silence stat block → FeaturesCarousel-4 → You Stay in Control → FAQ → FinalCTA. Narrative arc: pain → proof → trust → mechanism → close.
- **D-04:** `ScrollLinePath` wraps the middle block only: Integrations → Stat block → FeaturesCarousel-4 → You Stay in Control. Preserves Phase 11/13 visual identity; protects CLS budget by not spanning dynamically-imported hero and tail sections.

### Cost-of-Silence criterion
- **D-05:** No new interactive calculator on the landing page. Replace with a static stat block — big `$260,400/year lost` treatment (pattern already proven in current `PracticalObjectionsGrid` OBJ-03 card), headline "The math isn't close", and a `Calculate yours →` link that deep-links to `/pricing#calculator`.
- **D-06:** `ROICalculator` on `/pricing` stays untouched. Dropdowns preserved. No refactor, no slider conversion. This is an important non-change — avoids duplication and keeps landing-page JS budget minimal.
- **D-07:** Scope override flagged: this deviates from ROADMAP criterion #3 letter ("2-slider calculator on landing"). Satisfied in spirit via the `/pricing` link plus the static stat block. Rationale: research reports prioritized page brevity, and self-quantification is still one click away.

### Audio demo
- **D-08:** 2 real call samples: Emergency + Routine. High-ticket scenario is cut to stay tight (highest-volume triage split covered; high-ticket deferred to FAQ or future phase).
- **D-09:** Samples are scripted and newly recorded using the production Gemini voice agent in a staging environment. Not real anonymized customer calls — maximum control over clarity and what each sample demonstrates.
- **D-10:** UI pattern: tabbed player with 2 tabs (Emergency / Routine), single persistent `AudioPlayerCard`-style component. Tab switch swaps the audio source and transcript. Reuses `AudioPlayerCard` pattern introduced in Phase 47.
- **D-11:** Synchronized transcript panel rendered beside or below the player. Active line is highlighted as the audio plays. Accessibility (WCAG) + support for quiet environments where audio cannot play.

### You Stay in Control consolidation
- **D-12:** Section headline: `You Stay in Control`. Sub-headline: `Smart defaults. Your overrides. Two minutes if you want them — zero if you don't.` The sub-headline does dual duty — signals control AND defuses the setup-friction concern.
- **D-13:** 3 cards organized as the "three dimensions of control":
  - **Card 1 (WHO):** VIP whitelist. Promotes `tenants.vip_numbers` / `leads.is_vip` feature from Phase 46.
  - **Card 2 (WHEN):** Working hours + call forwarding schedule. Promotes `tenants.working_hours` + `tenants.call_forwarding_schedule`.
  - **Card 3 (WHAT):** Owner visibility. Compressed version of current `OBJ-05` card (escalation chain + recording + transcripts).
- **D-14:** Copy rules: outcome-led with character scenarios. Must use named scenarios like "Mary calls about her quarterly HVAC check", "Tuesday afternoons", "Sundays are yours". Forbidden vocabulary: `configure`, `settings`, `dashboard`, `routing rules`, `options panel`. Required framing: what *happens* for the owner, not what the owner *sets up*.
- **D-15:** Every card ends with an italic micro-proof reassurance line naming setup cost. Examples: `Takes 20 seconds per number. Skip it if you want — nothing breaks.` / `Pre-set for nights and weekends. Change anytime — or don't.` / `No setup. Just on.` This is a drumbeat, not a claim.
- **D-16:** Visual treatment: reuse existing `CARD_CLS` pattern from `PracticalObjectionsGrid` + existing `STAT_CHIP_CLS` pattern for the "Pre-set ✓" / "Takes 20 seconds" / "Zero setup" chips. No new component styles required.
- **D-20:** Section structure: identity framing (H2 + sub + 3-paragraph intro drawn from current `IdentitySection` copy) → 3 cards in a 2-column grid on desktop (single column at 375px) → closing dark-background pull quote `"You set the rules. Voco follows them."` on `bg-[#1C1412]` (pattern from current `OwnerControlPullQuote`).

### Displaced objection cards — disposition
- **D-17:** `OBJ-04` (5-minute setup) — dissolved. Its message moves into (a) the italic micro-proof line on every Stay-in-Control card, and (b) hero supporting copy under the CTA. No standalone card. Absorbed into the ambient drumbeat.
- **D-18:** `OBJ-09` (speaks your trade) — relocated to `FeaturesCarousel-4` as Pillar 3. Natural home — it is a feature.
- **D-19:** `OBJ-02` (sound robotic), `OBJ-03` (cost math), `OBJ-08` (before/after) — deleted from `PracticalObjectionsGrid`. Their jobs are done by the Audio demo section (OBJ-02), the static Cost-of-Silence stat block (OBJ-03), and implicitly by the Audio + Stat pair (OBJ-08).
- **D-21:** `IdentitySection`, `OwnerControlPullQuote`, `BeyondReceptionistSection` — their copy is absorbed into the You Stay in Control section (identity framing at top, pull quote at bottom, "more than receptionist" implicit in the section headline). Standalone components deleted.

### Hero
- **D-22:** Hero H1: `Stop losing $1,000+ every time you miss a call.` Exact roadmap example. Loss-framed, dollar-specific, declarative, uses the contractor's own vocabulary ("miss a call"). Do not reword.
- **D-23:** Hero sub (smaller, below H1): `Voco AI answers, triages, and books every call — in under 1 ring.` Mechanism in 12 words. Single use of "Voco AI" (disambiguation — see D-27).
- **D-24:** Hero supporting micro-proof line under the CTA buttons: `Answers in English, Spanish, Mandarin, Malay, and 70+ more. 5-minute setup. No installs.` This line carries both the Singapore/multilingual story (D-33) and the OBJ-04 setup reassurance (D-17).
- **D-25:** Primary CTA: `Hear Voco in action`. Anchor link to the Audio demo section. Low-friction proof-first path per ROADMAP criterion #2.
- **D-26:** Secondary CTA: `Start 5-minute setup →` rendered as a link-style button (smaller than primary). Links to the existing signup flow. For warm visitors who do not need more proof.

### Voco AI naming
- **D-27:** "Voco AI" naming scope is strictly limited:
  - Page `<title>` tag
  - `og:title` and Twitter card titles
  - Hero H1 and hero sub (single use per D-23)
  - Footer "About Voco AI" or similar link text where disambiguation helps
  Everywhere else on the landing — nav logo, body copy, FAQ, FinalCTA, all sections — remains `Voco`. Brand identity preserved. Codebase internal naming unchanged per ROADMAP.

### Integrations strip
- **D-28:** Placement: 3rd section on the page, between Audio demo and the Cost-of-Silence stat block. Satisfies ROADMAP criterion #5 ("above the fold or within first scroll").
- **D-29:** Visual: horizontal row of 6 integrations on desktop.
  - **Live (full-color logos, green chip):** Google Calendar, Outlook
  - **Coming soon (grayscale logos, muted chip):** Jobber, Housecall Pro, ServiceTitan, Zapier
  - Section headline (final copy during planning — working title): `Works with what you already use`
- **D-30:** Dimensions: ~200px desktop height. At 375px, logos wrap into a 2-row grid (3 + 3 or 2+2+2 — planner decides). ScrollLinePath passes through this section.

### FeaturesCarousel-4 pillars
- **D-31:** 4 pillars in this exact order (narrative arc: coverage → conversion → comprehension → safety net):
  1. `24/7 AI Answering`
  2. `Real-Time Calendar Booking`
  3. `Speaks Your Trade` (absorbs the former `OBJ-09` card per D-18)
  4. `Automated Lead Recovery`
- **D-32:** Features cut from the carousel (existed in the original 9, not retained): `Custom Pickup Rules` (now covered by You Stay in Control), `Lead Capture & CRM` (implied by Real-Time Booking), `Post-Call SMS & Notifications` (absorbed into You Stay in Control Card 3), `Invoicing & Estimates` (different product area — Phases 33-35), `Call Analytics & Dashboard` (dashboard product, not an inbound-call pillar), `70+ Languages` (survives outside pillars — see D-33).
- **D-33:** The 70+ Languages story survives in three non-pillar locations for Singapore/multilingual visitor coverage:
  1. Hero supporting micro-proof line (D-24)
  2. Inside Pillar 3 "Speaks Your Trade" body copy — explicitly mention Singlish and code-switching
  3. A dedicated FAQ entry covering the full language list

## Existing Code Insights

### Reusable Assets
- `AudioPlayerCard` (Phase 47): reuse for the Audio demo section's tabbed player. Extend with tab-switching logic and synchronized transcript. No new component file if possible.
- `AnimatedSection`, `AnimatedStagger`, `AnimatedItem`: required wrappers for all new content to honor `useReducedMotion`.
- `CARD_CLS` / `STAT_CHIP_CLS` constants in `PracticalObjectionsGrid.jsx`: reuse for Stay-in-Control cards + their reassurance chips. Avoid defining new card styles.
- `ScrollLinePath`: existing copper sine-wave component — wrap the middle block per D-04. Do not modify the component itself.
- `ROICalculator` (`/pricing`): landing links to `#calculator` anchor; verify that anchor exists or add it via a wrapping `<section id="calculator">` on pricing.
- Dynamic-import pattern in current `src/app/(public)/page.js`: every below-the-fold section uses `dynamic()` with a bespoke loading skeleton that matches target dimensions to prevent CLS. New sections MUST follow this pattern.

### Established Patterns
- Copper accent color `#F97316` on a stone palette (`#FAFAF9`, `#F5F5F4`, `#0F172A` text). All new sections must use these tokens — no new hex values.
- Dark-background sections use `#1C1412` with warm radial gradients (see `OwnerControlPullQuote`, `FinalCTASection`). The Stay-in-Control closing pull quote follows this pattern.
- Typography: `text-[14px]` for chips/labels, `text-[15px]` body, `text-[3xl] md:text-[2.25rem] font-semibold` for section H2s.
- 375px mobile-first responsive: every new section must single-column stack and verify no horizontal scroll. Use Chrome DevTools 375px preview during implementation.
- Performance contract (Phase 13, restated by 48.1 criterion #8): LCP < 2.5s, CLS < 0.1, INP < 200ms. Every new section with dynamic import must have a skeleton with explicit height reservation.

### Integration Points
- `src/app/(public)/page.js` is the single rewrite target — import statements, dynamic import block, and `HomePage()` return structure all change.
- `src/app/(public)/pricing/page.js` must have a `#calculator` anchor (likely needs a one-line change — wrap `<ROICalculator />` in `<section id="calculator">` if not already present).
- No backend changes. No API changes. No new database columns. No new migrations.
- No dashboard route changes. The Stay-in-Control section is purely marketing copy about existing dashboard features; it does not link into the dashboard routes beyond the signup CTA.
