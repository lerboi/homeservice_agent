## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | jest 29.x (node environment, source-string regex pattern — NO React rendering) |
| **Config file** | `jest.config.js` (project root) |
| **Quick run command** | `npm test -- --testPathPattern=landing` |
| **Full suite command** | `npm test` |
| **Estimated runtime** | ~8 seconds (quick); ~45 seconds (full suite) |
| **Test file (primary)** | `tests/unit/landing-sections.test.js` — updated in Plan 01 Task 1 |

Pattern: tests use `fs.readFileSync` to read component source files and regex-match against their JSX text. This means:
- No jsdom, no React renderer, no @testing-library
- Grep-verifiable criteria in each task's `<acceptance_criteria>` map directly to the Jest assertions
- Test failures are always "string X missing from file Y" or "regex R didn't match"
