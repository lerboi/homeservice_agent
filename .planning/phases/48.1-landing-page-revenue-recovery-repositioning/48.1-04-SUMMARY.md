## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] ssr:false not allowed in Server Component dynamic() in Next.js 16**
- **Found during:** Task 4 (dev server start) — Turbopack threw build error `ssr: false is not allowed with next/dynamic in Server Components`
- **Issue:** Next.js 16 (Turbopack) enforces that `ssr: false` cannot be used in dynamic imports from Server Components. This is a breaking change from earlier Next.js behavior.
- **Fix:** Removed `ssr: false` from the AudioDemoSection dynamic import in page.js. AudioDemoSection already declares `'use client'` which is the correct mechanism for preventing server rendering of browser-only audio APIs in Next.js App Router.
- **Files modified:** src/app/(public)/page.js
- **Commit:** `05ea79a`

**2. [Rule 1 - Bug] Test ordering assertion matched dynamic import declaration instead of JSX render tag**
- **Found during:** Test run after Task 3 — `indexOf('YouStayInControlSection')` found the dynamic import declaration at line 64 (before `<ScrollLinePath>` at line 109), causing ordering assertion to fail
- **Issue:** The Plan 01 test used bare string `indexOf('YouStayInControlSection')` which matches the first occurrence — the `const YouStayInControlSection = dynamic(...)` declaration — rather than the JSX render tag `<YouStayInControlSection />` in the return body
- **Fix:** Updated test to use `</ScrollLinePath>` as the boundary marker and `<YouStayInControlSection` (with angle bracket) as the JSX render anchor. This correctly tests that the render tag appears after the closing ScrollLinePath tag.
- **Files modified:** tests/unit/landing-sections.test.js
- **Commit:** `0b22255`
