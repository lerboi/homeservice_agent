## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px | 400 | 1.5 |
| Label / Caption | 13px | 700 | 1.4 |
| Heading (email) | 22px | 700 | 1.3 |
| Banner text | 14px (text-sm) | 400 | 1.4 |

Notes:
- Two weights only: 400 (body, banner prose) and 700 (labels, headings, emphasis). The 600 weight is not used in this phase.
- Banner strong/bold inline text uses weight 700 (same `<strong>` pattern as ImpersonationBanner).
- Email template sizes match `NewLeadEmail.jsx` established pattern (13px labels, 15px values, 22px heading).
- Dashboard UI uses Tailwind `text-sm` (14px) for banner copy, matching existing sidebar and breadcrumb patterns.
- 13px (email label) and 14px (banner) are intentionally distinct tokens. Email labels render in an HTML email context (uppercase, small-print detail rows); banner text renders in browser UI prose. Merging them would break alignment with the existing `NewLeadEmail.jsx` pattern. Both sizes are retained.

## Component Inventory

### BillingWarningBanner (new component)

Location: `src/app/dashboard/BillingWarningBanner.js`

Placement: Inserted above the main layout wrapper in `src/app/dashboard/layout.js`, below `ImpersonationBanner` (same sticky z-40 pattern, stacks when both present).

Visual spec:
- Height: 44px (h-11)
- Background: `bg-amber-50`
- Border: `border-b border-amber-300`
- Layout: `flex items-center justify-between px-4 lg:px-8`
- Icon: `AlertCircle` (lucide), `h-4 w-4`, `text-amber-800`
- Left copy: `text-sm text-amber-800` — "Your payment failed — update your payment method within **X days** to avoid service interruption."
- Right element: Link to Stripe Customer Portal — styled as `text-sm font-medium border border-amber-400 text-amber-800 px-3 py-1 rounded-md hover:bg-amber-100 transition-colors` (matches ImpersonationBanner exit button pattern)
- Right link label: "Update Payment Method"
- Visible condition: subscription status is `past_due` AND grace period has not expired (days remaining > 0)
- Dismissible: No — persistent until status resolves

### TrialReminderEmail (new React Email template)

Location: `src/emails/TrialReminderEmail.jsx`

Follows `NewLeadEmail.jsx` structure exactly:
- Same `bodyStyle` (warmSurface `#F5F5F4` background, 40px vertical padding)
- Same `containerStyle` (white card, 560px max-width, 12px border-radius, shadow)
- Header: navy (`#0F172A`) background, white Voco logotype text
- Heading: 22px, weight 700, navy, line-height 1.3
- Body text: 15px, weight 400, muted (`#475569`)
- Detail rows: label 13px / 700 / uppercase / `#475569`, value 15px / navy
- CTA button: `#C2410C` background, white text, 15px / 700, `12px 28px` padding, 8px border-radius, label "Upgrade Now"
- Footer: warmSurface background, 13px / `#94a3b8` footer text

Content varies by trigger:
- Day 7 reminder: heading "7 days into your trial — here's how it's going", body shows calls_used / calls_limit usage stat
- Day 12 reminder: heading "2 days left in your trial — keep the calls coming", urgency copy + usage stat + upgrade CTA
- trial_will_end webhook (3 days out): heading "Your trial ends in 3 days", identical structure to day 12 email

### PaymentFailedEmail (new React Email template)

Location: `src/emails/PaymentFailedEmail.jsx`

Follows same `NewLeadEmail.jsx` structure. Differences:
- Header background: amber-700 (`#b45309`) — distinct from navy to signal financial urgency
- Heading: "Your payment didn't go through"
- Body: "Hey [name], your payment didn't go through. Your service continues for 3 more days — update your card now to avoid any interruption."
- CTA button label: "Update Payment Method"
- CTA href: Stripe Customer Portal URL passed as prop

### Cron Route (no UI — backend only)

`/api/cron/trial-reminders` — no UI elements. Not in this contract.

### Middleware Gate (no UI — redirect only)

Expired/cancelled tenants redirected to `/billing/upgrade` — page lives in Phase 25. This phase only adds the middleware check.

## Accessibility

- BillingWarningBanner: include `role="alert"` so screen readers announce it on mount
- AlertCircle icon: `aria-hidden="true"` — decorative; copy conveys the meaning
- "Update Payment Method" link: sufficient contrast (amber-800 on amber-50 passes WCAG AA for text)
- Email templates: include `lang="en"` on `<Html>` (already present in NewLeadEmail pattern)
- SMS copy: plain text only, no special requirements
