## Bugs fixed during UAT (not in original plans)

1. **`supabase-browser.js`** — switched from `createClient` (@supabase/supabase-js) to `createBrowserClient` (@supabase/ssr) so auth tokens are stored in cookies, making them readable server-side. This fixed the "Unauthorized" error on the onboarding API route.

2. **`src/middleware.js`** — created auth middleware that redirects unauthenticated users hitting `/onboarding` or `/dashboard` to `/auth/signin?next=<path>`. Previously there was no protection.

3. **`src/app/auth/signin/page.js`** — added `useSearchParams` to read `?next=` param, redirects there after sign-in (both Google OAuth and email/password).

4. **`src/app/auth/callback/route.js`** — passes `next` param through so Google OAuth redirects to correct destination after callback.

5. **`supabase/migrations/001_initial_schema.sql`** — added `owner_id uuid UNIQUE NOT NULL` column to tenants table. Fixed RLS policies to use `owner_id = auth.uid()` instead of JWT `tenant_id` claim (which doesn't exist during onboarding).

6. **`supabase/migrations/002_onboarding_triage.sql`** — fixed services RLS policy to use `owner_id = auth.uid()` subquery.

7. **Wizard Step 3 simplified** — removed SMS OTP verification (requires Twilio setup) and Retell number provisioning. Now just collects phone/email directly and saves them. `sms-confirm/route.js` updated to skip `verifyOtp` call.

8. **`onboarding/complete/page.js`** — simplified to a plain "You're all set" completion page (no Retell number display, no test call button) since provisioning is skipped for now.
