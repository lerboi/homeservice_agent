## Spacing Scale

Declared values (multiples of 4 only — Tailwind defaults map 1 unit = 4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (1) | Icon gaps, badge padding, inline chip spacing |
| sm | 8px (2) | Compact stacked items, icon-to-label gaps |
| md | 16px (4) | Default padding inside cards and form fields |
| lg | 24px (6) | Between form groups within a step, card internal sections |
| xl | 32px (8) | Between wizard steps' primary content blocks |
| 2xl | 48px (12) | Vertical padding of wizard container from viewport edge |
| 3xl | 64px (16) | Page-level top/bottom breathing room on desktop |

**Exceptions:**
- Touch targets: minimum 44px height for all interactive controls (buttons, dropdown triggers, OTP input cells). Use `min-h-11` (44px) where component default is smaller.
- Progress bar: fixed height 4px — does not follow scale, intentional compact indicator.
- Tone preset cards (Step 1): minimum 80px height to accommodate icon + label + description line.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px (text-3xl) | 700 (bold) | 1.2 | Wizard step headings ("Set up your AI receptionist") |
| Heading | 20px (text-xl) | 600 (semibold) | 1.3 | Card section titles, post-activation headline |
| Body | 16px (text-base) | 400 (normal) | 1.5 | Form labels, helper text, description copy |
| Label/Small | 14px (text-sm) | 500 (medium) | 1.4 | Input sub-labels, service tag pills, badge text, step counter |

**Rules:**
- Only 4 sizes declared above — no intermediate sizes.
- Only 3 weights in use: 400, 500, 600/700. Never use 300 (too light for SME audience) or 800+.
- Minimum body text 16px to prevent iOS auto-zoom on form inputs (source: ui-ux-pro-max `readable-font-size`).
- All copy in `next-intl` translation keys — no hardcoded English strings (source: CONTEXT.md established patterns).

## Color

Palette: neutral-dominant SaaS with blue accent. Professional trust signal for SME tradespeople.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `gray-50` (light) | Page background, wizard container background, input backgrounds |
| Secondary (30%) | `#f8fafc` (slate-50) / `#f1f5f9` (slate-100) | Cards, step containers, trade template option cards, service list rows |
| Accent (10%) | `#2563eb` (blue-600) | Reserved for specific elements only — see list below |
| Destructive | `#dc2626` (red-600) | Remove service button, delete action confirmations only |
| Success | `#16a34a` (green-600) | Test call success state ("Your AI is live!"), OTP verified checkmark |
| Text primary | `#0f172a` (slate-900) | All body text, headings |
| Text secondary | `#64748b` (slate-500) | Helper text, placeholder-adjacent descriptions, step subtitles |
| Border | `#e2e8f0` (slate-200) | Card borders, input borders, dividers |

**Accent (`#2563eb` blue-600) reserved for:**
1. Primary CTA buttons (the single forward-action button per wizard step)
2. Active/selected state on tone preset cards and trade template cards
3. Active step indicator dot in progress bar
4. "Test your AI" button (the activation moment)
5. Links (underlined) in helper text

**Accent is NOT used for:** secondary buttons, service tag pills, navigation items, icons in non-selected state, error states, or decorative elements.

**Contrast compliance (source: ui-ux-pro-max `color-accessible-pairs`):**
- `#2563eb` on `#ffffff`: 4.7:1 — passes WCAG AA for normal text.
- `#0f172a` on `#ffffff`: 17:1 — passes WCAG AAA.
- `#64748b` on `#ffffff`: 4.6:1 — passes WCAG AA for normal text.
- White text on `#2563eb`: 4.7:1 — passes WCAG AA.
- White text on `#dc2626`: 4.6:1 — passes WCAG AA.

**Animation rules (source: ui-ux-pro-max `duration-timing`):**
- Wizard step transitions: slide-in from right (forward), slide-in from left (back). Duration 200ms, ease-out.
- Button press scale: `scale-95` at 80ms, restore at 150ms. Use `transition-transform duration-150`.
- Card selection: border color change at 150ms ease-out. No layout shift.
- Toast entrance: slide-up 200ms ease-out. Auto-dismiss 4000ms.
- Loading spinner: standard shadcn `Loader2` icon with `animate-spin`.
- Progress bar fill: `transition-all duration-300` on width change between steps.

## Layout

### Wizard Layout (Steps 1–3 and Activation)

```
┌──────────────────────────────────────────────────────┐
│  Logo (top-left, 24px height)         Step N of 3    │
├──────────────────────────────────────────────────────┤
│  Progress bar (4px, full width, blue fill)           │
├──────────────────────────────────────────────────────┤
│                                                      │
│          ┌────────────────────────────┐              │
│          │  Step heading (Display)    │              │
│          │  Step subtext (Body/sm)    │              │
│          │                            │              │
│          │  [Step content area]       │              │
│          │                            │              │
│          │  [Back]          [Continue]│              │
│          └────────────────────────────┘              │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- Wizard card: `max-w-lg` (512px) centered, `mx-auto`, `px-6 py-8`
- Full-page background: `slate-50`
- Card background: `white`, `rounded-xl`, `shadow-sm`, `border border-slate-200`
- Mobile: card fills viewport with `rounded-none`, `shadow-none`, `border-none`
- Back button: text-only, left-aligned, `slate-500` text. Hidden on Step 1.
- Continue/primary CTA: full-width on mobile, fixed width (`w-40`) on desktop, right-aligned.

### Service Manager Layout (Post-test-call)

- Page layout: standard dashboard shell (sidebar nav + main content area). Sidebar deferred to Phase 4 — for Phase 2, render as single-column page with breadcrumb nav.
- Service list: data table with columns: Service Name | Urgency Tag | Actions
- Rows: `h-14` (56px) minimum for comfortable touch targets.
- Tag dropdown: inline, `w-36`, positioned in Urgency Tag column.
- "Add service" button: secondary button, top-right of table header row.

## Accessibility Contract

Minimum requirements (source: ui-ux-pro-max Priority 1 + WCAG AA):

- All form inputs have a `<label>` with `htmlFor` — no placeholder-only labels.
- All icon-only buttons have `aria-label` (e.g., remove service "×" button).
- Error messages use `role="alert"` or `aria-live="polite"` for screen reader announcement.
- Wizard step transitions: `aria-live="polite"` region on main content for step changes.
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax`.
- Focus management: on wizard step advance, focus moves to the step heading `h1`.
- Keyboard navigation: all interactive elements reachable via Tab, actionable via Enter/Space.
- Touch targets: minimum 44px height on all interactive controls (enforced via `min-h-11`).
- Color: urgency tags use color + text label, never color alone (e.g., "Emergency" in red-100, not just a red dot).
