  <behavior>
    Layer 1 (runKeywordClassifier):
    - "my basement is flooding" -> { result: 'emergency', confident: true, matched: pattern }
    - "gas smell in the kitchen" -> { result: 'emergency', confident: true, matched: pattern }
    - "pipe burst" -> { result: 'emergency', confident: true }
    - "carbon monoxide" -> { result: 'emergency', confident: true }
    - "electrical fire" -> { result: 'emergency', confident: true }
    - "I need a quote for next month" -> { result: 'routine', confident: true }
    - "not urgent, whenever you can" -> { result: 'routine', confident: true }
    - "my faucet is dripping a little" -> { result: 'routine', confident: false } (no keyword match)
    - "" (empty) -> { result: 'routine', confident: false }
    - "hi" (< 10 chars) -> { result: 'routine', confident: false }

    Layer 2 (runLLMScorer):
    - Calls OpenAI chat.completions.create with model 'gpt-4o-mini', temperature 0, JSON response format
    - Returns { urgency: 'emergency'|'routine'|'high_ticket', confidence: 'high'|'medium'|'low', reason: string }
    - On parse error returns { urgency: 'routine', confidence: 'low', reason: 'parse error' }

    Layer 3 (applyOwnerRules):
    - Given base classification and tenant_id, queries services table
    - If transcript mentions a service tagged 'emergency' by owner, escalates to emergency
    - Never downgrades: if base is 'emergency', result stays 'emergency' regardless of owner tags
    - If no services found for tenant, returns base classification unchanged

    Classifier (classifyCall):
    - Runs Layer 1. If confident, skips Layer 2, applies Layer 3, returns.
    - If Layer 1 not confident, runs Layer 2, then Layer 3, returns.
    - Result shape: { urgency: string, confidence: string, layer: string, reason?: string }
    - Empty/short transcript: returns { urgency: 'routine', confidence: 'low', layer: 'layer1' } without calling LLM
  </behavior>

**RED phase — Write tests first:**

1. Create `tests/__mocks__/openai.js` — Mock OpenAI SDK that returns configurable JSON responses via a shared mock function.

2. Create `tests/triage/layer1.test.js` with tests for all Layer 1 behaviors listed above (~10 test cases).

3. Create `tests/triage/classifier.test.js` with tests:
   - Layer 1 confident emergency -> skips LLM, calls Layer 3, returns emergency
   - Layer 1 confident routine -> skips LLM, calls Layer 3, returns routine
   - Layer 1 not confident -> calls LLM (Layer 2), then Layer 3
   - Layer 3 escalates routine to emergency when owner has emergency-tagged service
   - Layer 3 does NOT downgrade emergency to routine
   - Empty transcript returns routine without calling LLM
   - LLM parse error falls back to routine/low

All tests use `jest.unstable_mockModule` for ES module mocking, following the mutable shared mock object pattern established in Phase 1.

**GREEN phase — Implement:**

1. `src/lib/triage/layer1-keywords.js` — Exact implementation from RESEARCH.md code example. Emergency patterns: flooding, gas smell/leak, no heat/hot water, sewer backup, pipe burst, electrical fire/sparks, carbon monoxide, "right now"/"emergency"/"urgent". Routine patterns: quote/estimate/next week/next month, not urgent/whenever/no rush.

2. `src/lib/triage/layer2-llm.js` — Uses `openai` package. `new OpenAI({ apiKey: process.env.OPENAI_API_KEY })`. Model: `gpt-4o-mini`. System prompt classifies home service calls into emergency/routine/high_ticket with JSON response format. Max tokens: 100. Temperature: 0. Wraps parse in try/catch.

3. `src/lib/triage/layer3-rules.js` — Queries `supabase.from('services').select('name, urgency_tag').eq('tenant_id', tenant_id).eq('is_active', true)`. Compares extracted service type (from Layer 1/2 result) against owner's service names using case-insensitive match. Severity ranking: emergency=3, high_ticket=2, routine=1. Returns max(base, owner_tag).

4. `src/lib/triage/classifier.js` — Orchestrator: `classifyCall({ transcript, tenant_id, detected_service })`. Guard for empty/short transcript. Calls layers in sequence per behavior spec. Returns unified result object with `urgency`, `confidence`, `layer`, `reason`.
