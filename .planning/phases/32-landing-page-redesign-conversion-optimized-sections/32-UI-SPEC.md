| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Card inner element spacing, badge padding |
| md | 16px | Card content padding, icon container size delta, section subtitle to heading gap |
| lg | 24px | Card padding (p-6), gap between section heading and content |
| xl | 32px | Card internal padding on desktop (p-8), section heading margin-bottom (mb-16 = 64px, see 3xl) |

- Sticky card folder-stack offsets: 80px, 130px, 180px, 230px (per D-13 — not pure 8-point scale, but driven by visual peeking requirement of ~50px per prior card)

- Sticky card inter-card scroll gap: `marginBottom: 25vh` (viewport-relative, drives scroll-trigger timing)
- 44px minimum touch target for mobile interactive elements (WCAG 2.5.5)

## Typography

This phase spans two surface contexts: dark hero (`#050505`) and light sections (`#FAFAF9` / `#F5F5F4`). Typography values are consistent across both contexts; only color changes.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 40px / 52px / 60px (responsive) | 600 (semibold) | 1.1 | Hero h1 with RotatingText |

| Subheading | 24px / 30px / 44px (responsive) | 700 (bold) | 1.15 | How It Works card titles |

| Small body | 16px | 400 (regular) | 1.6 | How It Works extended text, Feature card descriptions |

| Caption | 11px / 12px | 600 (semibold) | 1.0 | Metric tags, icon badge micro labels, uppercase trackers |

Tailwind class mapping:
- Display: `text-[2.5rem] md:text-[3.25rem] lg:text-[3.75rem] font-semibold leading-[1.1] tracking-tight`
- Heading: `text-3xl md:text-4xl font-semibold tracking-tight`

- Body: `text-lg md:text-xl leading-relaxed`
- Small body: `text-base leading-relaxed`
- Label: `text-sm font-semibold` (section eyebrows add `tracking-[0.15em] uppercase`)

Weights used: 400 (regular), 600 (semibold), 700 (bold). The bold weight is carried forward from existing How It Works card titles — changing it would break visual hierarchy between card titles and section headings.

## Visual Focal Point

**Features section:** The full-width 70+ Languages hero card at the top of the grid is the primary focal point. It occupies 100% width while all other cards share a 2-column grid — this size contrast creates immediate visual hierarchy. The animated language bubbles/flags visual draws the eye first before the visitor scans the 6 feature cards below.

**How It Works section:** The folder-stack cascading effect is the focal point. As the user scrolls, cards stack with visible peek-through of prior step badges/headers — creating a visual "complete flow" impression at the end of the scroll sequence when all 4 step badges are visible like folder tabs.

## Animation Contract

All animations must be **transform/opacity-only** per D-17. No backdrop-blur on large surfaces. No parallax. No canvas/Lottie.

| Element | Animation | Duration | Easing | Trigger |
|---------|-----------|----------|--------|---------|
| Section headings | Fade-up (AnimatedSection) | 200ms | `[0.22, 1, 0.36, 1]` | Scroll into viewport |
| Feature cards | Staggered fade-up (AnimatedStagger + AnimatedItem) | 200ms per card, 50ms stagger | `[0.22, 1, 0.36, 1]` | Scroll into viewport |
| Feature card hover | Border glow + lift | 200ms | `ease` | Mouse hover |
| Language hero visual bubbles | Gentle float (translateY oscillation) | 3s loop | `ease-in-out` | Always running (CSS keyframes) |
| Clock visual arc | Stroke-dashoffset rotation | 8s loop | `linear` | Always running (CSS keyframes, carried from existing) |
| How It Works sticky cards | Position sticky + scroll-driven stacking | n/a | Native scroll | Scroll position |
| How It Works progress dots | Width/opacity transition | 300ms | `ease` | IntersectionObserver activeIndex change |
| RotatingText | Per-character spring animation | Spring damping 25, stiffness 300 | framer-motion spring | 3000ms interval |
| RotatingText width | Container width transition | 200ms | `ease-in-out` | currentIndex change |

### Performance budget

- No `backdrop-filter: blur()` on any surface in this phase
- No `position: fixed` animations (only `position: sticky` for cards)
- All hover effects use `transform` and `box-shadow` only (GPU-composited)
- Feature card micro visuals: pure SVG inline or CSS, no `<canvas>`, no Lottie, no imported images
- `will-change` only on elements that actually animate (not preemptively applied)

Source: CONTEXT.md D-17, D-20; existing `AnimatedSection.jsx` patterns

## Accessibility Contract

- All feature card icons have `aria-hidden="true"` (decorative — titles provide context)
- Micro SVG visuals have `aria-hidden="true"` (decorative)
- Section headings are `<h2>` elements (correct heading hierarchy under page `<h1>` in hero)
- Feature card titles are `<h3>` elements (nested under section `<h2>`)
- How It Works step titles are `<h3>` elements
- How It Works progress dots: `aria-hidden="true"` (decorative, step content is accessible via scroll)
- How It Works background decorative numbers: `select-none pointer-events-none` (already applied)
- Color contrast: `#0F172A` on `#FAFAF9` = 15.4:1 (exceeds AAA). `#475569` on `#FAFAF9` = 5.8:1 (exceeds AA). `#64748B` on `#F5F5F4` = 4.6:1 (passes AA for body text 16px+)
- Reduced motion: `AnimatedSection` / `AnimatedStagger` / `AnimatedItem` already handle `useReducedMotion` — returns static layout. How It Works `IntersectionObserver` still functions for progress dots (no animation dependency).
- Feature card micro visual CSS keyframe animations: wrap in `@media (prefers-reduced-motion: no-preference)` to disable for reduced motion users
- Mobile scroll-snap: include `scroll-behavior: smooth` only if `prefers-reduced-motion` is not `reduce`

Source: Existing `AnimatedSection.jsx` reduced motion handling, WCAG 2.1 AA compliance
