    **Key insight:** ScrollLinePath is already self-adjusting. It measures `offsetHeight`, uses `getElementById` for section boundaries, and rebuilds the path on resize. Since FeaturesGrid keeps `id="features"` and SocialProofSection keeps `id="testimonials"`, the boundary detection will work automatically.

    **What to check/adjust:**

    1. **Remeasure timing:** The current delays (100ms, 1000ms) should be sufficient for the new sections to load via dynamic import. The 4-step HowItWorks with folder-stack is taller, but the section heights stabilize after DOM paint. No change likely needed, but verify.

Since it's a minor visual detail and the section hasn't fundamentally changed its boundary, the +60 offset should still work.

    3. **Wave amplitude:** Currently `Math.min(512 + 120, (w - 48) / 2)`. This is content-width-agnostic and works with any section layout. No change needed.

    **Decision: If no issues found, leave ScrollLinePath unchanged.** The component is already designed to self-adapt to section height changes via dynamic measurement. The comment in CONTEXT.md says "May need path recalculation for new section heights" — but the existing code already does this dynamically.
