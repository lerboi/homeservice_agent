## Page Style Contract

**Critical design decision (D-07):** All five SEO page templates use the **light content style** — NOT the dark landing page hero style. This is a hard constraint from CONTEXT.md.

| Style dimension | Light content pages (this phase) |
|----------------|----------------------------------|
| Page background | `#FFFFFF` (white) primary surface |
| Section alternates | `#F5F5F4` (warm stone) for visual rhythm |
| Text | `#0F172A` headings, `#475569` body |
| Accent | `#F97316` (orange) — CTAs and highlights only |
| Dark sections | Prohibited on SEO content pages — light-only |

Reference pattern: `/about` page sections alternate `#F5F5F4` and `#050505` — SEO pages use only the `#F5F5F4` / `#FFFFFF` variants, never the dark `#050505` sections.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, tag gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section inner padding, card vertical rhythm |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Section vertical padding (py-12) |
| 3xl | 64px | Major section breaks (py-16 md:py-24) |

Exceptions:
- Touch targets (nav links, CTA buttons, accordion triggers): minimum 44px height per WCAG 2.5.5
- OG image canvas: 1200x630px (fixed, not grid-based)
- Content prose max-width: 65ch (optimal reading line length)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 (regular) | 1.6 | Article prose, card descriptions, page copy |
| Label | 14px (0.875rem) | 500 (medium) | 1.4 | Meta text, dates, page-type badges, tags |
| Heading | 24px (1.5rem) | 600 (semibold) | 1.3 | H2 section headings, card titles |
| Display | 36px–48px (2.25rem–3rem) | 600 (semibold) | 1.15 | H1 page titles, hero headings |

Notes:
- All sizes from Inter variable font already loaded in root layout — no new font imports needed.
- Display size is 36px (`text-4xl`) on mobile, 48px (`text-5xl`) on md+ breakpoint.
- Heading size is 24px (`text-2xl`) on mobile, 30px (`text-3xl`) on md+.
- Body prose in blog detail uses `leading-relaxed` (1.625) for reading comfort.
- All font sizes use Tailwind utility classes — no raw `font-size` CSS.

## Page Template Contracts

### Blog Listing (`/blog`)

- Layout: `max-w-6xl mx-auto px-6`
- Grid: 2-col on md, 3-col on lg (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`)
- Card: shadcn `Card` component with `card.base` design token shadow (`bg-white rounded-2xl shadow-[0_1px_3px...] border border-stone-200/60`)
- Card contents: featured image (aspect-ratio 16:9, cover fit), page-type badge (BLOG), title, excerpt (2-line clamp), date (label size 14px)
- Hub page heading: H1 Display (36px / 48px), subheading Body at 18px
- Source: D-01, `src/components/ui/card.jsx`, `src/lib/design-tokens.js`

### Blog Detail (`/blog/[slug]`)

- Layout: two-column with `max-w-2xl` prose center + optional narrow sidebar (or single-column prose if no sidebar content)
- Prose: `max-w-[65ch]` body, 16px at 1.6 line-height, `#475569` color
- Headings within article: H2 at 24px semibold, H3 at 20px semibold — both `#0F172A`
- Featured image: full-width at top, `aspect-[16/9]` with `object-cover`, rounded-xl
- Article header: title (Display 36px/48px), date at 14px label, optional author line at 14px
- Blockquote: left border 3px `#F97316`, `pl-6`, italic body text
- CTA at bottom: full-width copper button (AuthAwareCTA component, `variant="cta"`)
- Source: D-02, existing AnimatedSection pattern

### Persona Pages (`/for/[persona]`)

- Layout: full-width sections, matching about page section alternation (`#FFFFFF` / `#F5F5F4`)
- Section 1 — Hero: trade-specific H1 (Display), subheadline (Body 18px), AuthAwareCTA button + secondary text link "See how it works"
- Section 2 — Pain Points: 3-column icon+text grid on md+, stacked on mobile. Icon: 40px container `rounded-xl bg-[#F97316]/10`, Lucide icon at 20px `text-[#F97316]`
- Section 3 — Feature Highlights: 2-col grid with trade-specific context. Each feature: title (Heading 20px semibold), description (Body), check list items at 14px
- Section 4 — Testimonial / Social Proof: single blockquote with left border `#F97316`, `text-xl` quote, attribution at 14px label
- Section 5 — CTA: matches existing About page CTA section — dark bg `#1C1412` with orange radial gradient, H2 + AuthAwareCTA. Reuse exact pattern from about/page.js CTA section.
- Source: D-03

### Comparison Pages (`/compare/[comparison]`)

- Layout: `max-w-5xl mx-auto px-6`
- Hero: H1 (Display) with comparison framing, subhead (Body 18px)
- Comparison table: shadcn `Table` component. Two columns: "Voco" and competitor name. Row header: feature name at 14px. Voco cells: Lucide `Check` icon 16px `text-[#F97316]` for supported; competitor cells: Lucide `X` icon 16px `text-slate-400` for unsupported; text cells at 14px.
- Table container: `bg-white rounded-2xl border border-stone-200/60 overflow-hidden`
- Voco column header: `bg-[#F97316]/5 font-semibold text-[#F97316]` to visually differentiate
- Verdict section: below table, `bg-[#F5F5F4] rounded-2xl p-8`, H2 (Heading 24px) + Body text + AuthAwareCTA
- Source: D-04, `src/components/ui/table.jsx`

### Glossary Pages (`/glossary/[term]`)

- Layout: `max-w-3xl mx-auto px-6` (narrower — definition content)
- Definition section: term as H1 (Display), definition as Body at 18px with `leading-relaxed`
- FAQ accordion: reuse `FAQSection` accordion pattern from `src/app/(public)/pricing/FAQSection.jsx` — Radix Accordion.Root, `border-b border-stone-200` dividers (light surface — change from `border-white/[0.08]` used in dark pricing context), ChevronDown Lucide icon `text-[#F97316]`
- Accordion trigger: `text-[#0F172A]` (dark) at 16px semibold, min-h 44px
- Accordion content: `text-[#475569]` at 15px, `leading-relaxed`
- Related terms: horizontal wrapping row of pill links below accordion — `rounded-full border border-stone-200 px-3 py-1 text-sm text-[#475569] hover:border-[#F97316] hover:text-[#F97316]`
- Source: D-05, `src/app/(public)/pricing/FAQSection.jsx`

### Integration Pages (`/integrations/[tool]`)

- Layout: `max-w-5xl mx-auto px-6`
- Hero: tool logo + Voco logo with "+" connector, H1 (Display), subhead (Body 18px)
- Integration description: Body prose section, `max-w-2xl`
- Use cases: 2x2 grid of cards on md+. Each card: shadcn Card, Lucide icon, title (Heading 18px), description (Body 14px)
- CTA: matches comparison page verdict section — `bg-[#F5F5F4] rounded-2xl p-8`
- Source: D-06

## Internal Linking Contract

Source: D-21, D-22

- Every hub page (`/blog`, `/for`, `/compare`, `/integrations`, `/glossary`) links to all its children.
- Every child page links back to its hub ("See all blog posts", "See all trades").
- Every child page cross-links to 2-3 siblings in a "Related" section.
- Every page includes AuthAwareCTA ("Start My 5-Minute Setup") as primary conversion.
- No orphan pages — hub pages must be reachable from nav or footer.

**Nav placement:** Hub pages go in the LandingFooter under a new "Resources" column (not in LandingNav — nav stays clean with Home / Pricing / About / Contact).

**Footer Resources column** (new, to be added to LandingFooter.jsx):
- Blog (`/blog`)
- For Plumbers (`/for/plumber`) — seed persona

- Integrations (`/integrations`) — hub

## Animation Contract

Source: CONTEXT.md "Claude's Discretion" — keep minimal on content pages.

| Element | Animation | Spec |
|---------|-----------|------|
| Page sections on scroll | Fade-up via AnimatedSection | duration 200ms, ease `[0.22,1,0.36,1]`, y offset 32px — reuse existing component |
| Card grids | AnimatedStagger + AnimatedItem | stagger 50ms per card |
| FAQ accordion | CSS accordion-down/up | Already in globals.css keyframes — 200ms ease-out |
| CTA button | hover translate-y-0.5 + shadow intensify | 150ms transition — matches AuthAwareCTA pattern |
| Hub page card hover | `hover:-translate-y-0.5 hover:shadow-[0_4px_12px...]` | 200ms — from `card.hover` design token |
| All animations | prefers-reduced-motion respected | AnimatedSection checks useReducedMotion; accordion CSS handles automatically |

No scroll-parallax, no scroll-draw SVG lines, no entrance animations beyond fade-up. Content readability is paramount over visual flourish.

## Accessibility Contract

- All images: descriptive `alt` text — featured images describe scene, logos include brand name
- FAQ accordion: Radix Accordion provides ARIA `role="region"`, `aria-expanded`, keyboard support — no additional ARIA needed
- Comparison table: `<th scope="col">` on column headers, `<th scope="row">` on feature name cells
- Check/X icons in comparison table: pair with visually-hidden text ("Supported" / "Not supported") for screen readers
- Color contrast: `#475569` on `#FFFFFF` = 5.9:1 (passes AA). `#0F172A` on `#F5F5F4` = 16.5:1 (passes AAA). `#FFFFFF` on `#F97316` = 3.1:1 (large text only — use for badge text at 14px bold only, pair with `font-semibold`)
- Touch targets: all interactive elements minimum 44px height
- Heading hierarchy: each page starts H1 (page title), uses H2 for sections, H3 for subsection headings — no skipped levels
- JSON-LD: rendered in `<head>` via component, not visible content — no accessibility impact
