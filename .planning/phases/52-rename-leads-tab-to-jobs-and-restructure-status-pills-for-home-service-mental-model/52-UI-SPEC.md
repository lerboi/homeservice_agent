### 1. LeadStatusPills.jsx — `PIPELINE_STATUSES` array

The array order changes (job-progression left-to-right) and one label changes. Colors, classes, count badges, markup shape: unchanged.

**Before:**
```
[new→"New", booked→"Booked", completed→"Completed", paid→"Paid", lost→"Lost"]
gap-2 between all pills
```

**After:**
```
[new→"New", booked→"Scheduled", completed→"Completed", paid→"Paid", lost→"Lost (ml-2 gap)"]
```

- Change `label: 'Booked'` to `label: 'Scheduled'`.
- Add `className: 'ml-2'` (or equivalent wrapper) to the Lost pill's `<button>` to produce the 8px visual separator from Paid. Implementation: add a `gap` property to the `PIPELINE_STATUSES` entry for `lost` and apply it conditionally in the map, OR wrap the Lost button in a `<span className="ml-2">` sibling element. Either approach is acceptable; prefer whichever keeps the array shape cleanest.
- The `aria-label` on the container div changes from `"Filter leads by status"` to `"Filter jobs by status"`.
- The `PIPELINE_STATUSES` constant name stays as-is (D-10).

## Interaction Contract

### Status pill strip

- Pill order (left-to-right): New → Scheduled → Completed → Paid → [8px gap] → Lost
- Lost pill has `ml-2` applied to create visual separation (D-03). This is a margin on the button element, not a divider element. Rationale: `ml-2` is the lighter touch; a divider element would add an extra DOM node to a `role="tablist"` container, which has accessibility implications. The `ml-2` treatment is invisible in the HTML structure but readable visually.
- Tap/click behavior: unchanged (toggle active filter; second tap on active pill clears filter).
- Count badge: retained on all 5 pills (D-04). Badge on Completed pill is the owner's "who owes me money" counter — no visual change, but its job-mental-model meaning is reinforced by the "Scheduled" rename that precedes it.
- `overflow-x-auto` scroll at 375px: unchanged, no new work.
- Focus ring: `focus:ring-2 focus:ring-[var(--brand-accent)] focus:ring-offset-1` — unchanged.

### Filter bar

- "Search name or phone..." placeholder: no change (does not contain "lead").
- "Filter jobs" sheet title on mobile: changed (see above).
- Active filter pill appearance: unchanged.
- "Clear all" link: unchanged.

### Flyout

- Status dropdown options: `new`, `booked`, `completed`, `paid`, `lost` (DB values unchanged). The option label for `booked` displays as `Scheduled` (via `STATUS_LABELS` map update).
- Toast timing and position: unchanged (sonner defaults).

### Navigation

- `DashboardSidebar.jsx`: active detection uses `pathname.startsWith(item.href)`. After rename to `/dashboard/jobs`, the active state activates for any path starting with `/dashboard/jobs`. No logic change needed — just the href value.
- `BottomTabBar.jsx`: same pattern, same requirement.
- 308 redirect (`next.config.js`): `{ source: '/dashboard/leads', destination: '/dashboard/jobs', permanent: true }`. Also add a wildcard redirect for deep links: `{ source: '/dashboard/leads/:path*', destination: '/dashboard/jobs/:path*', permanent: true }`.
