<objective>
Reframe the in-app AI chatbot's knowledge corpus from "Leads" to "Jobs" so the chatbot's natural-language answers and deep-link references match the renamed UI surface. Without this, owners would see the chatbot say "go to Leads" while the sidebar says "Jobs" — a credibility-destroying inconsistency that defeats the home-service mental model rename.

Purpose: D-13 in CONTEXT.md — the chatbot reads these markdown files at build time. Stale URLs survive via the 308 redirect (so the deep-links technically still work) but the chatbot's text answers would say "Leads" while the UI says "Jobs." This plan eliminates that drift.

Output: 8 chatbot-knowledge files updated to use "Jobs"/"jobs" + `/dashboard/jobs` paths, with the status pill label list updated to the new Phase 52 ordering wherever it appears.
</objective>
