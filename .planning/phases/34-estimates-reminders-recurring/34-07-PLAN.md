**src/lib/recurring-calculations.js** — Pure function for testability (matches Wave 0 test contract):

```javascript
import { addWeeks, addMonths, addYears } from 'date-fns';

/**
 * Calculate the next occurrence date from start_date to avoid drift.
 * Always computes from start_date + N*interval, not from currentNextDate + interval.
 *
 * @param {string} startDate - Original start date (YYYY-MM-DD)
 * @param {string} frequency - weekly|monthly|quarterly|annually
 * @param {string} currentNextDate - The current next_date being processed (YYYY-MM-DD), used as "today" reference
 * @returns {string} Next occurrence date (YYYY-MM-DD)
 */
export function calculateNextDate(startDate, frequency, currentNextDate) {
  const start = new Date(startDate + 'T00:00:00');
  const reference = new Date(currentNextDate + 'T00:00:00');

  const addFn = {
    weekly: (d, n) => addWeeks(d, n),
    monthly: (d, n) => addMonths(d, n),
    quarterly: (d, n) => addMonths(d, n * 3),
    annually: (d, n) => addYears(d, n),
  };

  let next = start;
  let n = 0;
  while (next <= reference) {
    n++;
    next = addFn[frequency](start, n);
  }
  return next.toISOString().split('T')[0];
}
```

**Recurring invoices cron** (`src/app/api/cron/recurring-invoices/route.js`):
GET handler following trial-reminders pattern:

1. CRON_SECRET Bearer auth check
2. Use service-role supabase client
3. Get today: `new Date().toISOString().split('T')[0]`
4. Query invoices WHERE:
   - `is_recurring_template = true`
   - `recurring_active = true`
   - `recurring_next_date <= today`
   - Optionally: `recurring_end_date IS NULL OR recurring_end_date >= today`
5. For each matching template:
   a. Fetch template's line items from `invoice_line_items`
   b. Fetch invoice_settings for the tenant (for prefix, tax_rate)
   c. Get next invoice number via `get_next_invoice_number` RPC
   d. Create new draft invoice:
      - Copy: customer_name, customer_phone, customer_email, customer_address, job_type, notes, payment_terms
      - Set: status='draft', invoice_number=new number, issued_date=today, due_date=calculate from payment_terms or NULL
      - Set: `generated_from_id = template.id`
      - Set: `is_recurring_template = false` (generated instances are NOT templates)
      - Calculate totals from line items
   e. Copy all line items to new invoice (new IDs, same data)
   f. Call `calculateNextDate(startDate, frequency, currentNextDate)` to get next date
   g. Update template: `recurring_next_date = nextDate`
   h. If `recurring_end_date` is set and nextDate > recurring_end_date: also set `recurring_active = false`
6. Return `{ generated: N }`
