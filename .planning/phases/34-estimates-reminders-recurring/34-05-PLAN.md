**Send route** (`src/app/api/estimates/[id]/send/route.js`):
Mirror the invoice send pattern. POST handler:

1. Fetch estimate + line items + tiers + invoice_settings (for business info + tax rate)
2. Generate PDF via `generateEstimatePDF(estimate, lineItems, tiers, settings)`
3. Send email via `getResendClient()`:
   - From: `{settings.business_name} <noreply@getvoco.ai>`
   - To: estimate.customer_email
   - Subject: `Estimate ${estimate.estimate_number} from ${settings.business_name}`
   - Attachments: `[{ filename: '${estimate.estimate_number}.pdf', content: pdfBuffer }]`
   - HTML body: Brief message with estimate summary (reuse or create simple template)
4. Send SMS via `getTwilioClient()` if estimate.customer_phone exists:
   - For single-price: `"{business_name}: Estimate #{number} for ${total}. Full estimate sent to your email. Questions? Call {phone}"`
   - For tiered: `"{business_name}: Estimate #{number} with options from ${min} to ${max}. Full estimate sent to your email. Questions? Call {phone}"`
   - SMS failure is non-fatal (try/catch, log error only, per existing invoice send pattern)
5. Update estimate: set `status = 'sent'`, `sent_at = now()`
6. Return `{ success: true }`

**Convert route** (`src/app/api/estimates/[id]/convert/route.js`):
POST handler per D-05. Accept optional body: `{ tier_id }` (for tiered estimates).

1. Fetch estimate with tenant_id guard
2. Check `converted_to_invoice_id` is NULL — if not NULL, return existing invoice ID (prevent double conversion per Research pitfall 5)
3. Determine line items to use:
   - If estimate has no tiers (single-price): use all estimate_line_items
   - If `tier_id` provided: use estimate_line_items WHERE tier_id matches, and get tier totals
   - If tiered but no tier_id: return 400 "tier_id required for tiered estimates"
4. Get next invoice number via `get_next_invoice_number` RPC
5. Insert new invoice with status='draft':
   - Copy customer_name, customer_phone, customer_email, customer_address, job_type from estimate
   - Copy subtotal/tax_amount/total (from estimate for single-price, from selected tier for tiered)
   - Set `issued_date = CURRENT_DATE`
6. Insert line items into `invoice_line_items` from the selected estimate line items (copy all fields except IDs)
7. Update estimate: set `converted_to_invoice_id = new_invoice.id`
8. Return `{ invoice_id: new_invoice.id, invoice_number }`
  </action>
