  <action>
**GET /api/estimates** (`src/app/api/estimates/route.js`):
Mirror the invoice list route pattern exactly. Use `createSupabaseServer()` + `getTenantId()`.

Query params: `status` (one of draft/sent/approved/declined/expired), `search` (customer_name or estimate_number substring), `lead_id` (filter by lead).

Return: `{ estimates, summary: { pending_count, approved_value, conversion_rate }, status_counts }`

Where:
- `pending_count` = count of estimates with status='sent'
- `approved_value` = SUM(total) of estimates with status='approved' (for tiered, use the estimate-level total which is NULL, so also query estimate_tiers SUM)
- `conversion_rate` = (approved count / (approved + declined + expired count)) * 100, or 0 if denominator is 0

For each estimate in the list, also fetch tier count: join with `estimate_tiers` to get tier count. If tier count > 0, include `tier_range: { min: lowest_tier_total, max: highest_tier_total }` so the list can display "$450 - $1,200" for tiered estimates.

Sort by `created_at DESC`.

**POST /api/estimates** (`src/app/api/estimates/route.js`):
Accept body: `{ lead_id?, customer_name, customer_phone, customer_email, customer_address, job_type, valid_until?, notes, line_items: [...], tiers?: [{ tier_label, line_items: [...] }] }`

1. Get `estimate_prefix` from `invoice_settings` (default 'EST')
2. Call `get_next_estimate_number` RPC to get next number
3. Format with `formatEstimateNumber(prefix, year, seqNum)`
4. If `tiers` array provided and length > 0:
   - Set estimate-level subtotal/tax_amount/total to NULL
   - Insert estimate row, then for each tier: insert into `estimate_tiers`, then insert tier's line items into `estimate_line_items` with `tier_id` set
   - Calculate per-tier totals using `calculateInvoiceTotals` with tax_rate from `invoice_settings`
5. If no tiers (single-price):
   - Calculate totals from `line_items` using `calculateInvoiceTotals`
   - Insert estimate row with totals, then insert line items with `tier_id = NULL`
6. Return the created estimate with its line items and tiers

**GET /api/estimates/[id]** (`src/app/api/estimates/[id]/route.js`):
Fetch estimate by ID with tenant_id guard. Join with `estimate_line_items` and `estimate_tiers` (ordered by sort_order). Return `{ estimate, line_items, tiers }`.

**PATCH /api/estimates/[id]**:
Accept partial updates. Handle status transitions per D-04:
- Set `sent_at` when status changes to 'sent'
- Set `approved_at` when status changes to 'approved'
- Set `declined_at` when status changes to 'declined'
- Recalculate totals if line_items provided

When `line_items` or `tiers` are provided in body:
- Delete existing line items and tiers
- Re-insert new ones (same atomic pattern as POST)

**DELETE /api/estimates/[id]**:
Delete estimate (cascades to line items and tiers). Only allow deletion if status is 'draft'.
