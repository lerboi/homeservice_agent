## Interaction Patterns

### Estimates List Page

1. **Navigation**: Estimates get their own page at `/dashboard/estimates`. Added to sidebar NAV_ITEMS as `{ href: '/dashboard/estimates', label: 'Estimates', icon: ClipboardList }` below the Invoices entry.
2. **Summary cards row**: 3 cards at top (Pending Estimates count, Approved Value amount, Conversion Rate percentage). Same grid layout as invoice summary cards.
3. **Status filter tabs**: Horizontal tab row: All / Draft / Sent / Approved / Declined / Expired. Same pattern as invoice status tabs.
4. **Estimate table**: Same column pattern as invoices: Estimate #, Customer, Job Type, Amount, Created, Valid Until, Status. Tiered estimates show amount range (e.g., "$450 - $1,200"). Rows clickable. Mobile: card layout.
5. **Empty state**: Centered. ClipboardList icon (lucide), heading, body, CTA button. Same layout pattern as invoice empty state.

### Estimate Editor

1. **Header section**: Estimate number (auto-generated, read-only with EST prefix), customer info fields (same as invoice editor), created date (default today), valid-until date (optional).
2. **Single-price mode (default)**: Identical to invoice editor line items section. One set of line items with subtotal/tax/total.
3. **Tiered mode**: "Add Tier" button below line items. When clicked, current line items become the first tier ("Good"). Up to 3 tiers. Each tier is a Card with: editable label input at top (default "Good"/"Better"/"Best"), its own set of line items via reused InvoiceEditor/LineItemRow, its own subtotal/tax/total. Tiers arranged vertically on mobile, side-by-side (3-col grid) on desktop (min-width 1024px).
4. **Removing tiers**: If only 1 tier remains and user removes it, revert to single-price mode with that tier's line items.
5. **Notes/Terms section**: Same as invoice editor.
6. **Action bar**: Same pattern as invoice editor. "Save as Draft" (outline), "Send Estimate" (primary/accent).

### Estimate Detail View

1. **Approach**: Full page at `/dashboard/estimates/[id]`.
2. **Layout**: Same 70/30 split as invoice detail. Left: estimate preview. Right: status badge, dates, actions, delivery history.
3. **Actions**: "Send Estimate" (if Draft), "Mark as Approved", "Mark as Declined", "Mark as Expired", "Convert to Invoice" (if Approved — primary CTA, accent color), "Download PDF", "Edit" (if Draft only).
4. **Convert to Invoice flow**: If single-price, one-click with confirmation dialog. If tiered, dialog shows tier cards side-by-side with radio selection, then creates draft invoice from selected tier. After conversion, success toast with "View Invoice" link.
5. **Mobile**: Single column, actions as sticky bottom bar.

### Payment Log (Invoice Detail Extension)

1. **Location**: New section on invoice detail view, between the invoice preview and the action buttons on the right column (desktop) or below the preview (mobile).
2. **Balance display**: Shows "Total: {total}" / "Payments: -{payments_sum}" / "Balance Due: {balance}" with Balance Due in Display typography and brandOrange color when > 0, emerald-600 when 0.
3. **"Record Payment" button**: Below balance display. Opens RecordPaymentDialog.
4. **RecordPaymentDialog**: Dialog with amount input (currency-formatted, auto-focus), date input (default today), note input (optional, placeholder text). "Record Payment" primary button, "Dismiss" outline button.
5. **Payment list**: Simple list below the balance. Each entry: date (left), amount (center), note (right, stone-500), delete icon button (Trash2, stone-400, hover:red-500, tooltip: "Remove payment", aria-label: "Remove payment"). Newest payment first. If no payments, show "No payments recorded" in stone-400 italic.

### Reminder Configuration (Invoice Detail Extension)

1. **Location**: Switch + description below the payment log section on invoice detail right column.
2. **Reminder toggle**: Switch component with label. Default ON for sent invoices with due date. Toggle persists immediately via PATCH.
3. **Schedule preview**: Below toggle, show next scheduled reminder date in stone-500 text. If reminders already sent, show sent history as a compact list (date + "Sent" badge).
4. **No separate settings page for reminders** — the schedule (3 days before, day-of, 3 days after, 7 days after) is fixed and described inline.

### Late Fee Settings (Invoice Settings Extension)

1. **Location**: New section in existing invoice settings page at `/dashboard/more/invoice-settings`, below the existing Tax Configuration section.
2. **Layout**: Switch to enable/disable. When enabled, reveal: fee type selector (flat/percentage), amount input. Flat shows currency input. Percentage shows number input with "% per month" suffix.
3. **Default state**: Late fees OFF.

### Recurring Invoices (Invoice Detail Extension)

1. **"Make Recurring" button**: On invoice detail view action area, outline style. Opens RecurringSetupDialog.
2. **RecurringSetupDialog**: Frequency selector (Select component with 4 options), start date (Popover date picker), optional end date (Popover date picker, clearable). "Set Up" primary button, "Dismiss" outline button.
3. **Recurring badge**: Once set, invoice shows RecurringBadge (lucide Repeat icon + "Recurring" text) next to the status badge. Badge uses `bg-violet-50 text-violet-700 border-violet-200`.
4. **Recurring info display**: Below the badge on detail view, show "Next invoice: {date}" and frequency. "Stop Recurring" link in stone-500 with confirmation dialog.
5. **Recurring filter**: Add "Recurring" tab to invoice status tabs. Shows only recurring templates (not generated instances). Generated instances show "Generated from recurring template" label in stone-400 text.
6. **Generated invoices**: Auto-generated drafts appear in the main invoice list as regular Draft invoices with a small "Recurring" indicator label.

### Void Invoice (Invoice Detail Extension)

1. **Location**: "Void Invoice" button in invoice detail view action area, destructive outline style (text-red-600, border-red-200, hover:bg-red-50).
2. **Confirmation**: AlertDialog with destructive confirm button. Copy per Copywriting Contract.
3. **After void**: Invoice status badge shows "Void" (red-50/red-700/red-200). All action buttons except "Download PDF" are hidden. Reminders and late fees stop.

### LeadFlyout Extension

1. **"Create Estimate" button**: New button in LeadFlyout action area, alongside existing "Create Invoice" button. Appears for leads in any status (estimates can be created pre-job). Uses brandOrange outline variant. Icon: ClipboardList (lucide).
2. **Pre-fill**: Navigates to `/dashboard/estimates/new?lead_id={id}`. Editor pre-fills customer name, phone, address, job type from lead data.

### Navigation Changes

1. **Sidebar**: Add `{ href: '/dashboard/estimates', label: 'Estimates', icon: ClipboardList }` to NAV_ITEMS after the Invoices entry.

## PDF Layout Contract (Estimates)

Extends Phase 33 PDF contract. Estimate PDF follows the same structure with these differences:

### Single-Price Estimate PDF

Identical to invoice PDF except:
- Title: "ESTIMATE" instead of "INVOICE"
- Number label: "Estimate #" instead of "Invoice #"
- Date labels: "Created" and "Valid Until" instead of "Issued" and "Due"
- Footer: "This estimate is valid until {valid_until_date}." instead of payment terms
- No "Total Due" — uses "Estimated Total"

### Tiered Estimate PDF

- Title: "ESTIMATE" with subtitle "Options for your consideration"
- Tiers rendered as side-by-side columns (2 or 3 columns depending on tier count)
- Each column: tier label as heading (14pt bold), line items table, subtotal, tax, tier total
- Tier columns separated by a thin vertical rule (0.5pt, stone-300)
- If 2 tiers: each column ~45% width with 10% gap
- If 3 tiers: each column ~30% width with 5% gaps
- Mobile/narrow PDF fallback: tiers stacked vertically with horizontal rule separator

### Tiered Estimate PDF Typography

| Element | Size | Weight |
|---------|------|--------|
| Tier label | 14pt | Bold |
| "Options for your consideration" | 10pt | Regular |
| All other elements | Same as invoice PDF | Same as invoice PDF |
