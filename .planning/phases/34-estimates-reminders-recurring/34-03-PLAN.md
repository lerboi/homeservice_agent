**GET** — List payments for an invoice:
```javascript
// Fetch payments ordered by payment_date DESC, created_at DESC
// Also fetch the invoice total

```

**POST** — Record a payment (per D-08):
Accept body: `{ amount, payment_date, note }`
- Validate: amount > 0, payment_date is a valid date
- Insert into `invoice_payments` with invoice_id from URL params, tenant_id
- After insert, recalculate balance and auto-update invoice status:
  - Fetch all payments for this invoice, sum amounts
  - Fetch invoice total
  - If balance (total - sum) <= 0: update invoice status to 'paid', set paid_at = now()
  - If balance > 0 and payments exist: update invoice status to 'partially_paid'
- Return: `{ payment, payments_sum, balance, status }`

**DELETE** — Remove a payment:
Accept query param: `?payment_id=uuid`
- Delete from `invoice_payments` WHERE id = payment_id AND tenant_id
- After delete, recalculate balance and auto-update invoice status:
  - If no payments remain: revert status to the previous logical status (if was partially_paid, check if overdue by due_date, else 'sent')
  - If payments remain but balance > 0: keep as 'partially_paid'
  - If payments remain and balance <= 0: keep as 'paid'
- Return: `{ success: true, payments_sum, balance, status }`

**Auto-status helper function** (shared by POST and DELETE):
```javascript
async function recalculateInvoiceStatus(supabase, invoiceId, tenantId) {
  // 1. Fetch invoice (total, due_date, status)
  // 2. Fetch SUM(amount) from invoice_payments WHERE invoice_id
  // 3. Calculate balance = total - payments_sum
  // 4. Determine new status:
  //    - balance <= 0 → 'paid'
  //    - payments_sum > 0 && balance > 0 → 'partially_paid'
  //    - payments_sum === 0 && due_date < today → 'overdue'
  //    - payments_sum === 0 → 'sent' (or keep current if draft/void)
  // 5. Only update status if it's not 'draft' or 'void' (don't auto-transition those)
  // 6. Update invoice status + paid_at (if newly paid)
  // Return { payments_sum, balance, status }
}
```
