8. **"PUT rejects invalid time format"** — day with `start: "9:00"` (missing leading zero) returns 400
9. **"PUT rejects start equals end time"** — day with `start: "09:00", end: "09:00"` returns 400
10. **"GET returns usage with null guard"** — when no calls exist, usage.used_minutes should be 0, not null or NaN

Create `src/app/api/call-routing/route.js` with GET and PUT handlers following the working-hours route pattern exactly.

**GET handler:**
1. Call `getTenantId()` — return 401 if null
2. Query tenants table: `.select('call_forwarding_schedule, pickup_numbers, dial_timeout_seconds, working_hours, phone_number, country').eq('id', tenantId).single()`
3. Query usage: `.from('calls').select('outbound_dial_duration_sec').eq('tenant_id', tenantId).gte('created_at', monthStart)` where `monthStart` is computed as:
   ```javascript
   const now = new Date();
   const monthStart = `${now.getUTCFullYear()}-${String(now.getUTCMonth() + 1).padStart(2, '0')}-01T00:00:00.000Z`;
   ```
4. Sum the outbound_dial_duration_sec values in JS (iterate rows, sum non-null values): `const usedSeconds = (usageRows || []).reduce((sum, r) => sum + (r.outbound_dial_duration_sec || 0), 0);`
5. Compute cap: `const capMinutes = tenant.country === 'SG' ? 2500 : 5000;`
6. Return JSON:
   ```javascript
   {
     call_forwarding_schedule: tenant.call_forwarding_schedule,
     pickup_numbers: tenant.pickup_numbers,
     dial_timeout_seconds: tenant.dial_timeout_seconds,
     usage: {
       used_seconds: usedSeconds,
       used_minutes: Math.floor(usedSeconds / 60),
       cap_minutes: capMinutes,
       country: tenant.country
     },
     working_hours: tenant.working_hours
   }
   ```

**PUT handler:**
1. Call `getTenantId()` — return 401 if null
2. Parse body: `const { call_forwarding_schedule, pickup_numbers, dial_timeout_seconds } = await request.json();`
3. Fetch tenant's phone_number for self-reference check: `.from('tenants').select('phone_number').eq('id', tenantId).single()`
4. Validate `dial_timeout_seconds`: must be integer, `>= 10`, `<= 30`. Return 400 with `{ error: 'Dial timeout must be between 10 and 30 seconds' }` if invalid.
5. Validate `call_forwarding_schedule`:
   - Must have `enabled` (boolean) and `days` (object)
   - Valid day keys: `['mon','tue','wed','thu','fri','sat','sun']`
   - Each day value must be an array of `{start, end}` objects
   - `start` and `end` must match `/^\d{2}:\d{2}$/` and `start !== end`
   - Return 400 with `{ error: 'Invalid schedule: {specific reason}' }` if invalid
6. Validate `pickup_numbers`:
   - Must be array with length <= 5. Return 400 `{ error: 'Maximum 5 pickup numbers allowed' }`
   - Each item must have `number` (string matching `E164_RE = /^\+[1-9]\d{1,14}$/`), `label` (string), `sms_forward` (boolean)
   - No duplicate `.number` values. Return 400 `{ error: 'Duplicate phone number: {number}' }`
   - No item where `.number === tenant.phone_number`. Return 400 `{ error: 'Cannot forward to your own Voco business number' }`
   - Invalid E.164: Return 400 `{ error: 'Invalid phone number format: {number}' }`
7. **Cross-field validation:** If `call_forwarding_schedule.enabled === true` AND `pickup_numbers.length === 0`, return 400 `{ error: 'Add at least one pickup number to route calls to you' }`
8. Update tenants: `.from('tenants').update({ call_forwarding_schedule, pickup_numbers, dial_timeout_seconds }).eq('id', tenantId).select('call_forwarding_schedule, pickup_numbers, dial_timeout_seconds').single()`
9. Return the updated values as JSON.

Wrap both handlers in try/catch, log errors with `console.log('500:', err.message)`, return 500 on unexpected errors.

Define `const E164_RE = /^\+[1-9]\d{1,14}$/;` and `const VALID_DAYS = ['mon','tue','wed','thu','fri','sat','sun'];` and `const TIME_RE = /^\d{2}:\d{2}$/;` at module top level.
