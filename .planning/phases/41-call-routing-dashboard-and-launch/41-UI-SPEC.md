## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact row spacing, toggle label gap |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Card padding (p-6), section padding |
| xl | 32px | Between section cards on call-routing page |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in dashboard) |

Exceptions: Touch targets on icon-only action buttons (edit/delete on pickup number rows) use minimum 44px hit area via `min-h-[44px] min-w-[44px]` even when the visible icon is 16px. Source: CONTEXT.md D-08, mobile usability.

## Page Structure

### `/dashboard/more/call-routing`

Single scrolling page. No tabs. Four section cards stacked vertically.

```
[Page heading: "Call Routing"]
[Page subtitle: "Route calls to yourself during business hours. AI answers when you're unavailable."]

[Card 1: Schedule & Timeout]
  [Master toggle row: "Route calls to me" Switch + "AI answers all calls" / "Routing active" label]
  [When master OFF: all day rows disabled, muted]
  [When master ON: day rows enabled]
  ["Copy from working hours" button — always visible, secondary style]
  [Day list: Mon–Sun, each row: Switch + day label + time range (start–end, native time inputs) OR muted "AI all day" if disabled]
  [Separator]
  [Dial timeout: "Ring for" label + Slider (10–30s) + "{N} seconds" value display]
  [Save button — right-aligned]

[Card 2: Pickup Numbers]
  [Header row: "Pickup Numbers" + "(N of 5)" counter right-aligned]
  [List of existing pickup number cards — each: phone formatted + label + SMS badge + edit icon + delete icon]
  [Inline edit form when editing (phone, label, SMS toggle)]
  [Add pickup number form (always at bottom, not a modal): phone input + label input + SMS forward checkbox + Add button]
  [Inline Alert warning if save attempted with 0 numbers and schedule enabled]

[Card 3: Usage This Month]
  [Label: "Outbound minutes used this month"]
  [Progress bar (full width) with color threshold]
  [Text below: "42 of 5,000 minutes used" (or 2,500 for SG)]
  [Muted note: "Resets on the 1st of each month"]

[Card 4: (implicit — no separate card needed)]
```

Source: CONTEXT.md D-05, D-06, D-07, D-08, D-09, D-10, D-12.

### Calls Page (`/dashboard/calls`) — Additive Change

Add `ROUTING_STYLE` map. Each `CallCard` renders the routing badge inline with urgency and outcome badges (same row). For `owner_pickup` routing mode, collapse the expandable panel to show only: formatted caller phone, call duration, "You handled this call directly." text, and a Call Back action. Recording player, transcript, urgency analysis, and booking outcome are hidden for owner-pickup cards.

## States and Interactions

### Call Routing Page

| State | Behavior |
|---|---|
| Loading | Full-page Skeleton — 3 skeleton blocks matching card heights |
| Master toggle OFF | All day rows opacity-40, pointer-events-none; schedule section shows "AI answers all calls" muted label |
| Master toggle ON | Day rows interactive; enabled days show time inputs; disabled days show "AI all day" muted |
| Add pickup number | Inline form at bottom of pickup numbers card, always visible (not toggled) |
| Editing pickup number | Row expands inline with pre-filled inputs; other rows remain visible |
| Delete pickup number | Immediate delete with sonner toast "Pickup number removed" — no confirmation dialog (non-destructive, easily re-added) |
| Save success | Sonner toast "Call routing settings saved" — 4s, no undo needed |
| Save error | Sonner toast "Failed to save — try again" |
| Zero numbers + enabled | `Alert` component (variant warning) inline above Add button: "Add at least one pickup number to route calls to you." Blocks PUT request. |
| Duplicate phone | Inline field error below phone input: "This number is already in your list." |
| Invalid E.164 | Inline field error: "Enter a valid phone number including country code (+1...)." |
| Self-reference | Inline field error: "You can't forward to your Voco business number." |
| Cap > 5 entries | Add button disabled, label changes to "Maximum 5 numbers reached" |
| Usage meter loading | Skeleton replaces progress bar |

Source: CONTEXT.md D-11, D-13, success criteria #7.

### Calls Page

| State | Behavior |
|---|---|
| `routing_mode = null` | No routing badge rendered — legacy calls are unaffected visually |
| `routing_mode = 'ai'` | Small stone badge "AI" rendered — low visual weight, does not compete with urgency/outcome badges |
| `routing_mode = 'owner_pickup'` | Blue badge "You answered" + collapsed expanded panel showing only caller + duration + "You handled this call directly." |
| `routing_mode = 'fallback_to_ai'` | Amber badge "Missed → AI" + full AI call details visible (the AI handled the call) |
