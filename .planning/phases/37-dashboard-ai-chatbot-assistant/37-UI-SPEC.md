## Layout Contracts

### Desktop — Right-side Sheet

- `side="right"` on SheetContent
- Width: `w-[400px]` — fixed, not fluid; dashboard content remains visible behind overlay
- Sheet height: full viewport height (`h-full`, inherited from sheet.jsx right-side default)
- Sheet header: 64px tall — "Voco AI" title (semibold 16px) + Bot icon + Badge "Help" on right side
- Message list: `flex-1 overflow-y-auto` — scrollable, padded `px-4 py-4`, `space-y-3` between bubbles
- Input row: fixed at sheet bottom — `px-4 pb-4 pt-2` — Input + Send Button side-by-side (Input flex-1, Button fixed 40px wide)
- Overlay: `bg-black/50` (default SheetOverlay) — dashboard content dimmed but still readable

### Mobile — Bottom Sheet

- `side="bottom"` on SheetContent
- Override class: `max-h-[85vh] rounded-t-2xl` (per D-04)
- Sheet fills bottom 85% of viewport on open
- Drag handle: 4px wide, 32px tall, `bg-stone-300 rounded-full mx-auto mt-3 mb-0` — visual affordance for swipe-to-dismiss
- Same header / message list / input layout as desktop, within the height constraint
- Swipe down to dismiss: inherited from Radix Dialog/Sheet behavior on touch

- `justify-start` row with 28px avatar slot on the left

- `inline-flex items-center gap-1 px-3 py-1.5 rounded-full text-xs font-medium`

- Three dots: `w-2 h-2 rounded-full bg-[#475569]` with staggered `animate-pulse` (0ms, 150ms, 300ms delay)

## Greeting Message Contract

On first open (empty messages array), the AI immediately renders one pre-loaded AI message without an API call:

> "Hi, I'm Voco AI. I can help you navigate the dashboard, understand your data, or answer questions about how features work. What would you like to know?"

This message is injected as a static initial state entry — no streaming, no skeleton.

## States and Interactions

| State | Visual Treatment |
|-------|-----------------|
| Sheet closed | Nothing visible — trigger button only |
| Sheet opening | Radix slide-in animation (300ms ease-in-out, inherited from sheet.jsx) |
| Empty chat (first open) | Greeting message pre-populated (see Greeting Message Contract) |
| User types | Input grows text inline, Send button activates (cursor-pointer) |
| User submits | User bubble appended immediately; TypingIndicator appears for AI slot |
| AI responding | TypingIndicator visible; input disabled (`disabled` attr on Input + Button) |
| AI response received | TypingIndicator replaced with AI message bubble; input re-enabled; auto-scroll to bottom |
| Navigation chip clicked | Sheet closes; router.push(href) navigates to target page |
| Groq error | TypingIndicator replaced with error AI bubble (see Copywriting Contract) |
| Sheet closing | Radix slide-out animation (300ms); message state preserved in React state (not cleared) |
| Tab close / refresh | Message history cleared (session-only, React state — no persistence per D-09) |

## Interaction Constraints

- Input field: `onKeyDown` — Enter key submits (no shift+Enter multiline; this is a help chatbot, not a composer)
- Input max length: 500 characters — enforced at input level (`maxLength={500}`)
- Send button: disabled when input is empty (trimmed) OR when AI is responding
- Auto-scroll: `useEffect` on messages array length change — `scrollIntoView({ behavior: 'smooth' })` on last message ref
- Sheet Z-index: z-50 (inherited from sheet.jsx) — renders above all dashboard content and mobile bottom nav
- Focus management: on sheet open, focus moves to Input field automatically (`autoFocus` on Input)
- Keyboard dismiss: Esc key closes sheet (inherited from Radix Dialog)

## Accessibility

- Sheet has `aria-label="Voco AI chat assistant"` on SheetContent
- SheetTitle "Voco AI" is the accessible dialog label (Radix wires this automatically)
- TypingIndicator has `aria-label="Voco AI is typing"` and `role="status"` for screen reader announcement
- AI message bubbles: no `aria-live` on individual bubbles — single `aria-live="polite"` region wraps the message list
- Navigation link chips are `<Link>` elements (not buttons) — keyboard navigable with focus ring `focus:ring-2 focus:ring-[#C2410C] focus:ring-offset-1`
- Send button icon-only with `<span className="sr-only">Send message</span>`
- Reduced motion: TypingIndicator uses `prefers-reduced-motion` — dots become static at 50% opacity instead of pulsing
