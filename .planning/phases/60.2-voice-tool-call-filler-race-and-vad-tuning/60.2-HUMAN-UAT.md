## Critical Finding — Fix H is a no-op in production

Railway logs contained three identical warnings during the call:
```
[check_availability] filler say() failed: trying to generate speech from text without a TTS model
[check_availability] filler say() failed: trying to generate speech from text without a TTS model
[book_appointment] filler say() failed: trying to generate speech from text without a TTS model
```

Root cause: `AgentSession` is wired as `AgentSession(llm=RealtimeModel)` (see `src/agent.py:255`). `livekit-agents==1.5.1` requires a TTS to back `session.say()`; a RealtimeModel-only session has no TTS. Every Fix H filler attempt raised; the `try/except` in each tool swallowed the error; the caller heard silence during tool execution.

The Plan 01 tests missed this because they mocked `context.session.say`. Gates 1–7 still passed because **Fix G alone** (VAD `silence_duration_ms` 1000 → 1500 ms) was sufficient to prevent the server-VAD cancellation race on this call — Gemini also kept emitting its own filler via the prompt-driven path that Plan 04 had *tried* to remove but didn't fully because the restored Plan-04-merged prompt still contained fallback tone guidance that Gemini picked up on.

## Secondary Finding — Goodbye cut-off

Caller reported: agent said "Thank you for calling" → **audio cut mid-sentence** → silence → caller hung up.

Log evidence:
```
20:37:17.969  closing agent session due to participant disconnect reason=CLIENT_INITIATED
20:37:17.980  _SegmentSynchronizerImpl.playback_finished called before text/audio input is done (text_done:false, audio_done:true)
```

This is NOT the same race as Fix H (tool-boundary VAD cancellation). It's a different, end-of-call race between model-emitted goodbye speech and either (a) `end_call` tool invocation in the same turn, (b) ambient-noise VAD trigger, or (c) model self-termination. The existing guardrail in `prompt.py:562-567` and the `end_call` tool description already tell Gemini to finish speaking first — Gemini didn't honour that on this call.

**Scope call:** this issue is deferred to Phase 60.3 — it needs diagnostic instrumentation (log `end_call` invocation timing + last-text-token timestamp) before a fix can be chosen. Blind prompt-hardening here would be churn given the user's directive to follow Gemini-Live + v1.5.1 best practices for the pinned plugin commit.
