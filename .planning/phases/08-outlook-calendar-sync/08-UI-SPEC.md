## Interaction States

### OAuth Connect Flow (both providers)

| State | Visual |
|-------|--------|
| Idle | Connect button enabled, normal styling |
| Connecting | Button shows `RefreshCw` spinner icon + "Connecting..." text, button disabled |
| Popup open | Button stays in connecting state; poll interval checks popup close |
| Success | Row updates to connected state, toast: "{Provider} Calendar connected" |
| OAuth error | Alert appears above card with error message + "Try Again" button |
| Admin consent denied (Outlook only) | Alert with admin consent copy (see copywriting contract) |

### Disconnect Flow (both providers)

| State | Visual |
|-------|--------|
| Idle | "Disconnect" button visible |
| Confirm dialog | AlertDialog opens with provider-specific copy |
| Disconnecting | Button shows spinner + "Disconnecting...", dialog closed |
| Success | Row updates to disconnected state, toast: "{Provider} Calendar disconnected" |
| Auto-promote | If other provider remains, it gets PRIMARY badge automatically (per D-04) |

### Make Primary Action

| State | Visual |
|-------|--------|
| Click | Immediate optimistic UI: badges swap, toast: "{Provider} set as primary calendar" |
| Error | Revert badges, toast error: "Could not change primary calendar. Try again." |

### Sync Status Display

| Status | Dot | Label | Trigger |
|--------|-----|-------|---------|
| synced | `bg-emerald-500` solid | "Up to date" `text-emerald-600` | Last sync < 5 min ago, no errors |
| syncing | `bg-[#C2410C] animate-pulse` | "Syncing..." `text-[#C2410C]` | Sync in progress |
| error | `bg-red-500` solid | "Sync error" `text-red-600` | Last sync failed |
| reauth | `bg-amber-500` solid | "Re-authorization required" `text-amber-600` | Token refresh failed |

Source: existing `SyncStatusDot` component in `CalendarSyncCard.js`

## Accessibility Contract

- AlertDialog: inherits Radix focus trapping and `aria-describedby` from shadcn AlertDialog
- Disconnect buttons: `aria-label="Disconnect {Provider} Calendar"` (distinguishes Google from Outlook for screen readers)
- Make Primary button: `aria-label="Make {Provider} Calendar the primary calendar"`
- Connect buttons in empty state: `aria-label="Connect {Provider} Calendar"`
- SyncStatusDot: `role="status" aria-label="{status label}"` (e.g., "Up to date", "Syncing", "Sync error")
- PRIMARY badge: `aria-label="Primary calendar"` on the badge element
- Provider icons: `aria-hidden="true"` (decorative, provider name is visible text)
- Focus ring: inherited from shadcn `outline-ring/50` base layer
- Touch targets: all buttons minimum 44px effective height via shadcn `size="sm"` (`h-9` = 36px) padded to 44px by row padding
- Color contrast: `#475569` on `#FFFFFF` = 5.07:1 (AA); `#C2410C` on `#FFFFFF` = 4.6:1 (AA for UI components); `#DC2626` on `#FFFFFF` = 4.0:1 (AA for large text -- used on buttons with text-red-600)

## Implementation Notes for Executor

1. **CalendarSyncCard rewrite scope**: The entire `CalendarSyncCard.js` component is rewritten. Extract `SyncStatusDot` into its own file or keep inline -- it is reused for both providers. Consider extracting `CalendarProviderRow` as a sub-component for DRY provider rendering.

2. **Status API change**: The `/api/calendar-sync/status` route must return both providers' status. Shape: `{ google: { connected, calendar_name, last_synced_at, is_primary } | null, outlook: { connected, calendar_name, last_synced_at, is_primary } | null }`.

3. **Disconnect API change**: The `/api/calendar-sync/disconnect` route must accept a `provider` body parameter (`'google' | 'outlook'`). After disconnect, if the other provider is still connected, auto-promote it to primary (D-04) server-side and return the updated status.

4. **New API route**: `/api/calendar-sync/set-primary` accepts `{ provider: 'google' | 'outlook' }`. Sets `is_primary = true` on the specified provider and `is_primary = false` on the other. Returns updated status.

5. **OAuth popup pattern**: Reuse the exact same `window.open` + `setInterval` poll pattern from the existing Google Calendar connect flow. The Outlook popup targets `/api/outlook-calendar/auth` instead of `/api/google-calendar/auth`.

6. **Provider icon rendering**: Use lucide `CalendarDays` for both providers but with different background colors (blue-50 for Google, sky-50 for Outlook). Alternatively, use inline SVG brand icons for Google and Microsoft logos if desired -- but CalendarDays with color differentiation is sufficient and matches the existing pattern.

7. **Admin consent error detection**: The Outlook OAuth callback route should detect `error=consent_required` or `error=interaction_required` query parameters and pass a `?admin_consent=true` flag when closing the popup window. The CalendarSyncCard reads `window.location` or a sessionStorage flag to show the admin consent error alert instead of the generic OAuth error.
