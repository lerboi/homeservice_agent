**Create `src/lib/scheduling/outlook-calendar.js`** mirroring the structure of `google-calendar.js` but using Microsoft Graph API:

1. **Imports:** `import { ConfidentialClientApplication } from '@azure/msal-node'` and `import { supabase } from '@/lib/supabase.js'`.

2. **Constants:**
   - `GRAPH_BASE = 'https://graph.microsoft.com/v1.0'`
   - `SCOPES = ['https://graph.microsoft.com/Calendars.ReadWrite', 'offline_access']`

3. **`getMsalClient()`** — Lazy-instantiated singleton (same pattern as OpenAI client in layer2-llm.js):
   ```javascript
   let _msalClient = null;
   function getMsalClient() {
     if (!_msalClient) {
       _msalClient = new ConfidentialClientApplication({
         auth: {
           clientId: process.env.MICROSOFT_CLIENT_ID,
           clientSecret: process.env.MICROSOFT_CLIENT_SECRET,
           authority: 'https://login.microsoftonline.com/common',
         },
       });
     }
     return _msalClient;
   }
   ```

4. **`graphFetch(urlOrPath, accessToken, options = {})`** — Centralized Graph REST wrapper:
   - If `urlOrPath` starts with 'https://', use as-is (for deltaLink URLs); otherwise prepend GRAPH_BASE
   - Set `Authorization: Bearer ${accessToken}` and `Content-Type: application/json`
   - On non-ok response, parse error and throw `Error(`Graph API ${res.status}: ${error.error?.message || res.statusText}`)`
   - For 204 responses, return null (no body)

5. **`getOutlookAuthUrl(tenantId)`** — Uses `getMsalClient().getAuthCodeUrl()` with:
   - `scopes: SCOPES`
   - `redirectUri: ${process.env.NEXT_PUBLIC_APP_URL}/api/outlook-calendar/callback`
   - `state: tenantId`

6. **`exchangeCodeForTokens(code)`** — Uses `getMsalClient().acquireTokenByCode()` with:
   - `code`, `scopes: SCOPES`, `redirectUri`
   - Returns `{ accessToken: tokenResponse.accessToken, account: tokenResponse.account }`
   - Also extracts refreshToken from `tokenResponse` (MSAL stores it in the cache)

7. **`refreshOutlookAccessToken(refreshToken)`** — Direct HTTP POST (serverless-safe, per Pitfall 3):
   - POST to `https://login.microsoftonline.com/common/oauth2/v2.0/token`
   - Body: `client_id, client_secret, refresh_token, grant_type=refresh_token, scope=https://graph.microsoft.com/Calendars.ReadWrite offline_access`
   - Returns `{ access_token, refresh_token, expires_in }`

8. **`getValidAccessToken(creds)`** — Helper:
   - If `creds.expiry_date > Date.now() + 300000` (5 min buffer), return `creds.access_token`
   - Otherwise call `refreshOutlookAccessToken(creds.refresh_token)`
   - Update `calendar_credentials` with new access_token, refresh_token, expiry_date
   - Return new access_token

9. **`createOutlookCalendarEvent({ credentials, appointment })`** — POST /me/events:
   - Build event body with `subject`, `start.dateTime`, `end.dateTime`, `location.displayName`, `singleValueExtendedProperties` with `platform_appointment_id`
   - Urgency prefix same as Google: `[URGENT] ` for emergency
   - Returns `event.id`

10. **`createOutlookSubscription(tenantId, accessToken)`** — POST /subscriptions:
    - `changeType: 'created,updated,deleted'`
    - `notificationUrl: ${NEXT_PUBLIC_APP_URL}/api/webhooks/outlook-calendar`
    - `resource: '/me/events'`
    - `expirationDateTime`: 7 days from now
    - `clientState: process.env.OUTLOOK_WEBHOOK_SECRET`
    - Persist `watch_channel_id = subscription.id`, `watch_expiration` to `calendar_credentials` with `.eq('provider', 'outlook')`

11. **`syncOutlookCalendarEvents(tenantId)`** — Delta query sync:
    - Load credentials with `.eq('tenant_id', tenantId).eq('provider', 'outlook').single()`
    - Get valid access token via `getValidAccessToken`
    - If `creds.last_sync_token` exists, use it as full deltaLink URL
    - Otherwise use `${GRAPH_BASE}/me/calendarView/delta?startDateTime=${now}&endDateTime=${sixMonths}` (per Pitfall 2)
    - Page through results using `@odata.nextLink`
    - Store `@odata.deltaLink` as `last_sync_token` (store FULL URL per anti-pattern warning)
    - Upsert non-removed events to `calendar_events` with `provider: 'outlook'`, `onConflict: 'tenant_id,provider,external_id'`
    - Delete events with `@removed` annotation
    - Update `last_synced_at` and `last_sync_token`

12. **`renewOutlookSubscription(cred)`** — PATCH /subscriptions/{id}:
    - Get valid access token
    - PATCH with new `expirationDateTime` (7 days)
    - Update `watch_expiration` in DB

13. **`revokeAndDisconnectOutlook(tenantId)`** — Disconnect flow:
    - Load credentials with `.eq('provider', 'outlook')`
    - DELETE subscription via `graphFetch(`/subscriptions/${creds.watch_channel_id}`, ...)` (try/catch, don't fail on 404)
    - Delete credentials row: `.eq('tenant_id', tenantId).eq('provider', 'outlook')`
    - Delete mirrored events: `.eq('tenant_id', tenantId).eq('provider', 'outlook')`
