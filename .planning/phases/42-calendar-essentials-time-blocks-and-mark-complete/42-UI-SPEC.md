## Interaction Contracts

### Time Block: Create Flow

1. User clicks "+" button in calendar toolbar (top-right area, 44px touch target, accent-colored icon)
   - OR clicks an empty time slot cell — Sheet opens with date/time pre-filled from click position
2. `TimeBlockSheet` opens from the right (side="right", max-w-md) — same as add-appointment Sheet
3. Fields in Sheet:
   - **Title** — `Input`, placeholder "Lunch break", required
   - **Date** — `Input` type="date", required
   - **Start time** — `Input` type="time", required (hidden when all-day is on)
   - **End time** — `Input` type="time", required (hidden when all-day is on)
   - **All-day** — `Switch` + `Label` inline row; toggling hides start/end time fields and sets times to span full working hours window
   - **Note** — `Textarea`, placeholder "Optional note", not required
4. SheetFooter: "Save Block" button (accent, full-width) + "Cancel" ghost button
5. On save: POST `/api/calendar-blocks` → optimistic UI update → Sheet closes → success toast "Time block saved"
6. Validation: If title empty or times invalid, show inline error below the field (`text-destructive text-12px`)

### Time Block: Edit/Delete Flow

1. User clicks a `TimeBlockEvent` on the calendar
2. Same `TimeBlockSheet` opens in edit mode — all fields pre-filled
3. Delete button appears in SheetFooter: `variant="destructive"` Button labeled "Delete Block"
   - No confirmation dialog. Clicking "Delete Block" immediately: DELETE `/api/calendar-blocks/:id` → optimistic remove → Sheet closes → toast "Time block deleted" with "Undo" action (5-second window, re-POSTs the block)
4. "Save changes" button (accent) saves edits via PATCH `/api/calendar-blocks/:id`

### Mark Complete Flow

1. User opens `AppointmentFlyout` by clicking an appointment on the calendar
2. "Mark Complete" button appears in SheetFooter alongside existing "Cancel Appointment" button
   - "Mark Complete": `variant="default"` with accent styling
   - "Cancel Appointment": existing destructive AlertDialog pattern (unchanged)
3. Clicking "Mark Complete":
   - An expandable `CompletionNotesInput` (`Textarea`) slides open below the button (animate height from 0 to auto, 200ms ease-out)
   - Placeholder: "Add completion notes (optional) — e.g., Replaced water heater thermostat"
   - A "Confirm Complete" button appears below the textarea
4. Clicking "Confirm Complete":
   - PATCH `/api/appointments/:id` with `{ status: "completed", completed_at: now, notes: appendedNotes }`
   - Flyout closes
   - Success toast: "Job marked as complete" with "Undo" action (5 seconds, reverts to `confirmed`)
5. If user clicks "Mark Complete" then clicks away without confirming — the textarea collapses, no action taken

### Completed Appointment Visual

- Existing urgency color block rendered at `opacity-40`
- Checkmark badge: absolute positioned `bottom-1 right-1`, 16x16px green circle with white check icon (lucide `Check` at 10px)
- Block remains clickable; flyout opens showing `completed` status
- In flyout: status badge shows "Completed" in `bg-green-100 text-green-700`; "Mark Complete" button is replaced by read-only "Completed" text with timestamp

### "Show Completed" Toggle

- Rendered above the calendar grid in the toolbar row, right-aligned
- `Switch` + `Label` inline: "Show completed jobs"
- Default state: ON (completed appointments visible)
- When toggled OFF: completed appointments disappear from calendar render (client-side filter, no network request)
- Toggle state persists in `localStorage` key `voco_calendar_show_completed` across sessions

### TimeBlockEvent Visual

- Absolute positioned, same coordinate math as `AppointmentBlock` (using `HOUR_HEIGHT = 64`, `DEFAULT_START`, `DEFAULT_END`)
- Full column width (left: 2px, right: 2px) — bypasses `layoutEventsInLanes`
- Renders BEHIND appointments: `z-index: 0` for time blocks, `z-index: 10` for appointments
- CSS stripe pattern:
  ```
  background: repeating-linear-gradient(
    45deg,
    transparent,
    transparent 4px,
    rgba(100, 116, 139, 0.25) 4px,
    rgba(100, 116, 139, 0.25) 8px
  ), #F1F5F9;
  ```
- Border-left: 3px solid `#94A3B8` (slate-400)
- Title text: 12px / weight 400 / `text-slate-600`, single line, truncated with ellipsis
- All-day blocks: span full working hours height (`(DEFAULT_END - DEFAULT_START) * HOUR_HEIGHT` = 832px)

## States

| State | Trigger | Visual |
|-------|---------|--------|
| Time block loading | Page fetch in progress | `Skeleton` bars matching time block height/width |
| Time block empty | No blocks in selected week | No element rendered — no empty state needed (absence is correct) |
| Time block saving | POST in flight | "Save Block" button shows `Loader2` spinner, disabled |
| Mark complete loading | PATCH in flight | "Confirm Complete" button shows `Loader2`, disabled |
| Completed (calendar) | `status === "completed"` | `opacity-40` + green checkmark badge on block |
| Completed (flyout) | `status === "completed"` | Status badge green, action buttons replaced by "Completed on {date}" |

## Accessibility

- All `Switch` components have an associated `Label` with `htmlFor` binding
- `TimeBlockSheet` traps focus when open (Radix Sheet handles this natively)
- "+" toolbar icon button has `aria-label="Add time block"`
- Delete button has `aria-label="Delete time block"` for screen readers
- Stripe CSS pattern has `aria-hidden="true"` on the pattern layer — the block title conveys meaning
- "Show completed" toggle has `aria-label="Show completed jobs on calendar"`
- Minimum contrast 4.5:1 for slate-600 on slate-100 (time block text on background)
- Completed appointment opacity-40 note: the block remains clickable and the flyout shows full detail — reduced opacity is purely decorative, not the only signal
